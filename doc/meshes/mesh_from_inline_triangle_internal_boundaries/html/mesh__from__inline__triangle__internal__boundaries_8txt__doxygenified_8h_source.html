3 \mainpage Creation of internal boundaries for Triangle based meshes
4 
5 In this document we show how to add internal boundaries and create
6 connections among them in 2D unstructured meshes with \c oomph-lib.
7 The mesh generation procedure is performed by using
8 <A HREF="http://www.cs.berkeley.edu/~jrs/">Jonathan Shewchuk's</A>
9 <A HREF="http://www.cs.cmu.edu/~quake/triangle.html"> Triangle</A>.
10 This tutorial complements the discussion in
11 <A HREF="../../../../doc/meshes/mesh_from_inline_triangle/html/index.html">
12 another tutorial</a>
13 where we illustrated how to build \c oomph-lib meshes using the
14 \c TriangleMeshPolygon and \c TriangleMeshClosedCurve objects.
15 
16 <HR>
17 
18  Additionally, we show via examples
19 - how to connect \c TriangleMeshCurveSection objects
20 - how to add addition holes and extra regions, i.e. regions not defined
21  via \c TriangleMeshPolygon or \c TriangleMeshClosedCurve objects

49 \section connections Creating connections among current boundaries in the domain and with internal boundaries
50 
51 Any internal boundary can be connected (via its ends) to any other
52 boundary currently in the domain.
53 
54 It is important to note that there are four different types of
55 connections:
56 - Connection between a \c TriangleMeshPolyLine and a \c TriangleMeshPolyLine
57 - Connection between a \c TriangleMeshCurviLine and a \c TriangleMeshPolyLine
58 - Connection between a \c TriangleMeshPolyLine and a \c TriangleMeshCurviLine
59 - Connection between a \c TriangleMeshCurviLine and a \c TriangleMeshCurviLine
60 
61 The next figure shows an sketch of the different types of connections.
62 
63 \image html connections.gif "Different types of connections. a) Connection between a \c TriangleMeshPolyLine and a \c TriangleMeshPolyLine. b) Connection between a \c TriangleMeshCurviLine and a \c TriangleMeshPolyLine. c) Connection between a \c TriangleMeshPolyLine and a \c TriangleMeshCurviLine. d) Connection between a \c TriangleMeshCurviLine and a \c TriangleMeshCurviLine. "
64 \image latex connections.eps "Different types of connections. a) Connection between a \c TriangleMeshPolyLine and a \c TriangleMeshPolyLine. b) Connection between a \c TriangleMeshCurviLine and a \c TriangleMeshPolyLine. c) Connection between a \c TriangleMeshPolyLine and a \c TriangleMeshCurviLine. d) Connection between a \c TriangleMeshCurviLine and a \c TriangleMeshCurviLine. " width=0.75\textwidth
65 
66 Connections to \c TriangleMeshPolyLine can only be made by using existing vertices
67 of the destination \c TriangleMeshPolyLine. Conversely, connections to \c TriangleMeshCurviLine
68 can be made by specifying the position along the \c TriangleMeshCurviLine. The
69 connection point is identified by the \c TriangleMeshCurviLine's intrinsic coordinates. The
70 figure below shows the case when a connection with a \c TriangleMeshPolyLine is possible.
71 
72 \image html figure_poly.gif "In order to connect a boundary to a \c TriangleMeshPolyLine it is necessary that the destination boundary has a vertex on the connection point. a) There is no vertex for connecting to the destination \c TriangleMeshPolyLine. b) There is a vertex in the destination \c TriangleMeshPolyLine to receive the initial end of the source boundary. "
73 \image latex figure_poly.eps "In order to connect a boundary to a \c TriangleMeshPolyLine it is necessary that the destination boundary has a vertex on the connection point. a) There is no vertex for connecting to the destination \c TriangleMeshPolyLine. b) There is a vertex in the destination \c TriangleMeshPolyLine to receive the initial end of the source boundary. 137 <b> Adding an internal boundary connected to an outer boundary </b>
138 
139 We need to add three more internal boundaries which have connections with the outer
140 boundary and with some internal boundaries; see the sketch of the domain.
141 
142 In order to connect an internal boundary with any other boundary in the domain
143 it is necessary to specify the following:
144 
145 - Is the initial or the final end of the boundary the one that we want to
146 connect?
147 
148 - Do we want to connect the boundary with a \c TriangleMeshPolyLine or with a
149 \c TriangleMeshCurviLine?
150 
151 - If we want to connect with a \c TriangleMeshPolyLine then we need to
152 identify the corresponding vertex number to which we want to connect. If we
153 want to connect with a \c TriangleMeshCurviLine we need to identify the
154 corresponding intrinsic coordinate \f$ s \f$ on the curve for connection.
155 
156 <HR>
157 
158 Consider the creation of the internal boundary 4 shown on the sketch of the domain.
159 This is an example of connecting a \c TriangleMeshCurviLine with a \c TriangleMeshPolyLine
160 (the outer boundary). In this case both ends of the \c TriangleMeshCurviLine are
161 connected so we need to use the interface for connecting the initial and the final
162 ends of the boundary to a \c TriangleMeshPolyLine. There is only one thing left to
163 identify, the vertex number of the destination boundary to which we want to connect.
164 By looking at figure 1.5 we see that these values are vertex #3 for the initial end
165 and vertex #1 for the final end. Once identified, we perform the connection as follow:

229 \subsection def_extra Defining additional holes and regions
230 
231 It is possible to define holes in the domain using the methods discussed in
232 <A HREF="../../../../doc/meshes/mesh_from_inline_triangle/html/index.html">
233 another tutorial</a>; where we explained how to create holes by using closed
234 curves. Holes could also be created by specifying additional parameters on the
235 \c TriangleMeshParameters object. The specification of regions or layers
236 follows the same principle.
237

255 \subsubsection def_extra_regions Defining regions
256 
257 The definition of regions follows the same principle that defining holes, it means,
258 one only need to specify the coordinates of a point that lies inside a region.
259 If we would like to define the region showed on light grey on the figure below
260 we use the method \c add_region_coordinates of the \c TriangleMeshParameters
261 object. You can specify any region id for the defined region.

280 \section ex Comments and exercises
281 -# Create two new internal boundaries without connections, one using a
282 \c TriangleMeshPolyLine object and other using a \c TriangleMeshCurviLine object.
283  \n\n
284 -# Define a new vertex on the new \c TriangleMeshPolyLine and create a connection
285  to that vertex.
286  \n\n
287 -# Once one performs a connection to a \c TriangleMeshPolyLine or to a
288  \c TriangleMeshCurviLine there is an internal checking on the connection
289  vertices coordinates for the \c TriangleMeshPolyLine and on the connection
290  intrinsic coordinate for the \c TriangleMesheCurviline case, they must be
291  close enough in order to allow the connection. The tolerance values are
292  \c tolerance_for_vertex_connection = \f$ 1.0e-14 \f$ for the
293  \c TriangleMeshPolyline case and \c tolerance_for_zeta_connection = \f$ 1.0e-14 \f$
294  for the \c TriangleMeshCurviline case. One can explicitly define the allowed
295  tolerance by adding an extra argument when using the methods
296  \c connect_initial_vertex_to_polyline, \c connect_final_vertex_to_polyline
297  \c connect_initial_vertex_to_curviline or \c connect_final_vertex_to_curviline.
298  As an exercise, use different values for the allowed connection tolerance.
