3 \mainpage Mesh generation based on Geompack++
4 
5 In this document we demonstrate how to generate quadrilateral \c oomph-lib
6 meshes, based on the output from Barry Joe's
7 mesh generator
8 <A HREF="http://members.shaw.ca/bjoe/">\c Geompack++, </A>
9 available as freeware at
10 <A HREF="http://members.shaw.ca/bjoe/">
11 http://members.shaw.ca/bjoe/.</A>
12 The mesh generation is performed in a two-stage process.
13 First we use <A HREF="http://members.shaw.ca/bjoe/">\c
14 Geompack++ </A> to generate the mesh "offline". Then we
15 process the output files generated by
16 <A HREF="http://members.shaw.ca/bjoe/">\c Geompack++ </A>
17 to generate an \c oomph-lib mesh.

22 \section geompack_use Example of the use of Geompack++
23 
24 The documents \c meshoper.pdf and \c regmesh.pdf distributed with
25 <A HREF="http://members.shaw.ca/bjoe/">\c Geompack++ </A>
26 contain a comprehensive User's Guide for the
27 code and its many options.
28 <A HREF="http://members.shaw.ca/bjoe/">\c Geompack++ </A>
29 is a very sophisticated mesh generator and can be used to create
30 2D and 3D meshes with a wide variety of element types.
31 Here we only discuss how to use
32 <A HREF="http://members.shaw.ca/bjoe/">\c Geompack++ </A>
33 to generate 2D quadrilateral meshes. [The development of
34 of \c oomph-lib meshes that process
35 <A HREF="http://members.shaw.ca/bjoe/">\c Geompack++'s </A>
36 output for other element types should be straightforward but is
37 still on our "To Do" list -- any volunteers?]
38 
39 
40 
41 <A HREF="http://members.shaw.ca/bjoe/">\c
42  Geompack++ </A>
43  creates quadrilateral meshes, based on the information
44 about the mesh boundaries (nodes and curves) provided in two input files,
45 \c filename.rg2 and \c filename.cs2, say. An output file,
46 \c filename.1.mh2 is created. It contains the information about the
47 mesh boundaries, the nodal positions and the element connectivity
48 lists.

53 \subsection geomview How to visualise a mesh generated by Geompack++
54 
55 To visualise the mesh, the program \c Geomview is available.
56 (\c Geomview can be downloaded
57 from the <A HREF="http://www.geomview.org/">\c Geomview home page
58 </A>.) The
59 <A HREF="http://members.shaw.ca/bjoe/">\c Geompack++
60 </A> output file cannot be processed directly by
61 <A HREF="http://www.geomview.org/">
62 \c Geomview </A> but a simple program that converts the
63 file \c filename.mh2 to a format that is readable by
64 <A HREF="http://www.geomview.org/">\c Geomview
65 </A> is
66 <A HREF="../../../../demo_drivers/meshing/mesh_from_geompack/convert_geom_file.cc">
67 available.</A>

71 \subsection example The example: A rectangle with a hole
72 
73 As an example we demonstrate how to use
74 <A HREF="http://members.shaw.ca/bjoe/">\c Geompack++
75 </A> to generate a mesh for the rectangular domain with a hole shown in the
76 figure below. The domain is defined by eight nodes and eight curves
77 (straight lines) which connect the nodes and define two boundaries.
78 The boundary numbers shown in the sketch correspond to those in the
79 <A HREF="http://members.shaw.ca/bjoe/">\c Geompack++ </A>
80 input region file
81 <A HREF="../../../../demo_drivers/meshing/mesh_from_geompack/box_hole.rg2">
82 box_hole.rg2 </A> and in the curve input
83 file <A HREF="../../../../demo_drivers/meshing/mesh_from_geompack/box_hole.cs2">
84 box_hole.cs2.</A> In the corresponding \c oomph-lib mesh, the boundaries are
85 numbered from zero.

114 \section oomph_use Creating an oomph-lib mesh based on output files generated by Geompack++
115 
116 \c oomph-lib provides a mesh, \c GeompackQuadMesh, that uses the
117 output from <A HREF="http://members.shaw.ca/bjoe/">
118 \c Geompack++ </A> to generate an \c oomph-lib Mesh containing
119 \c QElement<2,2> four-node quadrilateral elements. The relevant interface is:
120 
121 \dontinclude geompack_mesh.template.h
122 \skipline start_of_geompackquadmesh_class
123 \until Default_TimeStepper
124 
125 The driver code <A HREF="../../../../demo_drivers/meshing/mesh_from_geompack/mesh_from_geompack_poisson.cc">mesh_from_geompack_poisson.cc</A>
126 demonstrates the use of this mesh for the solution of a
127 2D Poisson problem in the "rectangular domain with a hole",
128 described in the previous section.
129 
130 The code expects the names of \c *.\c mh2 and *.\c cs2 files
131 generated by
132 <A HREF="http://members.shaw.ca/bjoe/">\c Geompack++
133 </A> as command line arguments and stores them
134 in the namespace \c CommandLineArgs
135 
136 \dontinclude mesh_from_geompack_poisson.cc
137 \skipline int main
138 \until }
139 
140 The names of these files are then passed to the mesh constructor.
141 Since the rest of the <A HREF="../../../../demo_drivers/meshing/mesh_from_geompack/mesh_from_geompack_poisson.cc">driver
142 code </A> is identical to that in the
143 <A HREF="../../../../doc/poisson/two_d_poisson/html/index.html">
144 corresponding example with a structured mesh</A>,
145 we do not provide a detailed code listing but simply show the plot of the
146 computed results, together with the tanh-shaped exact
147 solution of the problem:
148 
149 \image html validate.gif "Computed and exact solutions. "
150 \image latex validate.eps "Computed and exact solutions. " width=0.75\textwidth
