3\mainpage Hijacking degrees of freedom in oomph-lib
4
5 Certain problem formulations require changing the
6equation corresponding to (having the same global number as)
7a particular degree of freedom.
8
9For example,
10in steady free surface problems, there is a constraint on the total
11fluid volume, but no "obvious" associated degree of freedom. For these
12problems, a fluid pressure degree of freedom may be "traded" for the
13volume constraint. The "traded" degree of
14freedom remains a fluid pressure and must appear in the
15governing equations in that role. However, the equation associated
16with that unknown is no longer the continuity equation, but the volume
17constraint, see <a href="../../navier_stokes/static_single_layer/html/index.html">
18another tutorial </a> for more details.
19
20A similar situation arises when applying complex boundary conditions:
21perhaps the fluid velocity on the boundary is itself the solution of
22an equation depending on a local chemical concentration. One way of
23implementing such boundary conditions is to introduce a new
24variable representing the boundary velocity, solve the additional
25equation for that velocity and then impose that as a Dirichlet
26condition. However, the condition can be implemented without introducing a new
27variable by changing the equation associated with the fluid velocity
28on the boundary.
29
30In \c oomph-lib, this type of process is known as \b hijacking and uses the \c
31Hijacked wrapper class, which adds new member functions to the element:
32\code
33 Data* hijack_internal_value(const unsigned &n, const unsigned &i)
34 Data* hijack_external_value(const unsigned &n, const unsigned &i)
35 Data* hijack_nodal_value(const unsigned &n, const unsigned &i)
36 Data* hijack_nodal_position_value(const unsigned &n, const unsigned &i)
37 Data* hijack_nodal_spine_value(const unsigned &n, const unsigned &i)
38\endcode
39
40 Calling any of these functions instructs the bulk element to set the specified
41residual and associated row in the Jacobian matrix to zero. For example,
42\code
43 Hijacked<QPoissonElement<1,2> > *el_pt = new Hijacked<QPoissonElement<1,3> >;
44 el_pt->hijack_nodal_value(0,0);
45\endcode
46sets to zero the residual associated with the first data value stored at the
47first (local) node in the element.
48
49Note that
50hijacking is on a per element basis, so that if a degree of freedom
51is shared between elements it can be hijacked in all, none or some of
52them, as required.
53
54The return type of each member function is a pointer to a \c Data
55object. This \c Data object is a "custom" object that consists of a
56(copy of a)
57single value corresponding to the "hijacked" degree of freedom and its
58global equation number. This object is provided so that it can be used
59as external data in other elements that assemble the "new" equations
60for the unknown, see <a href="../../navier_stokes/static_single_layer/html/index.html">
61another tutorial </a> for an example. Note that this can
62lead to memory leaks unless the new object is safely deleted, so if
63you do not want to make use of the \c Data object you should precede
64the call to the member function by \c delete, e.g.
65\code
66delete el_pt->hijack_internal_value(0,0);
67\endcode

72\section notes Notes and Guidelines
73
74- If you do not assign a new equation to the "hijacked" unknown the
75 Jacobian matrix will be singular because there will be a zero row.
76
77- Hijacking should take place before equation numbering.
78
79- The member function \c Hijacked::unhijack_all_data() can be used to
80 reset all hijacked values in an element.
81
82- Hijacking will persist throughout adaptation, but only if the degree
83 of freedom remains "active". It is safest to \c unhijack_all_data()
84 before adaptation and then hijack again in \c actions_after_adapt().
85
86- All free surface elements are actually \c Hijacked, because
87 hijacking is used to enforce the contact angle condition in certain
88 circumstances.
89
90- The member function
91 \code
92 double* &Hijacked::residual_multiplier_pt()
93 \endcode
94 can be used to set a double value that multiplies the hijacked residuals. The
95 default and most common case it that the multiplier is zero, but
96 non-zero values may be useful in homotopy continuation calculations
97 in order to move smoothly between different types of boundary condition.
98.
