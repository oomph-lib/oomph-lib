3\mainpage Example problem: Adaptive solution of the 3D Poisson equation in a spherical domain
4
5Following the numerous 2D problems discussed in earlier examples we now
6demonstrate that the solution of 3D problems is just as easy.
7For this purpose we discuss the adaptive solution of the 3D Poisson problem
8<CENTER>
9<TABLE>
10<TR>
11<TD>
12<CENTER>
13<B>Three-dimensional model Poisson problem</B>
14</CENTER>
15Solve
16\f[
17\sum_{i=1}^3 \frac{\partial^2u}{\partial x_i^2} = f(x_1,x_2,x_3),
18 \ \ \ \ \ \ \ \ \ \ (1)
19\f]
20in the "eighth-of-a-sphere" domain \f$D\f$,
21with Dirichlet boundary conditions
22\f[
23\left. u\right|_{\partial D}=u_0
24\ \ \ \ \ \ \ \ \ \ (2)
25\f]
26where the function \f$ u_0 \f$ is given.
27</TD>
28</TR>
29</TABLE>
30</CENTER>
31
32We choose a source function and boundary conditions for which
33\f[
34u_0(x_1,x_2,x_3) =
35\tanh\left(1-\alpha(({\bf x}-{\bf x_0}) \cdot {\bf N})\right),
36\ \ \ \ \ \ \ \ \ (3)
37\f]
38is the exact solution. Here where \f${\bf x} = (x_1,x_2,x_3)\f$
39is the vector of the spatial coordinates, and the vectors
40\f${\bf x}_0 = \left(x_1^{(0)},x_2^{(0)},x_3^{(0)}\right)\f$ and
41\f${\bf N}_0 = (N_1,N_2,N_3)\f$ are constants. For large values of the
42constant \f$ \alpha \f$ the solution varies rapidly across the plane through
43\f${\bf x}_0\f$ whose normal is given by \f${\bf N}\f$.
44
45
46Here are some plots of the exact and computed solutions for
47\f${\bf x}_0 = (0,0,0)\f$, \f${\bf N}_0 = 1/\sqrt{3} \ (-1,-1,1)\f$,
48and \f$ \alpha=50\f$ at various levels of mesh refinement.
49Note that the plot of the exact solution was produced by
50setting the nodal values to the exact solution, obtained by evaluating
51(3) at the nodal positions. The elements' basis functions
52were then used to interpolate between the nodal values. On the coarse
53meshes, the interpolation between the "exact" nodal values is clearly
54inadequate to resolve the rapid variation of the solution.
55

78\section main The driver code
79
80The driver code solves the 3D Poisson problem with full spatial
81adaptivity -- a fairly time-consuming process. To minimise the
82run-times when the code is executed during \c oomph-lib's self-tests,
83we use command line arguments to optionally limit the number of
84adaptive refinements. If the code is run with a(ny) command line
85arguments, only a single adaptive refinement is performed;
86otherwise up to four levels of refinement are permitted.
87\c oomph-lib provides storage for the command line arguments
88in the namespace \c CommandLineArgs to make them accessible to
89other parts of the code.
90
91 Otherwise the driver code is very similar to that used in the
92<A HREF="../../../poisson/fish_poisson/html/index.html">corresponding 2D Poisson problems</A>: We construct the problem, passing the
93pointer to the source function. Next, we create a \c DocInfo object to
94specify the output directory, and execute the global self-test to
95assert that the problem has been set up correctly. Next we
96solve the problem on the coarse initial mesh (comprising
97four 27-node brick elements) and then adapt the problem
98based on the elemental error estimates, until the maximum number of
99adaptations has been reached or until the adaptation ceases to
100changes the mesh.
101
102\dontinclude eighth_sphere_poisson.cc
103\skipline start_of_main
104\until end of main

129\section constructor The Problem constructor
130In the \c Problem constructor, we set the "steepness parameter"
131\f$ \alpha \f$ to a large value and create the mesh for a
132a sphere of radius 5. Next, we create the error estimator and
133pass it to the adaptive mesh.
134
135\skipline start_of_constructor
136\until error_estimator_pt;
137
138We adjust the targets for the mesh adaptation so that
139the single mesh adaptation performed during a validation run
140produces a non-uniform refinement pattern. (The error targets
141for this case were determined by trial and error.) The tighter
142error tolerances specified otherwise are appropriate to
143properly resolve the solution, as shown in the animated gif files
144at the beginning of this document.
145\skipline Adjust error targets for adaptive refinement
146\until end adjustment
147
148 Next, we assign the boundary conditions. In the present problem all boundaries
149are Dirichlet boundaries, therefore we loop over all nodes
150on all boundaries and pin their values. If only a subset of the mesh
151boundaries were of Dirichlet type, only the nodes on those boundaries
152would have to be pinned. "Usually" the numbering of the mesh
153boundaries is (or at least should be!) documented in the mesh
154constructor but it can also be obtained from the function
155\c Mesh::output_boundaries(...) whose use is illustrated here.
156
157\skipline Doc the mesh
158\until end of
159
160
161Finally we loop over all elements to assign the source function
162pointer, and then call the generic \c
163Problem::assign_eqn_numbers() routine to set up the equation
164numbers.
165
166
167\skipline Find number
168\until end of constructor

176\section doc Post-processing
177The function \c doc_solution(...) writes the FE solution and the corresponding
178exact solution, defined in \c TanhSolnForPoisson::get_exact_u(...)
179to disk. The \c DocInfo object specifies the output directory
180and the label for the file names. [See the discussion of the
181<A HREF="../../../poisson/one_d_poisson/html/index.html">
1821D Poisson problem</A> for a more detailed discussion of the
183generic \c Mesh member functions \c Mesh::output(...),
