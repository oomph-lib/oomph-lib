3 \mainpage Example problem: Adaptive solution of the 3D Poisson equation in a spherical domain
4 
5 Following the numerous 2D problems discussed in earlier examples we now
6 demonstrate that the solution of 3D problems is just as easy.
7 For this purpose we discuss the adaptive solution of the 3D Poisson problem
8 <CENTER>
9 <TABLE>
10 <TR>
11 <TD>
12 <CENTER>
13 <B>Three-dimensional model Poisson problem</B>
14 </CENTER>
15 Solve
16 \f[
17 \sum_{i=1}^3 \frac{\partial^2u}{\partial x_i^2} = f(x_1,x_2,x_3),
18  \ \ \ \ \ \ \ \ \ \ (1)
19 \f]
20 in the "eighth-of-a-sphere" domain \f$D\f$,
21 with Dirichlet boundary conditions
22 \f[
23 \left. u\right|_{\partial D}=u_0
24 \ \ \ \ \ \ \ \ \ \ (2)
25 \f]
26 where the function \f$ u_0 \f$ is given.
27 </TD>
28 </TR>
29 </TABLE>
30 </CENTER>
31 
32 We choose a source function and boundary conditions for which
33 \f[
34 u_0(x_1,x_2,x_3) =
35 \tanh\left(1-\alpha(({\bf x}-{\bf x_0}) \cdot {\bf N})\right),
36 \ \ \ \ \ \ \ \ \ (3)
37 \f]
38 is the exact solution. Here where \f${\bf x} = (x_1,x_2,x_3)\f$
39 is the vector of the spatial coordinates, and the vectors
40 \f${\bf x}_0 = \left(x_1^{(0)},x_2^{(0)},x_3^{(0)}\right)\f$ and
41 \f${\bf N}_0 = (N_1,N_2,N_3)\f$ are constants. For large values of the
42 constant \f$ \alpha \f$ the solution varies rapidly across the plane through
43 \f${\bf x}_0\f$ whose normal is given by \f${\bf N}\f$.
44 
45 
46 Here are some plots of the exact and computed solutions for
47 \f${\bf x}_0 = (0,0,0)\f$, \f${\bf N}_0 = 1/\sqrt{3} \ (-1,-1,1)\f$,
48 and \f$ \alpha=50\f$ at various levels of mesh refinement.
49 Note that the plot of the exact solution was produced by
50 setting the nodal values to the exact solution, obtained by evaluating
51 (3) at the nodal positions. The elements' basis functions
52 were then used to interpolate between the nodal values. On the coarse
53 meshes, the interpolation between the "exact" nodal values is clearly
54 inadequate to resolve the rapid variation of the solution.

78 \section main The driver code
79 
80 The driver code solves the 3D Poisson problem with full spatial
81 adaptivity -- a fairly time-consuming process. To minimise the
82 run-times when the code is executed during \c oomph-lib's self-tests,
83 we use command line arguments to optionally limit the number of
84 adaptive refinements. If the code is run with a(ny) command line
85 arguments, only a single adaptive refinement is performed;
86 otherwise up to four levels of refinement are permitted.
87 \c oomph-lib provides storage for the command line arguments
88 in the namespace \c CommandLineArgs to make them accessible to
89 other parts of the code.
90 
91  Otherwise the driver code is very similar to that used in the
92 <A HREF="../../../poisson/fish_poisson/html/index.html">corresponding 2D Poisson problems</A>: We construct the problem, passing the
93 pointer to the source function. Next, we create a \c DocInfo object to
94 specify the output directory, and execute the global self-test to
95 assert that the problem has been set up correctly. Next we
96 solve the problem on the coarse initial mesh (comprising
97 four 27-node brick elements) and then adapt the problem
98 based on the elemental error estimates, until the maximum number of
99 adaptations has been reached or until the adaptation ceases to
100 changes the mesh.

129 \section constructor The Problem constructor
130 In the \c Problem constructor, we set the "steepness parameter"
131 \f$ \alpha \f$ to a large value and create the mesh for a
132 a sphere of radius 5. Next, we create the error estimator and
133 pass it to the adaptive mesh.
134 
135 \skipline start_of_constructor
136 \until error_estimator_pt;
137 
138 We adjust the targets for the mesh adaptation so that
139 the single mesh adaptation performed during a validation run
140 produces a non-uniform refinement pattern. (The error targets
141 for this case were determined by trial and error.) The tighter
142 error tolerances specified otherwise are appropriate to
143 properly resolve the solution, as shown in the animated gif files
144 at the beginning of this document.
145 \skipline Adjust error targets for adaptive refinement
146 \until end adjustment
147 
148  Next, we assign the boundary conditions. In the present problem all boundaries
149 are Dirichlet boundaries, therefore we loop over all nodes
150 on all boundaries and pin their values. If only a subset of the mesh
151 boundaries were of Dirichlet type, only the nodes on those boundaries
152 would have to be pinned. "Usually" the numbering of the mesh
153 boundaries is (or at least should be!) documented in the mesh
154 constructor but it can also be obtained from the function
155 \c Mesh::output_boundaries(...) whose use is illustrated here.
156
