3\mainpage Example problem: The azimuthally Fourier-decomposed 3D Helmholtz equation and the use of perfectly matched layers
4
5In this document we discuss the finite-element-based solution of the
6the Helmholtz equation in cylindrical polar coordinates, using
7a Fourier-decomposition of the solution in the azimuthal
8direction and with perfectly matched layers.
9
10Compared to the Fourier-decomposed Helmholtz equation discussed in
11<a href="../../../fourier_decomposed_helmholtz/sphere_scattering/html/index.html">
12another tutorial, </a> the formulation used here allows the
13imposition of the Sommerfeld radiation condition by means of
14so-called "perfectly matched layers" (PMLs) as an alternative
15to classical absorbing/approximate boundary conditions or DtN maps.
16
17We start by reviewing the relevant theory and then present the
18solution of a simple model problem - the outward propagation of waves
19from the surface of a unit sphere.

37\section theory Theory: The azimuthally Fourier-decomposed Helmholtz equation
38
39The Helmholtz equation governs time-harmonic solutions of problems
40governed by the linear wave equation
41\f[
42\nabla^2 U(x,y,z,t) = \frac{1}{c^2} \frac{\partial^2 U(x,y,z,t)}{\partial t^2},
43 \ \ \ \ \ \ \ \ \ \ \ \ (1)
44\f]
45where \f$ c \f$ is the wavespeed. Assuming that \f$ U(x,y,z,t) \f$
46is time-harmonic, with frequency \f$ \omega \f$, we write the real
47function \f$ U(x,y,z,t) \f$ as
48\f[
49U(x,y,z,t) = Re (u(x,y,z) \ e^{-i \omega t})
50 \ \ \ \ \ \ \ \ \ \ \ \ (2)
51\f]
52where \f$ u(x,y,z) \f$ is complex-valued. This transforms
53(1) into the Helmholtz equation
54\f[
55\nabla^2 u(x,y,z) + k^2 u(x,y,z) = 0 \ ,
56 \ \ \ \ \ \ \ \ \ \ \ \ (3)
57\f]
58where \f$ k = \omega/c \f$ is the wavenumber. Like other elliptic PDEs the Helmholtz equation
59admits Dirichlet, Neumann (flux) and Robin boundary conditions.
60
61If the equation is solved in an unbounded spatial domain (e.g. in scattering
62problems) the solution must also satisfy the so-called
63<a href="http://en.wikipedia.org/wiki/Sommerfeld_radiation_condition">
64Sommerfeld radiation condition</a>, which in 3D has the form
65\f[
66\lim_{r\to \infty} r \left(\frac{\partial u}{\partial r} - iku
67\right) =0.
68\f]
69Mathematically, this condition is required to ensure the uniqueness
70of the solution (and hence the well-posedness of the problem).
71In a physical context, such as a scattering problem, the condition
72ensures that scattering of an incoming wave only produces outgoing not
73incoming waves from infinity.
74
75 These equations can be solved using \c oomph-lib's cartesian
76Helmholtz elements, described in
77<a href="../../../helmholtz/scattering/html/index.html">another
78tutorial.</a> Here we consider an alternative approach in which we
79solve the equations in cylindrical polar coordinates \f$ (r,\varphi,z)
80\f$, related to the cartesian coordinates \f$ (x,y,z) \f$ via
81\f[
82x = r \cos(\varphi),
83\f]
84\f[
85y = r \sin(\varphi),
86\f]
87\f[
88z = z.
89\f]
90We then decompose the solution into its Fourier components by writing
91\f[
92u(r,\varphi,z) = \sum_{N=-\infty}^{\infty} u_N(r,z) \exp({\rm i}
93N \varphi).
94\f]
95Since the governing equations are linear we can compute each Fourier
96component \f$ u_N(r,z) \f$ individually by solving
97\f[
98\nabla^2 {u_{N}}(r,z) + \left(k^2-\frac{N^2}{r^2}\right) u_N(r,z) = 0
99 \ \ \ \ \ \ \ \ \ \ \ \ (4)
100\f]
101while specifying the Fourier wavenumber \f$ N \f$ as a parameter.

105\section discr Discretisation by finite elements
106The discretisation of the Fourier-decomposed Helmholtz equation itself
107only requires a trivial modification of its
108<a href="../../../helmholtz/scattering/html/index.html">cartesian
109counterpart</a>. Since most practical
110applications of the Helmholtz equation involve complex-valued
111solutions, we provide separate storage for the real and imaginary
112parts of the solution -- each \c Node therefore stores two unknowns
113values. By default,the real and imaginary parts are stored as values
1140 and 1, respectively;
115
116The application of Dirichlet and Neumann boundary conditions is
117straightforward and follows the pattern employed for the solution
118of the Poisson equation:
119- Dirichlet conditions are imposed by pinning the relevant nodal
120 values and setting them to the appropriate prescribed values.
121
122- Neumann (flux) boundary conditions are imposed via
123 \c FaceElements (here the \c PMLFourierDecomposedHelmholtzFluxElements).
124 <a href="../../../poisson/two_d_poisson_flux_bc/html/index.html">
125 As usual</a> we attach these to the faces of the "bulk" elements
126 that are subject to the Neumann boundary conditions.
127.
128
129The imposition of the Sommerfeld radiation condition for problems in
130infinite domains is slightly more complicated. In the next section we
131will discuss a method of representing the Sommerfeld radiation
132condition numerically by means of perfectly matched layers.

136\section pml Perfectly matched layers
137
138The idea behind perfectly matched layers is illustrated in the
139figure below. The actual physical/mathematical problem
140has to be solved in the infinite domain \f$ D \f$ (shown on the left),
141with the Sommerfeld radiation condition ensuring the suitable decay
142of the solution at large distances from the region of interest
143(the vicinity of the scatterer, say).
144
145If computations are performed in a finite computational domain, \f$
146D_c \f$ , (shown in the middle), spurious wave reflections are likely to
147be generated at the artificial boundary \f$
148\partial D_c \f$ of the computational domain.
149
150The idea behind PML methods is to surround the actual computational
151domain \f$ D_c \f$ with a layer of "absorbing" material whose
152properties are chosen such that the outgoing waves are absorbed
153within it, without creating any artificial reflected waves at the interface
154between the PML layer and the computational domain.
155
156Our implementation of the perfectly matched layers follows the
157development in
158<a href="http://www.sciencedirect.com/science/article/pii/S0021999106004487">
159A. Bermudez, L. Hervella-Nieto, A. Prieto, and
160R. Rodriguez "An optimal perfectly matched layer with unbounded
161absorbing function for time-harmonic acoustic scattering problems"
162Journal of Computational Physics <b>223</b> 469-488 (2007)</a>
163and we assume the boundaries of the computational domain to be aligned
164with the coordinate axes, as shown in the sketch below.
165
166The method requires a slight further generalisation of the equations,
167achieved by introducing the complex coordinate mapping
168\f[
169\frac{\partial}{\partial x_j} \to \frac{1}{s_j(x_j)} \frac{\partial}{\partial x_j}
170 \ \ \ \ \mbox{where $j ="r","z"$} \ \ \ \ \ \ \ \ (5)
171\f]
172within the perfectly matched layers. The choice of
173\f$ s_r(r) \f$ and \f$ s_z(z) \f$ depends on the orientation of the
174PML layer. Since we are restricting ourselves to axis-aligned
175mesh boundaries we distinguish three different cases
176
177- For layers that are aligned with the r axis (such as the top
178 and bottom PML layers) we set
179 \f[
180 s_z(z) =
181 1 +\frac{i}{k}\ \sigma_{z}(z)
182 \ \ \ \ \ \ \ \mbox{ \ \ \ with \ \ \ }
183 \sigma_{z}(z) = \frac{1}{|Z_{PML}-z|}, \ \ \ \ \
184 (6)
185 \f]
186 where \f$ Z_{PML} \f$ is the z-coordinate of the outer boundary
187 of the PML layer, and
188 \f[
189 s_r(r) = 1.
190 \f]
191
192- For the right layer that is aligned with the z axis we set
193 \f[
194 s_z(z) = 1,
195 \f]
196 and
197 \f[
198 s_r(r) =
199 1+\frac{i}{k} \ \sigma_{r}(r)
200 \ \ \ \ \ \ \ \mbox{ \ \ \ with \ \ \ }
201 \sigma_{r}(r) = \frac{1}{|R_{PML}-r|}, \ \ \ \ \
202 (7)
203 \f]
204 where \f$ R_{PML} \f$ is the r-coordinate of the outer boundary
205 of the PML layer.
206
207- In corner regions that are bounded by two axis-aligned PML layers
208 (with outer coordinates \f$ R_{PML} \f$ and
209 \f$ Z_{PML} \f$) we set
210 \f[
211 s_r(r) =
212 1 +\frac{i}{k}\ \sigma_{r}(r)
213 \ \ \ \ \ \ \ \mbox{ \ \ \ with \ \ \ }
214 \sigma_{r}(r) = \frac{1}{|R_{PML}-r|} \ \ \ \ \
215 (8)
216 \f]
217 and
218 \f[
219 s_z(z) =
220 1+\frac{i}{k} \ \sigma_{z}(z)
221 \ \ \ \ \ \ \ \mbox{ \ \ \ with \ \ \ }
222 \sigma_{z}(z) = \frac{1}{|Z_{PML}-z|}. \ \ \ \ \
223 (9)
224 \f]
225
226- Finally, in the actual computational domain (outside
227 the PML layers) we set
228 \f[
229 s_r(r) = s_z(z) = 1.
230 \f]
231.

236\section impl Implementation within oomph-lib
237
238The finite-element-discretised equations (modified by the PML terms
239discussed above) are implemented in the
240\c PMLFourierDecomposedHelmholtzEquations class.
241As usual, we provide
242fully functional elements by combining these with geometric
243finite elements (from the Q and T families -- corresponding (in 2D)
244to triangles and quad elements). By default, the PML modifications
245are disabled, i.e. \f$ s_{r}(r) \f$ and \f$ s_{z}(z) \f$ are both
246set to 1.
247
248 The generation of suitable 2D PML meshes along the axis-aligned
249boundaries of a given bulk mesh is facilitated by helper
250functions which automatically erect layers of (quadrilateral)
251PML elements. The layers are built from
252\c QPMLFourierDecomposedHelmholtzElement<NNODE_1D> elements and
253the parameter \c NNODE_1D is automatically chosen to match that of
254the elements in the bulk mesh. The bulk mesh can contain quads or triangles
255(as shown in the specific example presented below).

259\section osc_sph A specific example: Outward propagation of waves from the surface of an oscillating sphere
260
261We will now demonstrate the methodology for a specific example:
262the propagation of waves from the surface of a unit sphere.
263
264The specific domain used in this case can be seen in the figure below.
265We create an unstructured mesh of six-noded
266\c TPMLFourierDecomposedHelmholtzElements
267to create the finite computational domain surrounding a sphere.
268This is surrounded by three axis-aligned PML layers and two
269corner meshes (each made of nine-noded \c QPMLFourierDecomposedHelmholtzElements).
270
271\image html comp_domain.gif "The computational domain used in the example problem. "
272\image latex comp_domain.eps "The computational domain used in the example problem. " width=0.5\textwidth
273
274We construct an exact solution to the problem by applying
275Neumann/flux boundary condition on the inner
276spherical boundary such that the imposed flux \f$ \partial u/\partial
277n \f$ is consistent with the exact solution
278\f$ u(\rho,\varphi,\theta) \f$ in spherical polar coordinates \f$
279(\rho,\theta,\varphi) \f$, given by
280\f[
281u(\rho,\theta,\varphi)=
282\sum_{l=0}^{+\infty}\sum_{n=-l}^{l}
283\left(
284a_{ln} \ h_{l}^{(1)}(k\rho)+
285b_{ln} \ h_{l}^{(2)}(k\rho)
286\right)P_{l}^{n}
287(\cos\theta)\exp({\rm i} n \varphi).
288 \ \ \ \ \ \ \ (10)
289\f]
290where the \f$a_{ln}, b_{ln} \f$ are arbitrary coefficients and the functions
291\f[
292h_{l}^{(1)}(x)=j_{l}(x)+{\rm i} y_{l}(x) \mbox{ \ \ \ \ and \ \ \ \ }
293h_{l}^{(2)}(x)=j_{l}(x)-{\rm i} y_{l}(x)
294\f]
295are the spherical Hankel functions of first and second kind,
296respectively, expressed in terms the spherical Bessel functions
297\f[
298j_{l}(x)=\sqrt{\frac{\pi}{2x}}J_{l+1/2}(x) \mbox{ \ \ \ \ and \ \ \ \ }
299y_{l}(x)=\sqrt{\frac{\pi}{2x}}Y_{l+1/2}(x).
300\f]
301The functions
302\f[
303P_{l}^{m}(x)=(-1)^{m}(1-x^2)^{m/2}\frac{d^m}{dx^m}P_{l}(x)
304\f]
305are the associated Legendre functions, expressed in terms of the
306Legendre polynomials
307\f[
308P_{n}(x)=\frac{1}{2^{n} \, n!}\frac{d^n}{dx^n}[(x^2-1)^{n}].
309\f]
310This definition shows that \f$ P_{l}^{m}(x)=0 \f$ for \f$ m>l \f$
311which explains the limited range of summation indices in the second
312sum in (10).
313
314The relation between the cylindrical polar coordinates \f$
315(r,\varphi,z)\f$ and spherical polar coordinates \f$
316(\rho,\theta,\varphi) \f$
317is given by
318\f[
319\rho = \sqrt{r^2 + z^2},
320\f]
321\f[
322\theta = \arctan(r/z),
323\f]
324\f[
325\varphi = \varphi,
326\f]
327so \f$\varphi \in [0,2\pi] \f$ remains unchanged, and
328\f$\theta \in [0,\pi] \f$ sweeps from the north pole
329(\f$ \theta = 0 \f$), via the equator (\f$ \theta = \pi/2 \f$ )
330to the south pole (\f$ \theta = \pi \f$).

620\subsection unkns The enumeration of the unknowns
621
622As discussed in the introduction, most practically relevant
623solutions of the Helmholtz equation are complex valued. Since \c oomph-lib's
624solvers only deal with real (double precision) unknowns, the equations
625are separated into their real and imaginary parts.
626In the implementation of the Helmholtz elements, we store the real
627and imaginary parts of the solution as two separate values at each
628node. By default, the real and imaginary parts are accessible via
629\c Node::value(0) and \c Node::value(1). However,
630to facilitate the use of the elements
631in multi-physics problems we avoid accessing the unknowns
632directly in this manner but provide the virtual function
633\code
634std::complex<unsigned> PMLFourierDecomposedHelmholtzEquations::u_index_pml_fourier_decomposed_helmholtz()
635\endcode
636which returns a complex number made of the two unsigneds that indicate
637which nodal value represents the real and imaginary parts of the solution.
638This function may be overloaded in combined multi-physics elements
639in which a Helmholtz element is combined (by multiple inheritance)
640with another element, using the strategy described in
641<a href="../../../multi_physics/b_convection/html/index.html">
642the Boussinesq convection tutorial</a>.
643
644
645\subsection dmp_fcts PML damping functions
646
647The choice for the absorbing functions in our implementation of
648the PMLs is not unique. There are alternatives varying in both order and
649continuity properties. The current form is the result of several
650feasibility studies and comparisons found in both
651<a href="http://www.sciencedirect.com/science/article/pii/S0021999106004487">
652Bermudez et al.</a>
653These damping functions produce an
654acceptable result in most practical situations without further
655modifications. For very specific applications, alternatives may
656need to be used and can easily be implemented by constructing a PML Mapping
657class and passing a pointer to the elements.

687\subsubsection large_k Large wavenumbers
688For Helmholtz problems in general, ill-conditioning appears as the
689wavenumber becomes very large. By altering \f$ k^2 \f$,
690explore the limitations of both the mesh and the solver in terms
691of this parameter. Try adjusting the target element size in order to alleviate
692resolution-related effects. Assess the effectiveness of the perfectly matched
693layers in high wavenumber problems.

695\subsubsection adaptivity Spatial adaptivity
696The driver code discussed above already contains the straightforward
697modifications required to enable spatial adaptivity. Explore this
698(by recompiling the code with -DADAPTIVE). You will note that
699the driver code for this case is modified slightly -- the system
700is no longer driven by flux boundary conditions on the boundary
701of the sphere, but by a point source inside the domain. This was done
702to demonstrate the advantage of spatial adaptivity for such problems.
703The benefits of spatial adaptation in problems without any
704singularities tends to be limited since Helmholtz (and most other
705wave-type problems) require fairly uniform meshes throughout the domain.

707\subsubsection default Default values for problem parameters
708Following our usual convention, we provide default values
709for problem parameters where this is sensible. For instance,
710if the pointer to the PML damping class is not set, it will default to the best
711known PML mapping function proposed by Bermudez et al.
712Some parameters, such as the wavenumber squared \f$ k^2 \f$, do need to be set
713since there are no obvious defaults. If \c oomph-lib is compiled in \c PARANOID
714mode, an error is thrown if the relevant pointers haven't been set.
715Without paranoia, you get a segmentation fault...
716
717Confirm that this is the case by commenting out the relevant
718assignments.
