3\mainpage Demo problem: Bending of a 3D non-symmetric cantilever beam made of incompressible material
4
5In this tutorial we demonstrate the solution of a 3D solid mechanics
6problem: the large-amplitude bending deformation of a non-symmetric
7cantilever beam made of incompressible Mooney-Rivlin material.
8
9Here is an animation of the beam's deformation. In its undeformed
10configuration, the beam is straight and its cross-section is given
11by a quarter circle. The beam is loaded by an increasing gravitational
12body force, acting in the negative \f$ y\f$-direction, while its left
13end (at \f$ z=0 \f$) is held fixed. Because of its non-symmetric
14cross-section, the beam's downward bending deformation is
15accompanied by a sideways deflection.
16
17\image html cantilever.gif "Animation of the beam's bending deformation. "
18\image latex cantilever.eps "Animation of the beam's bending deformation. " width=0.75\textwidth
19
20Note how the automatic mesh adaptation refines the mesh in the region
21of strongest bending.

168\section exercises Comments and exercises
169
170\subsection incompr How to enforce incompressibility
171We stress that the imposition of incompressibility must be requested
172explicitly via the element's member function \c incompressible().
173Mathematically, incompressibility is enforced via a Lagrange
174multiplier which manifests itself physically as the pressure.
175Incompressibility can therefore only be enforced for
176elements that employ the pressure-displacement formulation
177of the principle of virtual displacements. This is why we
178the \c Problem constructor checked if the element is
179derived from the \c PVDEquationsWithPressure class before
180setting the element's \c incompressible flag to \c true.
181
182As usual, \c oomph-lib provides self-tests that assess if the
183enforcement incompressibility (or the lack thereof) is consistent:
184- The compiler will not allow the user to enforce incompressibility
185 on elements that are based on the displacement form of the
186 principle of virtual displacements.
187 \n\n
188- Certain constitutive laws, such as the Mooney-Rivlin law used in
189 the present example require an incompressible formulation. If \c oomph-lib
190 is compiled with the \c PARANOID flag, an error is thrown if
191 such a constitutive law is used by an element for which
192 incompressibility has not been requested.
193 \n\n
194 <center>
195 <b>Recall that the default setting is not to enforce
196 incompressibility!</b>
197 </center>
198 \n
199 If the library is
200 compiled without the \c PARANOID flag no warning will be issued
201 but the results will be "wrong" at least in the sense that
202 the material does not behave like an incompressible Mooney-Rivlin
203 solid. In fact, it is likely that the Newton solver will
204 diverge. Anyway, as we keep saying, without the \c PARANOID flag,
205 you're on your own!
206 \n\n
207- Some constitutive laws can be used for compressible and
208 incompressible behaviour. In this case it is important to
209 set the \c incompressible() flag to the correct value.
210 This issue is discussed in more detail in
211 <a href="../../compressed_square/html/index.html">another tutorial.</a>
212.
213You should experiment with different combinations of constitutive laws and
214element types to familiarise yourself with these.
