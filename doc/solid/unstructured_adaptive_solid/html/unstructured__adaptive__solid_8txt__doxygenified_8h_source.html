3\mainpage Demo problem: Solid Mechanics using unstructured meshes with adaptivity
4
5The purpose of this tutorial is to demonstrate the adaptive solution of
6solid mechanics problems using unstructured meshes generated by \c oomph-lib's
7<a href="../../../../doc/meshes/mesh_from_inline_triangle/html/index.html">
8inline unstructured mesh generation </a> procedures. The use of these
9methods for solid mechanics problems required no additional effort
10on the part of the user and the setup is essentially the same as that
11described for <a href="../../unstructured_solid/html/index.html">
12unstructured solid mechanics without mesh adaptation. </a>
13Lagrangian coordinates are projected
14between meshes in the same way as all other field variables, Eulerian
15coordinates and history values, see the description in
16 <a href="../../../meshes/mesh_from_inline_triangle/html/index.html">
17another tutorial. </a>
18
19The solid mechanics problem described here can be regarded as a
20sub-problem for the
21<a href="../../../../doc/interaction/unstructured_adaptive_fsi/html/index.html">
22unstructured adaptive fluid--structure interaction tutorial.</a> In
23addition, we can use the problem to assess the
24errors incurred when projecting the solution between different meshes.

29\section problem The problem
30An elastic bar is fixed at the base and
31loaded by a constant pressure on its left-hand
32side. The pressure load is increased and then decreased so that at the
33end of the simulation the bar should return to its undeformed
34position. The strain energy in the final configuration is a measure of
35the projection error because if there were no projection at all it
36would be exactly zero (or certainly zero to less than machine precision).
37

51The initial strain energy is \f$ O(10^{-28}) \f$, and the strain
52energy in the final configuration after the external pressure has been
53reset to zero, but the mesh has been adapted, is \f$ O(10^{-8})
54\f$. The strain energy at the maximum deflection is \f$ O(10^{-3}) \f$.

75\section main The driver code
76
77The driver code consists of essentially the same code repeated for three
78different formulations of solid mechanics: (i) (compressible)
79displacement only; (ii) (compressible) displacement-pressure; and
80(iii) incompressible displacement-pressure. We shall describe
81the code only for the first formulation.
82
83Initially, we specify an output directory
84and instantiate a constitutive equation. (Recall that the
85single-argument constructor to the \c GeneralisedHookean
86constitutive law implies that all stresses are non-dimensionalised
87on Young's modulus \f$ E \f$).
88
89\dontinclude unstructured_adaptive_solid.cc
90\skipline start_main
91\until ::Nu);
92
93We then open an output file for the strain energy, create
94the \c Problem object using a displacement formulation of
95the equations and output the initial configuration.
96
97\until ()++
98
99Finally, we perform the parameter study by slowly increasing and then
100reducing the pressure on the left-hand boundary. Note that one round
101of mesh adaptation is specified for every Newton solve.
102
103\until end_displacement_formulation

108\section class The Problem class
109The \c Problem class has the obvious member functions as well as a
110function to set whether the material is incompressible and a function
111to compute the strain energy of the elastic body. The class provides
112storage for the two sub-meshes: the bulk mesh of 2D solid elements
113and the mesh of 1D traction elements that will be attached to the
114left-hand boundary. In addition, storage is provided for the polygon that
115represents the initial outer boundary of the solid body and a boolean
116flag that is used to specify whether the material is incompressible or not.
117
118\dontinclude unstructured_adaptive_solid.cc
119\skipline start_problem
120\until };

125\section constructor The Problem constructor
126
127 We begin by building the closed, piecewise linear
128 boundary of the undeformed solid body \f$ x_{1} \in [0,1] \f$, \f$
129 x_{2} \in [0,5]. \f$ The boundaries are labelled anticlockwise with
130 boundary with the left-hand boundary being boundary 0, see the sketch
131 above. This process is a simplified version of the construction used in
132<a href="../../../../doc/meshes/mesh_from_inline_triangle/html/index.html">
133another tutorial. </a>
134

138
139We next construct an error estimator and
140specify the target errors and element sizes.
141
142\until min_element_size()
143
144 We output the boundaries, construct an empty traction
145 mesh and combine the bulk and traction meshes into a global mesh.
146
147\until build_global_mesh()
148
149 Finally we call \c actions_after_adapt(), which constructs the
150 traction elements, sets the boundary conditions and completes the build

170\section adapt_after Actions after adaptation
171
172 The function \c actions_after_adapt() first builds the traction
173 elements adjacent to the left-hand boundary (boundary 0) and rebuilds
174 the global mesh. The \c constant_pressure() load function is passed
175 to each of the traction elements.
176
177\until rebuild_global_mesh()
178
179 Next, the boundary conditions of a fixed base (boundary 3) are
180 set. These must be reset every time after an adaptation because
181 completely new nodes are generated.
182
183\until End of set boundary
184
185 Finally, the constitutive law and, if required, incompressibility
186 flag are passed to the bulk (solid) elements. Again, this must be
187 performed after every adaptation because a completely new mesh is
188 generated.
189
190\until end_actions_after
