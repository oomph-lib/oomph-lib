3 \mainpage Demo problem: Solid Mechanics using unstructured meshes with adaptivity
4 
5 The purpose of this tutorial is to demonstrate the adaptive solution of
6 solid mechanics problems using unstructured meshes generated by \c oomph-lib's
7 <a href="../../../../doc/meshes/mesh_from_inline_triangle/html/index.html">
8 inline unstructured mesh generation </a> procedures. The use of these
9 methods for solid mechanics problems required no additional effort
10 on the part of the user and the setup is essentially the same as that
11 described for <a href="../../unstructured_solid/html/index.html">
12 unstructured solid mechanics without mesh adaptation. </a>
13 Lagrangian coordinates are projected
14 between meshes in the same way as all other field variables, Eulerian
15 coordinates and history values, see the description in
16  <a href="../../../meshes/mesh_from_inline_triangle/html/index.html">
17 another tutorial. </a>
18 
19 The solid mechanics problem described here can be regarded as a
20 sub-problem for the
21 <a href="../../../../doc/interaction/unstructured_adaptive_fsi/html/index.html">
22 unstructured adaptive fluid--structure interaction tutorial.</a> In
23 addition, we can use the problem to assess the
24 errors incurred when projecting the solution between different meshes.

29 \section problem The problem
30 An elastic bar is fixed at the base and
31 loaded by a constant pressure on its left-hand
32 side. The pressure load is increased and then decreased so that at the
33 end of the simulation the bar should return to its undeformed
34 position. The strain energy in the final configuration is a measure of
35 the projection error because if there were no projection at all it
36 would be exactly zero (or certainly zero to less than machine precision).

51 The initial strain energy is \f$ O(10^{-28}) \f$, and the strain
52 energy in the final configuration after the external pressure has been
53 reset to zero, but the mesh has been adapted, is \f$ O(10^{-8})
54 \f$. The strain energy at the maximum deflection is \f$ O(10^{-3}) \f$.

75 \section main The driver code
76 
77 The driver code consists of essentially the same code repeated for three
78 different formulations of solid mechanics: (i) (compressible)
79 displacement only; (ii) (compressible) displacement-pressure; and
80 (iii) incompressible displacement-pressure. We shall describe
81 the code only for the first formulation.
82 
83 Initially, we specify an output directory
84 and instantiate a constitutive equation. (Recall that the
85 single-argument constructor to the \c GeneralisedHookean
86 constitutive law implies that all stresses are non-dimensionalised
87 on Young's modulus \f$ E \f$).
88 
89 \dontinclude unstructured_adaptive_solid.cc
90 \skipline start_main
91 \until ::Nu);
92 
93 We then open an output file for the strain energy, create
94 the \c Problem object using a displacement formulation of
95 the equations and output the initial configuration.
96 
97 \until ()++
98 
99 Finally, we perform the parameter study by slowly increasing and then
100 reducing the pressure on the left-hand boundary. Note that one round
101 of mesh adaptation is specified for every Newton solve.
102 
103 \until end_displacement_formulation

108 \section class The Problem class
109 The \c Problem class has the obvious member functions as well as a
110 function to set whether the material is incompressible and a function
111 to compute the strain energy of the elastic body. The class provides
112 storage for the two sub-meshes: the bulk mesh of 2D solid elements
113 and the mesh of 1D traction elements that will be attached to the
114 left-hand boundary. In addition, storage is provided for the polygon that
115 represents the initial outer boundary of the solid body and a boolean
116 flag that is used to specify whether the material is incompressible or not.

170 \section adapt_after Actions after adaptation
171 
172  The function \c actions_after_adapt() first builds the traction
173  elements adjacent to the left-hand boundary (boundary 0) and rebuilds
174  the global mesh. The \c constant_pressure() load function is passed
175  to each of the traction elements.
176 
177 \until rebuild_global_mesh()
178 
179  Next, the boundary conditions of a fixed base (boundary 3) are
180  set. These must be reset every time after an adaptation because
181  completely new nodes are generated.
182 
183 \until End of set boundary
184 
185  Finally, the constitutive law and, if required, incompressibility
186  flag are passed to the bulk (solid) elements. Again, this must be
187  performed after every adaptation because a completely new mesh is
188  generated.
189 
190 \until end_actions_after

208 \section doc Post-processing
