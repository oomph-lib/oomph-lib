229<TABLE BORDER="1">
230<TR>
231<TD WIDTH="50%" VALIGN="TOP">
232- The setup phase is now complete and we can determine the
233 (current) FE representation of all quantities in element \f$ e\f$:\n\n
234 - The global coordinates \f$ x_i \f$ are given by
235 \f[
236 x_i = \sum_{j=1}^{n} X_{i{\cal J}(j,e)} \ \psi_j(s)
237 \f]
238 - Their derivatives with respect to the local coordinates are
239 \f[
240 \widehat{\cal J}_{ik} = \frac{\partial x_i}{\partial s_j} =
241 \sum_{k=1}^n X_{i{\cal J}(k,e)}
242 \ \frac{\partial \psi_k(s)}{\partial s_j}
243 \f]
244 - The Jacobian of the mapping between the local and global coordinates
245 is given by
246 \f[
247 \widehat{\cal J} = \det \widehat{\cal J}_{ik}
248 \f]
249 - The function \f$ u\f$ is obtained by interpolation between the
250 the nodal values:
251 \f[
252 u = \sum_{j=1}^{n} U_{{\cal J}(j,e)} \ \psi_j(s_1,s_2)
253 \f]
254 - Its derivatives with respect to the global coordinates are
255 \f[ u = \frac{\partial u}{\partial x_i} =
256 {\widehat J}^{-1}_{ij}
257 \left(\sum_{k=1}^n U_{{\cal J}(k,e)}
258 \ \frac{\partial \psi_k(s_1,s_2)}{\partial s_j}
259 \right)
260 \f]
261 - Finally, the derivatives of the local shape functions,
262 \f$ \psi_i \f$, with respect to the global coordinates
263 \f$ x_j \f$ are given by
264 \f[ \frac{\partial \psi_i}{\partial x_j} = {\widehat J}^{-1}_{ik} \
265 \frac{\partial \psi_i(s_1,s_2)}{\partial s_k}
266 \mbox{\ \ \ for $j=1,..,n$}
267 \f]
268</TD>
269<TD WIDTH="50%" VALIGN="TOP">

272 member functions. Here are a few representative member functions for our

274 - The FE representation of the global (Eulerian) coordinate \f$x_i\f$ at
275 the local coordinate
276 \f$ (s_1,s_2)\f$:
277 \code
278 Vector<double> s(2);
279 double x=QPoissonElement<3,2>::interpolated_x(s,0);
280 double y=QPoissonElement<3,2>::interpolated_x(s,1);
281 \endcode
282 [\b Note: In the C++ implementation, all indices start at 0, so the
283 global coordinates are \f$(x_0,x_1)\f$.]
284 - The Jacobian of the mapping between the global (Eulerian) and
285 local coordinates, \f$x_i(s_j)\f$, at the local coordinate
286 \f$ (s_0,s_`)\f$:
287 \code
288 Vector<double> s(2);
289 double jacobian=QPoissonElement<3,2>::J_eulerian(s);
290 \endcode
291 - The FE representation of the solution at the local coordinate
292 \f$ (s_1,s_2)\f$:
293 \code
294 Vector<double> s(2);
295 double u=QPoissonElement<3,2>::interpolated_u(s);
296 \endcode

298 the local coordinates at the local coordinate
299 \f$ (s_0,s_1)\f$:
300 \code
301 Shape psi(9); // We have 9 shape functions
302 DShape dpsi(9,2); // The 9 shape functions
303 // have 2 derivatives each
304 // (w.r.t to s_0 and s_1)
305 Vector<double> s(2);
306 QPoissonElement<3,2>::dshape_local(s,psi,dpsi);
307 \endcode
308 - The shape functions and their first derivatives with respect to
309 the global (Eulerian) coordinates at the local coordinate
310 \f$ (s_0,s_1)\f$:
311 \code
312 Shape psi(9); // We have 9 shape functions
313 DShape dpsidx(9,2); // The 9 shape functions
314 // have 2 derivatives each
315 // (w.r.t to x_0 and x_1)

317 QPoissonElement<3,2>::dshape_eulerian(s,psi,dpsidx);
318 \endcode

368 <B> Phase 2a: Setup the linear system </B>
369
370 <TABLE BORDER="1" >
371 <TR>
372 <TD WIDTH="50%" VALIGN="TOP">
373
374- Initialise the global residual vector, \f$ r_j=0 \f$ for \f$j=1,...,M \f$
375 and the global Jacobian matrix \f$ J_{jk}=0 \f$ for \f$j,k=1,...,M \f$
376- Loop over the elements \f$ e=1,...,N_E\f$
377 \n \n
378 <TABLE BORDER="0">
379 <TR>
380 <TD bgcolor="cornsilk">
381 <CENTER> <B> Compute the element's residual vector and
382 Jacobian matrix </B> </CENTER>
383 \n
384 - Determine the number of degrees of freedom in this element,
385 \f$ n_{dof}={\cal N}_{dof}(e) \f$.
386 - Initialise the element residual vector,
387 \f$ r_{j}^{(e)}=0 \f$ for \f$j=1,...,n_{dof} \f$
388 and the element Jacobian matrix \f$ J_{jk}^{(e)}=0 \f$
389 for \f$j,k=1,...,n_{dof} \f$
390 - Loop over the Gauss points \f$ i_{int}=1,...,N_{int}\f$
391 - Determine the local coordinate of the integration point
392 \f$ s_j = S_{j\, i_{int}} \mbox{\ \ \ for $j=1,2$.} \f$
393 - Compute
394 - the global coordinates \f$ x_i = x_i(s_j) \f$
395 - the source function \f$ f = f(x_i) =f(x_i(s_j)) \f$
396 - the derivative \f$ \partial u/\partial x_i \f$ and
397 - the shape functions \f$ \psi_j \f$ and their derivatives
398 \f$ \partial \psi_j/\partial x_k \f$
399 - the Jacobian of the mapping between local and global
400 coordinates, \f$ \widehat{\cal J} =
401 \det \partial x_i/\partial s_j\f$
402 .
403 - Loop over the local nodes \f$ j_{local}=1,...,n \f$
404 - Determine the global node number
405 \f$ j_{global}={\cal J}(j_{local},e)\f$
406 - Determine the global equation number
407 \f$ {\cal E}( j_{global}) \f$
408 - If \f$ {\cal E}( j_{global}) \ne -1 \f$
409 - Determine the local equation number from the element's
410 lookup scheme \f$ i_{dof}= {\cal L}(j_{local},e)\f$.
411 - Add the contribution to the element's residual vector
412 \f[ r_{i_{dof}}^{(e)} = r_{i_{dof}}^{(e)} +
413 \left( \sum_{l=1}^2 \frac{\partial
414 u}{\partial x_l} \
415 \frac{\partial \psi_{j_{local}}}{\partial x_l} +
416 f \ \psi_{j_{local}} \right)
417 \widehat{\cal J} \ W_{i_{int}} \f]
418 - Loop over the local nodes \f$ k_{local}=1,...,n \f$
419 - Determine the global node number
420 \f$ k_{global}={\cal J}(k_{local},e)\f$
421 - Determine the global equation number
422 \f$ {\cal E}( k_{global}) \f$
423 - If \f$ {\cal E}( k_{global}) \ne -1 \f$
424 - Determine the local equation number from the element's
425 lookup scheme \f$ j_{dof}= {\cal L}(j_{local},e)\f$.
426 - Add the contribution to the element's Jacobian matrix
427 \f[ J_{i_{dof}j_{dof}}^{(e)} = J_{i_{dof}j_{dof}}^{(e)} +
428 \left( \sum_{l=1}^2
429 \frac{\partial \psi_{k_{local}}}{\partial x_l} \
430 \frac{\partial \psi_{j_{local}}}{\partial x_l}
431 \right)
432 \widehat{\cal J} \ W_{i_{int}} \f]
433 </TD>
434 </TR>
435 </TABLE>
436 \n \n \n \n \n
437 <CENTER> <B> Add the element's contribution to the global residual
438 vector and Jacobian matrix </B> </CENTER>
439 \n
440 - Loop over the local degrees of freedom \f$ i_{dof}=1,...,n_{dof} \f$
441 - Add the element's contribution to the global residual vector
442 \f[ r_{\widehat{\cal E}(i_{dof},e)} =
443 r_{\widehat{\cal E}(i_{dof},e)} + r_{i_{dof}}^{(e)}
444 \f]
445 - Loop over the local degrees of freedom \f$ j_{dof}=1,...,n_{dof}\f$
446 - Add the element's contribution to the global Jacobian matrix
447 \f[ J_{\widehat{\cal E}(i_{dof},e)\widehat{\cal E}(j_{dof},e)}=
448 J_{\widehat{\cal E}(i_{dof},e)\widehat{\cal E}(j_{dof},e)}+
449 J_{i_{dof} j_{dof}}^{(e)} \f]
450 .
451 .
452 .
453 .
454 </TD>
455 <TD WIDTH="50%" VALIGN="TOP">
456 - The assembly process is completely generic:
457 - Find out how many degrees of freedom we have from
458 \code
459 unsigned ndof_global=Problem::ndofs();
460 \endcode
461 This allows us to create and initialise the global Jacobian matrix
462 and the global residual vector.
463 \code
464 Vector<double> residual(ndof_global);
465 DenseDoubleMatrix jacobian(ndof_global,ndof_global);
466 \endcode
467 [\b Note: This is just an illustration! Do \b not store
468 the global Jacobian matrix as a \c DenseMatrix --
469 the \c oomph-lib implementation of a dense matrix!].
470 - Find out how many elements there are (in the Mesh)
471 from
472 \code
473 Problem::mesh_pt()->nelement();
474 \endcode
475 - Now we can loop over all Elements. Elements are accessed
476 via the Mesh's access function \c Mesh::element_pt(), so
477 we obtain a pointer to the \f$ e \f$-th element in the
478 Problem's Mesh, by calling
479 \code
480 FiniteElement* element_pt=Problem::mesh_pt()->element_pt(e);
481 \endcode
482 - How many degrees of freedom does this Element have?
483 \code
484 unsigned ndof=element_pt->ndof();
485 \endcode
486 - We can now allocate the Element's Jacobian matrix,
487 \c el_jacobian, and its residual vector, \c el_residual.
488 \code
489 Vector<double> el_residual(ndof);
490 DenseDoubleMatrix el_jacobian(ndof);
491 \endcode
492 - Get their entries from the Element by calling
493 \code
494 element_pt->get_jacobian(el_jacobian,el_residual);
495 \endcode
496 Note that the implementation of this function
497 depends on the (mathematical) problem to be solved but its interface
498 is completely generic. The function is therefore defined as
499 a virtual function in the abstract base class \c GeneralisedElement,
500 and needs to be implemented in a derived class.
501 For instance,
502 <TABLE BORDER="0">
503 <TR>
504 <TD bgcolor="cornsilk">
505 \n
506 \c QPoissonElement<NNODE_1D,DIM>::get_jacobian(el_jacobian,el_residual);
507 \n \n
508 </TD>
509 </TR>
510 </TABLE>
511 implements the computation of the element residual and
512 Jacobian matrix
513 for the \c DIM -dimensional Poisson equation, discretised with
514 isoparametric elements from the \c QElement family, so that
515 - \c DIM=1 corresponds to 1D line elements
516 - \c DIM=2 corresponds to 2D quadrilateral elements
517 - \c DIM=3 corresponds to 3D brick elements.
518 .
519 The second template parameter, \c NNODE_1D indicates the
520 number of nodes along the element edges so that
521 - \c NNODE_1D=2 corresponds to elements with linear
522 (or {bi/tri}linear) shape functions
523 - \c NNODE_1D=3 corresponds to elements with quadratic
524 (or {bi/tri}quadratic) shape functions
525 - etc.
526 .
527 \n \n
528
529 - Copying of the Element's Jacobian matrix and residual vector
530 into their global counterparts is again completely generic
531 and involves calls to the Element's \c eqn_number() function
532 which returns the global equation number corresponding
533 to a local equation number. For instance, the assembly
534 loop for the global residual vector \c residual can be
535 implemented as follows
536 \code
537 // Loop over the number of dofs in the element
538 for (unsigned i_local=0;i_local<ndof;i_local++)
539 {
540 // Get the corresponding global equation number
541 unsigned i_global=element_pt->eqn_number(i_local);
542
543 // Add the entry to the global residual vector
544 residual[i_global]+=el_residual[i_local];
545 }
546 \endcode

625\subsection non_dirichlet_bc Non-Dirichlet boundary conditions
626The examples considered so far have all been problems with
627Dirichlet boundary conditions, i.e. problems in which the
628value of the unknown function was prescribed on the entire
629domain boundary. To illustrate how non-Dirichlet
630boundary conditions are incorporated into a finite element discretisation,
631we consider the following variant of problem P2:
632
633
634<CENTER>
635<TABLE>
636<TR>
637<TD>
638<CENTER>
639<B> Problem P4 </B>
640</CENTER>
641Solve
642\f[ {\cal R}(x_i; u(x_i)) = \sum_{k=1}^2
643\frac{\partial^2 u(x_i)}{\partial x_k^2} -
644f(x_i) =0 \mbox{ \ \ \ \ for $x_i\in D$} \f]
645subject to
646\f[ u|_{\partial D_{Dirichlet}} = g(\zeta)
647\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ (1)\f]
648and
649\f[ \left. \frac{\partial u}{\partial n}\right|_{\partial
650D_{Neuman}} = h(\zeta),
651\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ (2) \f]
652where \f$ f(x_i), g(\zeta)\f$ and \f$ h(\zeta)\f$ are given.
653The domain boundary \f$ \partial D\f$ is parametrised by
654the arclength \f$ \zeta \f$, so that the position vector
655to the boundary is given by
656\f[
657{\bf r}^{\partial D}(\zeta) = \left( x_1^{\partial D}(\zeta),
658 x_2^{\partial D}(\zeta) \right)^T
659\f]
660The boundary consists of two parts so that
661\f$\partial D = \partial D_{Dirichlet} \cup \partial D_{Neuman} \f$
662and \f$ \partial D_{Dirichlet} \cap \partial D_{Neuman} = 0. \f$ \n
663\f$ \partial/\partial n \f$ denotes the derivative in
664direction of the outer unit normal to the domain.
665\image html dirichlet_and_neumann_bc.gif "Sketch of the domain and its "
666\image latex dirichlet_and_neumann_bc.eps "Sketch of the domain and its " width=0.55\textwidth
667
668</TD>
669</TR>
670</TABLE>
671</CENTER>
672
673In the context of the physical problem of steady heat conduction,
674where \f$ u(x_i) \f$ represents the temperature,
675the Dirichlet boundary condition \f$ u|_{\partial D_{Dirichlet}} =
676g\f$ prescribes the temperature along a certain part of the
677boundary, while the Neumann condition
678\f$\partial u/\partial n|_{\partial D_{Neuman}} = h\f$ prescribes
679the heat flux into the body along the remainder of its
680surface.
681
682As before, we define a weak solution of this problem
683as a(ny) function \f$ u_w(x_i) \f$ that satisfies the
684Dirichlet boundary conditions (1),
685\f[ u_w|_{\partial D_{Dirichlet}} = g(\zeta),\f]
686and for which the weighted residual
687\f[ r = \int_{D} {\cal R}(x_i; u_w(x_i)) \ \phi^{(test)}(x_i)
688\ \mbox{d}x_1 \mbox{d}x_2 \ \ \ \ \ \ \ \ (3) \f]
689vanishes for \e any "test function"
690\f$ \phi^{(test)}(x_i) \f$ that satisfies homogeneous
691boundary conditions on the \f$\partial D_{Dirichlet}\f$,
692\f[ \phi^{(test)}|_{\partial D_{Dirichlet}} = 0. \f]
693
694The definition is consistent with
695P2<SUB>weak</SUB>: the two are identical if
696\f$ \partial D = \partial D_{Dirichlet} \f$,
697or, equivalently, if \f$ \partial D_{Neumann}=0 \f$.
698The role of the test function in the weak
699formulation of our
700problem is to test whether the candidate solution violates any of the
701conditions imposed on it. There are two types of test:
702-# Does the candidate solution satisfy the PDE in the interior of
703 the domain?
704-# Does the candidate solution satisfy the boundary conditions?
705.
706We assume the Dirichlet conditions to be satisfied a priori, so
707it is not necessary to perform any tests on
708\f$ \partial D_{Dirichlet} \f$. The easiest way to exclude
709\f$ \partial D_{Dirichlet} \f$ from the "test" is to set
710\f$ \phi^{(test)}|_{\partial D_{Dirichlet}} = 0 \f$ as we have
711done. If \f$ \partial D_{Neumann} \ne 0 \f$, we need to
712test whether our candidate solution satisfies
713the Neumann boundary condition (2).
714Certain types of Neumann boundary conditions can be
715naturally incorporated into the weighted residual.
716
717Performing the integration by parts in our present problem
718converts (3) into
719\f[ \int_D \left[ \sum_{k=1}^2 \frac{\partial u(x_i)}{\partial x_k} \
720 \frac{\partial \phi^{(test)}(x_i)}{\partial x_k} +
721 f(x_i) \ \phi^{(test)}(x_i) \right] \mbox{d}x_1 \mbox{d}x_2 = \f]
722\f[
723= \int_{\partial D_{Dirichlet}} \phi^{(test)} \
724 \frac{\partial u}{\partial n} \ \mbox{d}\zeta \ + \
725 \int_{\partial D_{Neumann}}
726 \ \phi^{(test)} \ \frac{\partial u}{\partial n} \ \mbox{d}\zeta, \ \ \ \ \
727(4) \f]
728where \f$\zeta\f$ is the arclength along the domain boundary
729\f$ \partial D\f$. We have split the integral along \f$ \partial D\f$
730into two separate integrals over
731\f$ \partial D_{Dirichlet}\f$ and \f$ \partial D_{Neumann}\f$.
732Since \f$ \phi^{(test)}|_{\partial D_{Dirichlet}} = 0, \f$
733the integral over \f$\partial D_{Dirichlet}\f$ vanishes
734automatically. The second integral has to be retained because
735\f$ \phi^{(test)} \f$ can take any value on
736\f$\partial D_{Neumann}\f$. However, \f$ \partial u/\partial n \f$
737in its integrand is specified by the Neumann boundary
738condition (2). We can, therefore, incorporate
739(2) into the weighted residual by writing
740\f[ \int_D \left[ \sum_{k=1}^2 \frac{\partial u(x_i)}{\partial x_k} \
741 \frac{\partial \phi^{(test)}(x_i)}{\partial x_k} +
742 f(x_i) \ \phi^{(test)}(x_i) \right] \mbox{d}x_1 \mbox{d}x_2 =
743 \int_{\partial D_{Neumann}}
744 \phi^{(test)}(\zeta) \ h(\zeta) \ d\zeta. \ \ \ \ \
745(5) \f]
746We now introduce the usual finite-element expansion for \f$ u(x_i)\f$
747in terms of the \f$ N \f$ nodal values and the global
748shape functions \f$ \psi_j(x_i) \f$,
749\f[ u(x_i) = \sum_{j=1}^{N} \ U_j \ \psi_j(x_i), \f]
750where the Dirichlet
751boundary conditions determine the nodal values of all nodes
752that are located on \f$\partial
753D_{Dirichlet}\f$. The nodal values of
754the nodes in the interior of the domain and on
755\f$\partial D_{Neumann}\f$ are unknown and must
756be determined from the solution of the discrete equations
757\f[ f_{{\cal E}(l)} = \int_D \left[ \sum_{j=1}^N U_j \sum_{k=1}^2
758 \frac{\partial \psi_j(x_i)}{\partial x_k} \
759 \frac{\partial \psi_l(x_i)}{\partial x_k} +
760 f(x_i) \ \psi_l(x_i) \right] \mbox{d}x_1 \mbox{d}x_2\ \f]
761 \f[ - \
762 \int_{\partial D_{Neumann}}
763 \psi_l(\zeta) \ h(\zeta) \ \mbox{d}\zeta =0 \\ \mbox{\ \ \ for all $l$ for
764 which ${\cal E}(l)>0$,}
765\ \ \ \ \ \
766(6) \f]
767where we have used the global equation number \f${\cal E}(l)\f$
768to identify nodes whose values are not pinned.
769Let us now assume that we have discretised the domain by using
770isoparametric elements, so that
771within element \f$ e \f$ the solution is represented as
772\f[ u(s_1,s_2) = \sum_{j_{local}=1}^n U_{{\cal J}(j_{local},e)}
773\ \psi_{j_{local}}(s_1,s_2) \f]
774and
775\f[ x_i(s_1,s_2) = \sum_{j_{local}=1}^n X_{i{\cal J}(j_{local},e)}
776\ \psi_{j_{local}}(s_1,s_2), \ \ \ \ \ \ \
777(7) \f]
778where \f$ n \f$ is the number of nodes in the element.
779We employ the element-by-element assembly procedure described
780in Algorithm 1 to evaluate the domain integral
781in (6).
782During this procedure, we visit each element in the mesh, loop over
783the element's \f$ n \f$ nodes
784\f$ l_{local}=1,...,n \f$, check whether the node's global equation number
785\f$ i_{global} = {{\cal E}({\cal J}(l_{local},e))}\f$ is positive, and, if so,
786add the "bulk" contribution
787\f[
788\int_{-1}^{1} \int_{-1}^{1}
789\left[ \sum_{j_{local}=1}^n U_{{\cal J}(j_{local},e)} \sum_{k=1}^2
790\frac{\partial \psi_{j_{local}}}{\partial x_k} \
791\frac{\partial \psi_{l_{local}}}{\partial x_k} +
792f \ \psi_{l_{local}} \right] \widehat{\cal J} \ \mbox{d}s_1 \mbox{d}s_2
793\f]
794to the \f$ i_{global}\f$-th entry of the global residual vector.
795
796It remains to assemble the contributions from the line integral around
797the boundary of the domain, which can also be done element-by-element.
798To illustrate the
799procedure, we consider the example shown below: A (topologically
800rectangular) domain has been
801decomposed into 12 four-node elements. Neumann boundary
802conditions are to be applied along the domain's "southern " and
803"eastern" boundaries. The "northern" and "western" boundaries
804are subject to Dirichlet boundary conditions.
805\image html dirichlet_and_neumann_mesh.gif "Sketch of a discretised domain with Dirichlet and Neumann boundary "
806\image latex dirichlet_and_neumann_mesh.eps "Sketch of a discretised domain with Dirichlet and Neumann boundary " width=0.75\textwidth
807A discrete representation of the domain's southern boundary
808is given by the southern edges of elements 1,2 and 3.
809Along these edges the local coordinate \f$ s_1\f$ varies
810from -1 to +1 while \f$ s_2 \equiv -1\f$. We obtain
811a piecewise linear approximation for the continuous
812domain boundary \f$ {\bf r}^{\partial D}\f$ by substituting
813\f$ s_1 = S \in [-1,1]\f$ and \f$ s_2 = -1 \f$ into the
814mapping (7) between
815the local and global coordinates,
816\f[
817x_i^{\partial D} \approx x_i(s_1=S,s_2=-1)
818\f]
819The integral
820of any function \f$ {\cal F}\big(x_1,x_2\big) \f$
821along the elements' southern edges can be performed
822in terms of element's local coordinates by writing
823\f[
824\int_{\mbox{Southern edge of element $e$}} {\cal F}\big(x_1(\zeta) ,x_2(\zeta)\big) \ d\zeta =
825\int_{-1}^{1} \left. {\cal F}\big(x_1(s_1,s_2),x_2(s_1,s_2)\big) \right|_{
826s_2=-1}
827\ \widehat{\cal J}^{(1D)}(s_1) \ ds_1,
828\f]
829where
830\f[ \widehat{\cal J}^{(1D)}(s_1) = \left. \sqrt{
831\left( \frac{\partial x_1(s_1,s_2)}{\partial s_1} \right)^2 +
832\left( \frac{\partial x_2(s_1,s_2)}{\partial s_1} \right)^2 }
833\right|_{s_2=-1}
834\f]
835is the Jacobian of the mapping between the local coordinate
836that parametrises the edge of the element, \f$s_1\f$, and the arclength,
837\f$ \zeta\f$.
838Similarly, the domain's eastern boundary is represented
839by the eastern edges of elements 3, 6, 9 and 12 whose position
840is given by \f$ x_i(s_1=1,s_2=S) \f$ where \f$ S \in [-1,1]\f$,
841and
842\f[
843\int_{\mbox{Eastern edge of element $e$}} {\cal
844F}\big(x_1(\zeta) ,x_2(\zeta)\big) \ d\zeta =
845\int_{-1}^{1} \left. {\cal F}\big(x_1(s_1,s_2),x_2(s_1,s_2)\big) \right|_{
846s_1=1}
847\ \widehat{\cal J}^{(1D)}(s_2) \ ds_2,
848\f]
849where
850\f[ \widehat{\cal J}^{(1D)}(s_2) = \left. \sqrt{
851\left( \frac{\partial x_1(s_1,s_2)}{\partial s_2} \right)^2 +
852\left( \frac{\partial x_2(s_1,s_2)}{\partial s_2} \right)^2 }
853\right|_{s_1=1}
854\f]
855In general, the line integrals are evaluated
856numerically, e.g. using the 1D Gauss rules discussed in
857section \ref local_coords.
858
859Inspection of equation
860(6) shows that the line
861integral only makes a contribution to the discrete residuals whose
862nodes are located on the Neumann boundaries. [Recall that the
863global shape function \f$ \psi_l \f$
864is equal to one at global node \f$ l \f$ and vanishes at all
865other nodes. Furthermore, \f$ \psi_l \f$ is nonzero
866only in the patch of elements adjacent to node \f$ l \f$ and
867vanishes on the boundary of that patch.]
868The line integral in (6)
869can, therefore, be evaluated by augmenting the assembly
870of the elemental' residual vector as follows:
871
872- Compute the "standard" element residual vector that arises
873 from the area integral, as discussed in algorithm 1.
874- If the element's southern edge is located on a Neumann boundary:
875 - Set \f$s_2=-1\f$.
876 - Loop over the 1D Gauss points \f$ i_{int}=1,...,N_{int} \f$.
877 - Determine the local coordinate of the integration point
878 \f$ s_1 = S_{i_{int}} \f$ and the associated
879 weight \f$W_{i_{int}}.\f$
880 - Compute
881 - the global coordinates \f$ x_i = x_i(s_1,s_2) \f$
882 - the prescribed-flux function \f$ h=h(x_1,x_2)\f$
883 - the shape functions \f$ \psi_j(s_1,s_2) \f$
884 - the Jacobian of the 1D mapping
885 \f[
886 \widehat{\cal J}^{(1D)} = \left. \sqrt{
887 \left( \frac{\partial x_1(s_1,s_2)}{\partial s_1} \right)^2 +
888 \left( \frac{\partial x_2(s_1,s_2)}{\partial s_1} \right)^2 }
889 \right|_{s_2=-1}
890 \f]
891 .
892 - Loop over those local nodes that are located on the
893 element's southern edge (e.g. \f$ j_{local}=1,2\f$ for a four-node
894 quadrilateral element).
895 - Determine the global node number
896 \f$ j_{global} = {\cal J}(j_{local},e)\f$.
897 - Determine the global equation number \f${\cal E}(j_{global})\f$.
898 - If \f${\cal E}(j_{global})\ne -1\f$:
899 - Determine the local equation number
900 \f$i_{dof}={\cal L}(j_{local},e)\f$.
901 - Add the contribution to the element's residual vector
902 \f[
903 r^{(e)}_{i_{dof}} = r^{(e)}_{i_{dof}} +
904 h \ \psi_{j_{local}} \ \widehat{\cal J}^{(1D)} \ W_{i_{int}}
905 \f]
906 .
907 .
908 .
909 .
910- Perform equivalent operations for the eastern, northern
911 and western edges, if required.
912
913Note that it is not necessary to change the setup of the Jacobian matrix
914because the boundary terms are independent of the unknown variables.
915

918elements such as 3D Poisson elements for which
919Neumann boundary conditions are applied on the elements'

924\subsubsection neumann_implementation Implementation
925The procedure outlined in the previous section could easily
926be implemented within the general framework of \c oomph-lib.
927One possible implementation would be the following:
928- Add a vector of bools, \c Vector<bool> \c Neumann_bc, say,
929 to the private data of \c PoissonEquations and use
930 the value of \c Neumann_bc[i] to indicate whether
931 the element's \c i-th face is subject to
932 a Neumann boundary condition (assuming that \c
933 Neumann_bc[i]=false by default.)
934- Provide storage for function pointers to the (global) functions
935 that specify the prescribed fluxes on the element's
936 faces.
937- When assembling the element's residual vector
938 in \c PoissonEquations::get_residuals(), execute the
939 assembly loop for the boundary integrals only for those edges
940 for which \c Neumann_bc[i]=true.
941- To solve a problem with Neumann boundary conditions we would
942 discretise the domain as before, loop over
943 all elements that have nodes on \f$ \partial D_{Neumann}\f$,
944 setting the appropriate
945 \c Neumann_bc flag to \c true and specifying
946 a function pointer to the prescribed-flux function.
947.
948The implementation is straightforward and actually
949presents an instructive exercise. [Suggestion: Rather than
950incorporating the additional functionality directly
951into the existing Poisson elements, use inheritance to create a new
952class, \c PoissonEquationsWithFluxBC, say, which contains
953the additional data and provides the necessary access functions.
954The derived class must
955overload \c get_residuals(...) with a function that initially calls \c
956PoissonEquations:get_residuals(...) and then adds the
957contribution from the line integrals].
958
959Unfortunately, the implementation suggested above has
960several serious disadvantages
961-# The test of \c Neumann_bc[i] must be performed
962 in every element, even though only a relatively small number of
963 elements are likely to be located on the domain boundaries.
964-# The code that implements the computation of the integrals
965 along the four possible element edges is very similar
966 (the main difference being which local coordinate is held
967 constant along the appropriate element edge, and which
968 local nodes make a contribution to the element residual vector)
969 but this similarly is not exploited.
970-# Finally, and most significantly, many steps in the above
971 procedure, such as the identification
972 of the nodes that are located on specific element edges, are
973 completely generic and will also occur in the application of
974 Neumann-type boundary conditions in other equations (see
975 section ***). Incorporating the procedure directly into
976 the Poisson elements makes it impossible to "reuse" the
977 relevant algorithms elsewhere.
978
979Our implementation employs a slightly different
980approach: rather than incorporating the assembly of the
981boundary integral into the Poisson element itself, we treat
982the boundary integral over the element's face as a distinct
983lower-dimensional \c FiniteElement. We separate
984the specification of the element geometry from the implementation
985of the specific boundary conditions, and use templating and inheritance
986to create Neumann-type elements for entire families
987of elements. Here will discuss the \c PoissonFluxElement class,
988a class of \c FiniteElements that impose flux boundary conditions
989on the "faces" of associated "bulk" Poisson elements. The
990\c PoissonFluxElement class is
991templated by the element type of the associated "bulk" element
992so that a \c PoissonFluxElement<QPoissonElement<2,3> >
993is the three-node line element (from the \c QElement family) that
994imposes a prescribed-flux boundary condition along one of the
9951D edges of a nine-node quadrilateral Poisson element.
996Similarly, a \c PoissonFluxElement<QPoissonElement<3,2> >
997is the four-node quadrilateral element that
998imposes a prescribed-flux boundary condition on a
9992D face of an eight-node brick-shaped Poisson element.
1000
1001
1002Our implementation involves two new classes:
1003- The \c FaceElement class implements all the functionality that could
1004 conceivably be required by any \c FiniteElement that imposes
1005 Neumann-type boundary conditions on the faces of higher-dimensional
1006 "bulk" elements. (The \c PoissonFluxElements requires none
1007 of this functionality; and we defer a complete explanation of
1008 the structure of this class.)
1009- The \c FaceGeometry class, a policy class that defines
1010 the geometry of the lower-dimensional \c FiniteElements
1011 that form the faces for a given "bulk" element.
1012.

1014example in which we solve Poisson's equation in a rectangular
1015domain, with Neumann boundary conditions

1784\subsection mixed_interpolation Mixed interpolation
1785
1786In the examples considered so far, each equation in the system of
1787PDEs had the same structure and it was
1788natural to use the same interpolation for all unknowns
1789and the same test functions for each PDE in the system. However,
1790there are many systems of PDEs whose constituent equations
1791have different structures. As an example, we consider the
1792steady 2D Navier-Stokes equations, which comprise the two momentum equations
1793\f[
1794Re \sum_{k=1}^2 u_k \frac{\partial u_i}{\partial x_k} +
1795\frac{\partial p}{\partial x_i} -
1796\sum_{k=1}^2 \frac{\partial^2 u_i}{\partial x_k^2}
1797 =0 \mbox{\ \ \ \ $(i=1,...,2)$}
1798\f]
1799and the continuity equation
1800\f[
1801\sum_{k=1}^2 \frac{\partial u_k}{\partial x_k} =0.
1802\f]
1803The structure of the momentum equations differs from
1804that of the continuity equation. Furthermore, the equations contain
1805second derivatives of the two velocity components,
1806\f$ u_i \ (i=1,...2),\f$ but only first derivatives of the
1807pressure, \f$p\f$.
1808This raises two important questions:
1809- How should we define the weak form for this coupled problem? Should
1810 we use the same class of test functions for both types of equations
1811 or should we use a different class for each one?
1812- How should we represent the two types of unknowns? Should we use
1813 the same basis functions for the pressure and the velocity or should
1814 we use a different type of interpolation for each one?
1815.
1816For a general system of PDEs, these are difficult questions which can only
1817be answered by careful analysis of the governing
1818equations. Luckily, in the case of the Navier-Stokes equations,
1819the analysis is well-established and we refer
1820to the standard literature [e.g. P.M. Gresho and R.L. Sani:
1821"Incompressible Flow and the Finite Element Method" Vol. 1 and 2.
1822Wiley (2000)] which provides several possible answers to these
1823questions. "Equal-order interpolations", in which the
1824same shape functions are used to represent the pressure
1825and the velocity components, are generally found to result
1826in unstable elements. Most stable discretisations are
1827based on "mixed interpolations" in which the velocity components are
1828represented by one set of basis functions,
1829\f[
1830u_i(x_k) = \sum_{j=1}^N U_{ij} \ \psi_j(x_k),\ \ \ \ \ \ (8)
1831\f]
1832and another set of basis functions, \f$ \psi^{(P)}_j(x_k) \
1833(j=1,...,N_P)\f$, is employed to approximate the pressure
1834\f[
1835p(x_k) = \sum_{j=1}^{N_P} P_j \ \psi^{(P)}_j(x_k). \ \ \ \ \ (9)
1836\f]
1837The number of nodes involved in the representation of
1838the velocity, \f$N\f$, can (and usually does) differ from the
1839number of nodes involved in the representation of the pressure,
1840\f$N_P\f$.
1841
1842Stability constraints (the infamous LBB *REFERENCE*condition) impose
1843certain restrictions on the choice of basis functions.
1844One well-known stable discretisation employs piecewise quadratic interpolations
1845for the velocities and a piecewise linear interpolation for the
1846pressure. The resulting elements are known as "Taylor-Hood" elements.
1847When implemented as an isoparametric, 2D quadrilateral element, the
1848element geometry and the velocity components are represented
1849by the bi-quadratic local shape functions \f$ \psi_j(x_k)
1850\ \ (j=1,...,9)\f$ which interpolate between the nodal values
1851and coordinates of the element's \f$ n=9 \f$ local nodes.
1852The pressure is represented in terms of the bilinear local shape functions,
1853\f$ \psi^{(P)}_j(x_k) \ \ (j=1,...,4)\f$, which interpolate between
1854pressure values at the element's \f$ n_p=4 \f$ corner nodes, as
1855shown in this sketch:
1856
1857\image html taylor_hood.gif "Velocity (blue) and pressure (red) nodes in a quadrilateral Taylor-Hood "
1858\image latex taylor_hood.eps "Velocity (blue) and pressure (red) nodes in a quadrilateral Taylor-Hood " width=0.35\textwidth
1859
1860In this case, the number of nodal values is not the same for each
1861node: the corner nodes store two velocity components and one pressure;
1862all other nodes only store the two velocity components.
1863
1864 The weak form of the Navier-Stokes equations is obtained by
1865using the pressure shape functions \f$ \psi^{(P)}_j(s_k)\f$ as the
1866test functions for the continuity equation and the velocity shape
1867functions to test the momentum equations. Upon performing the
1868usual integration by parts and using the divergence
1869theorem (Exercise!), the (global) residuals
1870are then given by
1871\f[
1872f_{il} = \int_D \left[
1873\left( Re \sum_{k=1}^2 u_k \frac{\partial u_i}{\partial x_k} \right)
1874\psi_l - p \frac{\partial \psi_l}{\partial x_i} +
1875\sum_{k=1}^2 \frac{\partial u_i}{\partial x_k}
1876 \frac{\partial \psi_l}{\partial x_k} \right] dV = 0,
1877\ \ \ \ \ (10)
1878\f]
1879where we have ignored any boundary terms (hence, implying
1880the velocity is prescribed along the entire domain boundary),
1881and
1882\f[
1883f_{l}^{(P)} = \int_D \left( \sum_{k=1}^2
1884 \frac{\partial u_k}{\partial x_k} \right) \psi^{(P)}_l dV = 0.
1885\ \ \ \ \ (11)
1886\f]
1887The system of nonlinear algebraic equations (10)
1888provides one equation, \f$ f_{il} \f$, for each velocity
1889component \f$ (i=1,2) \f$ at those nodes \f$(l=1,...,N)\f$ where
1890the velocity is not prescribed by a Dirichlet boundary condition.
1891The weak continuity equations (11) provides
1892the remaining equations \f$ f_{l}^{(P)} =0 \f$ for the
1893pressure values \f$ P_j \ (j=1,...,N_p)\f$ that are not prescribed.
1894[Usually there are no boundary conditions for the pressure -- the
1895pressure can only be set indirectly through applied-traction boundary
1896conditions in which the normal component of the applied traction
1897contains a contribution from the pressure and the viscous normal stresses; only
1898if the velocity is prescribed along the entire boundary is it legal,
1899and, in fact, necessary, to pin a single pressure value.]

1918\subsection discontinuous_interpolation Discontinuous interpolation -- "Internal Data"
1919
1920The choice of basis functions in Taylor-Hood-type Navier-Stokes elements
1921appears to be perfectly natural. The bi-quadratic interpolation of the velocity
1922components between the element's nodal values enforces the inter-element
1923continuity of the velocity, which ensures that the integrals in
1924(10) "make sense". The LBB constraint requires the
1925use of a lower-order representation for the pressure; the "obvious"
1926choice is bilinear
1927interpolation between the pressure values at the four corner nodes,
1928which also ensures inter-element continuity of the
1929pressure. An
1930inspection of the weak form of the Navier-Stokes equations,
1931(10) and (11),
1932shows that this is not strictly required because the integration by parts has
1933removed the derivatives on the pressure. Hence the weak form "makes
1934sense" even if the pressure is globally discontinuous. This suggests the
1935use of different types of basis functions for the representation of
1936the pressure. An example is given by the Crouzeix-Raviart Navier-Stokes
1937elements in which the pressure within each element is represented
1938by the three basis functions
1939\f[
1940\psi_1^{(P)}(s_1,s_2) = 1,\ \ \ \ \ \
1941\psi_2^{(P)}(s_1,s_2) = s_1,\ \ \ \ \ \
1942\psi_3^{(P)}(s_1,s_2) = s_2,
1943\f]
1944so that the pressure in element \f$ e \f$ is given by the complete
1945linear polynomial,
1946\f[
1947p(s_1,s_2) = \sum_{j_p=1}^3 P_{j_p e} \ \psi_j^{(P)}(s_1,s_2).
1948\f]
1949In this representation, the discrete pressure values,
1950\f$P_{j_p e}\f$, are not associated with any Nodes -- instead,
1951they represent "internal" data for each individual element.
1952
1953<B>Implementation</B> \n\n

1957-# We split the \c Node class into two separate classes:
1958 - The base class \c Data which contains only the "values"

1960 - The derived class \c Node which inherits from \c Data and
1961 adds the additional functionality that is required by actual Nodes
1962 (storage for the Eulerian coordinates and the associated
1963 access functions).
1964-# Each FiniteElement provides storage for "Internal Data" in

1967 Data" is a vector of zero length. Specific FiniteElements

1969 amount of storage for their Internal Data (if they require any).
1970 Internal Data is included in the
1971 global equation numbering procedure by providing
1972 the function \c FiniteElement::assign_internal_eqn_numbers()
1973 which allocates distinct global equation numbers to all values
1974 that are stored in the element's "Internal Data". the
1975 function is executed for each element when the global equation
1976 numbers are assigned by \c Problem::assign_eqn_numbers().

1981\subsection higher_order_pdes Higher-order PDEs -- Hermite interpolants
1982
1983All the examples considered so far involved second-order PDEs
1984whose weak form could be integrated by parts. Therefore we only needed
1985to evaluate integrals that involve first derivatives
1986of the basis functions. Such integrals can be evaluated
1987for basis functions that are globally continuous but have
1988e.g. discontinuous first derivatives at isolated points (at the
1989element boundaries). The piecewise
1990linear/quadratic/... (global) finite-element shape functions
1991introduced in section \ref fe_basis_fct
1992satisfied these criteria. Since these representation of the function
1993with these shape functions implements a (Lagrange) interpolation
1994between the nodal values, such finite elements are often referred
1995to as Lagrange finite elements.
1996
1997Let us now consider the fourth-order problem
1998\f[ \frac{\mbox{d}^4 u(x)}{\mbox{d} x^4} = f(x) \mbox{ \ \ \
1999for $x\in[0,1]$\ \ \ \ \ \ \ \ \ (1)} \f]
2000subject to the essential boundary conditions
2001\f[ u(0)=g_0, \ \ u(1)=g_1, \ \
2002\left. \frac{\mbox{d}u}{\mbox{d}x}\right|_{x=0}=h_0, \ \
2003\left.\frac{\mbox{d}u}{\mbox{d}x}\right|_{x=1}=h_1,\f]
2004where \f$f(x)\f$ and the constants \f$g_0,g_1,h_0\f$ and \f$h_1\f$ are
2005given.
2006
2007We define the weak form of (1) by requiring that the (weak) solution
2008satisfies the essential boundary conditions and that the
2009weighted residual
2010\f[
2011r = \int_0^1 \left( \frac{\mbox{d}^4 u(x)}{\mbox{d} x^4} - f(x) \right)
2012 \phi^{(test)}(x) \ \mbox{d}x \ \ \ \ \ \ \ \ \ (2)
2013\f]
2014vanishes for all test functions \f$ \phi^{(test)}(x) \f$ which satisfy
2015the homogeneous boundary conditions
2016\f[ \phi^{(test)}(0)=0, \ \ \phi^{(test)}(1)=0, \ \
2017\left. \frac{d\phi^{(test)}}{dx}\right|_{x=0}=0, \ \
2018\left.\frac{d\phi^{(test)}}{dx}\right|_{x=1}=0.
2019\ \ \ \ \ \ \ \ \ (3) \f]
2020We can integrate the weak form (2) twice by parts to obtain
2021\f[
2022r = \int_0^1 \left(
2023 \frac{\mbox{d}^2 u(x)}{\mbox{d} x^2}
2024 \frac{\mbox{d}^2 \phi^{(test)}(x)}{\mbox{d} x^2}
2025 - f(x) \phi^{(test)}(x) \right) \ dx
2026 \ \ + \ \ \left[ \left. \frac{\mbox{d}^3 u(x)}{\mbox{d} x^3}
2027 \phi^{(test)}(x) \right] \right|_{0}^1
2028 \ \ - \ \ \left[ \left. \frac{\mbox{d}^2 u(x)}{\mbox{d} x^2}
2029 \frac{\mbox{d} \phi^{(test)}(x)}{\mbox{d} x} \right]
2030 \right|_{0}^1.
2031\f]
2032As before, the boundary terms vanish because \f$ \phi^{(test)}(x)\f$
2033satisfies the homogeneous boundary conditions (3), and we obtain
2034\f[
2035r = \int_0^1 \left(
2036 \frac{\mbox{d}^2 u(x)}{\mbox{d} x^2}
2037 \frac{\mbox{d}^2 \phi^{(test)}(x)}{\mbox{d} x^2}
2038 - f(x) \phi^{(test)}(x) \right) \ \mbox{d}x \ \ \ \ \ \ \ \ \ (4).
2039\f]
2040We now follow the usual (Galerkin) procedure of
2041expanding the solution \f$ u(x) \f$ and the test function
2042\f$ \phi^{(test)}(x)\f$ in terms of
2043the same set of basis functions \f$ \psi_j(x)\f$. The integral (4) shows
2044that the basis functions now
2045need to have square-integrable second derivatives, i.e.
2046we have to require that \f$ \psi_j(x) \in H^2_0(D)\f$ rather
2047than \f$ \psi_j(x) \in H^1_0(D)\f$ as in the case of the
2048second-order problem P1.
2049The Lagrange interpolants that are generated by piecewise
2050linear or quadratic finite element shape functions are not members
2051of \f$ H^2_0(D)\f$ because their first derivatives are discontinuous between
2052elements.
2053
2054Basis functions of sufficient differentiability can be
2055constructed by Hermite interpolation. For this purpose we introduce
2056two different types of basis functions:
2057- Basis functions of type 1 provide an interpolation between the
2058 nodal values and thus satisfy the conditions
2059 \f[ \psi_{j1}(X_k) = \delta_{jk}, \f]
2060 implying that they are equal to one at exactly \e one node while they
2061 vanish at all others. Furthermore, these basis functions satisfy
2062 \f[ \left. \frac{\mbox{d}\psi_{j1}}{\mbox{d}x} \right|_{X_k} = 0
2063 \mbox{\ \ for $j,k=1,...,N$}, \f]
2064 so that their derivatives vanish at \e all nodal points.
2065- Basis functions of type 2 provide an independent interpolation
2066 for the derivatives of the function and thus satisfy the conditions
2067 \f[ \left. \frac{\mbox{d}\psi_{j2}}{\mbox{d}x} \right|_{X_k} = \delta_{jk} , \f]
2068 implying that their derivative is equal to one at exactly
2069 \e one nodal point and zero at all others. Furthermore, these
2070 basis functions satisfy
2071 \f[ \psi_{j2}(X_k) = 0 \mbox{\ \ for $j,k=1,...,N$}, \f]
2072 so that they vanish at \e all nodal points.
2073
2074The global Hermite shape functions are illustrated in this sketch:
2075

2077\image latex 1Dmesh_with_hermite_shape_fcts.eps "1D Hermite shape " width=0.75\textwidth
2078
2079The global Hermite shape functions have local equivalents which are
2080again expressed in terms of a local coordinate \f$s \in [0,1]\f$
2081(note that the coordinate range differs from that for the Lagrange
2082elements) as

2088\f[ \psi_{21}(s) = 3s^2 - 2s^3 \f]
2089\f[ \psi_{22}(s) = s^3 - s^2. \f]

2116\subsection hyperbol_parabol_timestep Parabolic and hyperbolic problems -- timestepping
2117
2118The PDEs discussed so far were all of elliptic type. However,
2119the finite element method can also be used for the (semi-)discretisation
2120of parabolic and hyperbolic problems. We will demonstrate
2121this by considering the two classical examples of
2122these equations: i) the unsteady heat equation and ii) the wave equation.
2123\subsubsection unsteady_heat The unsteady heat equation
2124The 2D unsteady heat equation is given by
2125\f[
2126\sum_{k=1}^2 \frac{\partial^2 u(x_i,t)}{\partial x_k^2}
2127= \frac{\partial u(x_i,t)}{\partial t} + f(x_i,t)
2128\mbox{\ \ \ \ \ in $D$}
2129\f]
2130subject to the boundary condition
2131\f[
2132u\left(x_i^{\partial D}(\zeta),t\right) = g(\zeta,t)
2133\ \ \ \ \ \ \ \ (12)
2134\f]
2135and the initial condition
2136\f[
2137u(x_i,t=t_0) = u^{(IC)}(x_i), \ \ \ \ \ \ \ \ (13)
2138\f]
2139for the given functions \f$ f(x_i,t), \ g(\zeta,t) \f$
2140and \f$ u^{(IC)}(x_i)\f$.
2141Superficially, the only difference between this parabolic problem
2142and the elliptic Poisson problem P2 is the presence of the
2143time-derivative on the RHS of the PDE and the fact that the
2144boundary conditions and the source function are time-dependent.
2145Motivated by this observation, we employ the same spatial discretisation
2146as in the Poisson problem by writing
2147\f[
2148u(x_i,t) = \sum_{j=1}^N U_j(t) \ \psi_j(x_i)
2149\f]
2150where the coefficients \f$ U_j(t) \f$ are now
2151(continuous) functions of time \f$ t \f$. Following the
2152usual procedures, we obtain the residuals
2153\f[
2154r_l(t) = \int_D \left[ \sum_{j=1}^N \left( U_j(t)
2155\sum_{k=1}^2
2156\frac{\partial \psi_j(x_i)}{\partial x_k}
2157\frac{\partial \psi_l(x_i)}{\partial x_k} +
2158\frac{\mbox{d} U_j(t)}{\mbox{d} t} \psi_j(x_i) \ \psi_l(x_i) \right) +
2159f(x_i,t) \ \psi_l(x_i) \right] \mbox{d}x_1 \mbox{d}x_2 = 0
2160\ \ \ \ \ \ (14)
2161\f]
2162for all coefficients \f$ U_l(t) \f$ whose values are not determined
2163by the boundary conditions. The residuals now form
2164a system of ODEs, rather than a system of algebraic equations --
2165the spatial discretisation has only achieved a semi-discretisation.
2166We obtain the initial conditions for the unknown coefficients
2167\f$ U_l(t) \f$ by evaluating (13) at the nodal
2168positions
2169\f[
2170U_j(t=t_0) = u^{(IC)}(X_{ij})
2171\ \ \ \ \ \ (15)
2172\f]
2173The solution of the initial-value problem defined by
2174the system of first-order ODEs (14) and the
2175initial conditions (15) can now be obtained by
2176any ODE solver. Within \c oomph-lib we provide the implicit
2177timesteppers from the BDF family as the default
2178timesteppers for parabolic problems. In these schemes,
2179the time-derivatives are approximated by
2180\f[
2181\frac{\mbox{d} U_j}{\mbox{d} t} \approx
2182\sum_{\tau=0}^{N_{history}} \left.^\tau U_j\right. \left.^\tau W\right.
2183\ \ \ \ \ \ \ (16)
2184\f]
2185where the values \f$ \left.^\tau U_j\right. \f$ represent
2186the history of the nodal value \f$ U_j \f$, i.e. its values
2187at current and previous (discrete) timesteps.
2188If the time-integration is performed with a constant timestep
2189 \f$ \Delta t \f$, we have
2190\f[
2191 \left.^\tau U_j\right. = U_j(t-\tau \Delta t)
2192\f]
2193so \f$ \left.^0 U_j\right. \f$
2194represents the (unknown) nodal value at the present time.
2195The weights \f$ \left.^\tau W\right. \f$ are constants that depend
2196only on the timestep \f$ \Delta t\f$. And, yes,
2197the \f$ \tau \f$ is a left superscript.
2198
2199The different members of the BDF family are distinguished by the
2200number of previous values used in the approximation (16).
2201For the BDF1 scheme we have \f$ N_{history}=1\f$ with the
2202weights
2203\f[
2204\left.^0 W\right. = \frac{1}{\Delta t} \mbox{\ \ \ \ and \ \ \ }
2205\left.^1 W\right. = - \frac{1}{\Delta t}
2206\f]
2207Inserting the weights into (16), we recover the
2208classical first-order-accurate backward Euler scheme:
2209\f[
2210\frac{\mbox{d} U_j}{\mbox{d} t} \approx
2211\frac{ \left.^0 U_j\right. - \left.^1 U_j\right. }{\Delta t}
2212\f]
2213Similarly, in the BDF2 scheme we have \f$ N_{history}=2\f$ with the
2214weights
2215\f[
2216\left.^0 W\right. = \frac{3}{2\Delta t}, \
2217\left.^1 W\right. = - \frac{2}{\Delta t}\mbox{\ \ \ \ and \ \ \ }
2218\left.^2 W\right. = \frac{1}{2\Delta t}
2219\f]
2220which yields the second-order backward Euler scheme:
2221\f[
2222\frac{\mbox{d} U_j}{\mbox{d} t} \approx
2223\frac{ 3\left.^0 U_j\right. - 4 \left.^1 U_j\right. +
22241\left.^2 U_j\right. }{2\Delta t}
2225\f]
2226Inserting the approximations (16) for the time-derivatives
2227into (14) gives a system of algebraic
2228equations
2229 \f[
2230 r_l = \int_D \left\{ \sum_{j=1}^N \left.^{0}U_j \right. \left[
2231 \sum_{k=1}^2 \left(
2232 \frac{\partial \psi_j(x_i)}{\partial x_k}
2233 \frac{\partial \psi_l(x_i)}{\partial x_k} \right) +
2234 \left.^{0} W \right.
2235 \psi_j(x_i) \ \psi_l(x_i) \right] +
2236 \sum_{\tau=1}^{N_{history}}
2237 \left.^{\tau} W \right. \left.^{\tau} U_j\right.
2238 f(x_i,t) \ \psi_l(x_i) \right\} dx_1 dx_2 = 0
2239 \ \ \ \ \ \ (17)
2240 \f]
2241that can be solved to determine the unknown values of the coefficients
2242\f$ \left.^{0} U_j\right. = U_j \f$ at the present time.
2243
2244<CENTER> <B>Algorithm *** Timestepping</B></CENTER>
2245-# Set time \f$ t \f$ to its initial value, \f$ t=t_0. \f$
2246-# Choose a timestep \f$\Delta t \f$.
2247-# Compute the weights \f$ \left.^{\tau} W \right. \f$
2248 for \f$ \tau=0,...,N_{history}\f$.
2249-# Assign initial values for all history
2250 values so that the initial conditions are satisfied
2251 at \f$ t=t_0 \f$.
2252-# Move to the next timestep:
2253 - Increase the value of the continuous time:
2254 \f[
2255 t=t+\Delta t
2256 \f]
2257 - Update the history values. For a BDF scheme, this
2258 involves shifting the history values back by one
2259 time level:
2260 - Loop over the time levels backwards, \f$ \tau=N_{history},...,1 \f$
2261 \f[
2262 \left.^{\tau} U_j\right. = \left.^{\tau-1} U_j\right.
2263 \f]
2264 .
2265 - Update the nodal values of all nodes on the boundary according to the
2266 time-dependent boundary conditions (12).
2267 - Solve ((17)) (evaluating the
2268 source function at the present time \f$ t \f$) for the
2269 unknown nodal values \f$ \left.^{0} U_j\right. = U_j \f$.
2270 - Document the solution.
2271-# Go to *** for the next timestep.
2272
2273A key step in this algorithm is the provision of
2274history values for the nodal values at \f$ t=t_0 \f$. Recall that the
2275state of the system at this time is determined
2276by the initial condition (13) which determines
2277the values of \f$ \left.^{0} U_j\right. \f$.
2278The value of \f$ \left.^{1} U_j\right. \f$ is
2279not determined by (13) but could be obtained
2280by solving (***)
2281
2282Before advancing to the next (=first) timestep, we shift
2283this history values back therefore the initial value for

2301\subsection solids_lagrange Solid mechanics problems -- Lagrangian coordinates
2302All problems discussed so far were formulated in an Eulerian frame,
2303i.e. we assumed all unknowns to be functions of the spatially-fixed,
2304Eulerian coordinates, \f$x_i \ (i=1,...,3)\f$, and of time,
2305\f$ t \f$. Many problems
2306in solid mechanics are formulated more conveniently in terms of
2307body-attached, Lagrangian coordinates. In this section we will
2308briefly review the essential concepts of nonlinear continuum mechanics
2309and present the principle of virtual displacements which forms the
2310basis for all large-displacement solid mechanics computations in \c oomph-lib.
2311
2312Throughout this section we will use the summation convention
2313so that repeated indices are assumed to be summed over the range
2314of the three spatial coordinates and all free indices range
2315from 1 to 3. We will retain the summation signs
2316for all other sums, such as sums over the nodes etc.
2317
2318\subsubsection geometry The geometry
2319The figure below introduces the essential geometrical concepts:
2320We employ a set of Lagrangian coordinates, \f$ \xi^i\f$, to
2321parametrise the (Eulerian) position vector to material points in the
2322 body's undeformed position:
2323\f[
2324{\bf r} = {\bf r}(\xi^i).
2325\f]
2326The specific choice of the Lagrangian coordinates is
2327irrelevant for the subsequent development. For analytical studies, it is
2328advantageous to choose a body-fitted Lagrangian coordinate
2329system (as shown in the sketch) because this allows boundary
2330conditions to be applied on iso-levels of the coordinates;
2331in computational studies, it is usually preferable to use a
2332coordinate system in which the governing equations are as compact
2333as possible. A cartesian coordinate system is best suited for this
2334purpose.
2335
2336We denote the tangent vectors to the coordinate lines \f$ \xi^i = const.\f$
2337in the undeformed configuration by
2338\f[
2339{\bf g}_i = \frac{\partial {\bf r}}{\partial \xi^i}
2340\f]

2342the inner products
2343\f[

2345\f]

2347length, \f$ ds \f$, of a line element with coordinate increments
2348\f$ d\xi^i\f$ is given by
2349\f[
2350(ds)^2 = g_{ij} \ d\xi^i d\xi^j.
2351\f]
2352The volume of the infinitesimal parallelepiped formed by
2353the coordinate increments \f$ d\xi^i\f$ is given by
2354\f[
2355dv = \sqrt{g} \ d\xi^1 d\xi^2 d\xi^3,
2356\f]
2357where
2358\f[
2359g = \det g_{ij}.
2360\f]

2364As the body deforms, the Lagrangian coordinates remain "attached" to the
2365same material points. The body's deformation can therefore be
2366described bt the vector field that specifies the position vectors
2367to material particles in the deformed configuration,
2368\f[
2369{\bf R} = {\bf R}(\xi^i).
2370\f]
2371As in the undeformed coordinate system, we form the
2372tangent vectors to the deformed coordinate lines \f$ \xi^i = const.\f$
2373and denote them by
2374\f[
2375{\bf G}_i = \frac{\partial {\bf R}}{\partial \xi^i}.
2376\f]
2377The inner product of these vectors defines the metric tensor in
2378the deformed configuration
2379\f[
2380G_{ij} = {\bf G}_i \cdot {\bf G}_j
2381 = \frac{\partial {\bf R}}{\partial \xi^i} \cdot
2382 \frac{\partial {\bf R}}{\partial \xi^j},
2383\f]
2384and we have equivalent relations for the lengths of line elements,
2385\f[
2386(dS)^2 = G_{ij} \ d\xi^i d\xi^j
2387\f]
2388and the volume of infinitesimal parallelepipeds,
2389\f[
2390dV = \sqrt{G} \ d\xi^1 d\xi^2 d\xi^3,
2391\f]
2392where
2393\f[
2394G = \det G_{ij}.
2395\f]
2396Since the metric tensors \f$ G_{ij }\f$ and \f$ g_{ij }\f$ provide
2397a measure of the length of
2398material line elements in the deformed and undeformed configurations,
2399respectively, their difference
2400\f[
2401\gamma_{ij} = \frac{1}{2} (G_{ij} - g_{ij})
2402\f]
2403provides an objective measure of the strain and is known as
2404the Green strain tensor.

2406\subsubsection equilibrium Equilibrium and the Principle of Virtual Displacements
2407
2408Let us now assume that the body is subjected to
2409- an applied surface traction \f$ {\bf t} \f$ --
2410 a force per unit deformed surface area, applied on (part of) the
2411 body's deformed surface area \f$ A_{tract} \subset \partial V\f$,
2412- a body force \f$ {\bf f} \f$ --
2413 a force per unit volume of the undeformed (!) body [This can
2414 easily be expressed in terms of a force per unit deformed volume,
2415 \f$ {\bf F }\f$, by invoking conservation of mass, which shows that
2416 \f$ {\bf f } = \sqrt{G/g} \ {\bf F }\f$.],
2417.
2418while its displacements are prescribed on the remaining part of
2419the boundary, \f$ A_{displ} \subset \partial V \f$ (where
2420\f$ A_{tract} \cap A_{displ}=0 \f$ and \f$A_{tract} \cup A_{displ}
2421= \partial V\f$ ), so that
2422\f[
2423{\bf R}(\xi^k) = {\bf R}^{(BC)}(\xi^k)
2424\mbox{ \ \ \ \ on \ \ \ $ A_{displ},$ }
2425 \ \ \ \ \ \ \ \ (19)
2426\f]
2427where \f$ {\bf R}^{(BC)}(\xi^k) \f$ is given.
2428\image html Solid_boundary_conds.gif "Sketch of the boundary conditions: The surface of the body is either subject to a prescribed traction or its displacement is "
2429\image latex Solid_boundary_conds.eps "Sketch of the boundary conditions: The surface of the body is either subject to a prescribed traction or its displacement is " width=0.75\textwidth
2430The deformation is governed by the principle of virtual displacements
2431\f[
2432\int \left\{ \sigma^{ij} \ \delta \gamma_{ij} - \left( {\bf f} -
2433 \rho \frac{\partial^2 {\bf R}}{\partial t^2} \right) \cdot
2434 \delta {\bf R} \right\} \ dv -
2435 \oint_{A_{tract}} {\bf t} \cdot \delta {\bf R} \ dA =0,
2436 \ \ \ \ \ \ \ \ (20)
2437 \f]
2438 where \f$ \sigma^{ij} = \sigma^{ji}\f$ is the (symmetric)
2439 second Piola-Kirchhoff stress tensor, \f$ \rho \f$ is the density, and
2440 \f$ \delta (.) \f$ represents the variation of \f$ (.)\f$.
2441 See, e.g., Green, A.E. \& Zerna, W. "Theoretical Elasticity",
2442 Dover (1992); or Wempner, G. "Mechanics of Solids with Applications
2443 to Thin Bodies", Kluwer (1982) for more details.
2444
2445 Upon choosing the particles' position vector in the deformed
2446 configuration, \f${\bf R}(\xi^j)\f$, as the unknown, the variation
2447 of the strain tensor becomes
2448 \f[
2449 \delta \gamma_{ij} = \frac{1}{2} \left(
2450 \frac{\partial {\bf R}}{\partial \xi^i} \cdot
2451 \delta \ \frac{\partial {\bf R}}{\partial \xi^j} +
2452 \frac{\partial {\bf R}}{\partial \xi^j} \cdot
2453 \delta \ \frac{\partial {\bf R}}{\partial \xi^i}
2454 \right)
2455 \f]
2456 and we have
2457 \f[
2458 \delta {\bf R} = {\bf 0} \mbox{ \ \ \ \ on \ \ \ $ A_{displ}.$ }
2459 \ \ \ \ \ \ \ \ (21)
2460 \f]
2461 The 2nd Piola Kirchhoff stress tensor is symmetric, therefore
2462 we can write the variation of the strain energy in
2463 (20) as
2464 \f[
2465 \int \sigma^{ij} \ \delta \gamma_{ij} \ dv =
2466 \int \sigma^{ij} \ \frac{\partial {\bf R}}{\partial \xi^i} \cdot
2467 \delta \ \frac{\partial {\bf R}}{\partial \xi^j} \ dv
2468 \f]
2469 and obtain the displacement form of the principle of
2470 virtual displacements:
2471 \f[
2472 \int \left\{ \sigma^{ij} \ \frac{\partial {\bf R}}{\partial \xi^i} \cdot
2473 \delta \ \frac{\partial {\bf R}}{\partial \xi^j}
2474 - \left( {\bf f} -
2475 \rho \frac{\partial^2 {\bf R}}{\partial t^2} \right) \cdot
2476 \delta {\bf R} \right\} \ dv
2477 - \oint_{A_{tract}} {\bf t} \cdot \delta {\bf R} \ dA =0.
2478 \f]
2479This needs to be augmented by a constitutive equation which
2480determines the stress as a function of the body's deformation,
2481(and possibly the history of its deformation). Here we will only
2482consider elastic behaviour, where the stress is only a function of the
2483strain.

2485\subsubsection elastic_constitutive Constitutive Equations for Purely Elastic Behaviour
2486For purely
2487elastic behaviour, the stress is only a function of the
2488instantaneous, local strain and the constitutive equation
2489has the form
2490\f[
2491\sigma^{ij} = \sigma^{ij} (\gamma_{kl}).
2492\f]
2493
2494The functional form of the constitutive equation is different
2495for compressible/incompressible/near-incompressible behaviour:
2496-# \b Compressible \b Behaviour: \n If the material is compressible,
2497 the stress can be computed directly from the deformed and undeformed
2498 metric tensors,
2499 \f[
2500 \sigma^{ij} = \sigma^{ij}(\gamma_{kl}) =
2501 \sigma^{ij}\bigg( \frac{1}{2} (G_{kl} - g_{kl})\bigg).
2502 \f]
2503-# \b Incompressible \b Behaviour: \n If the material is incompressible,
2504 its deformation is constrained by the condition that
2505 \f[
2506 \det G_{ij} - \det g_{ij}= 0
2507 \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ (22)
2508 \f]
2509 which ensures that the volume of infinitesimal material
2510 elements remains constant during the deformation. This
2511 condition is typically enforced by a Lagrange multiplier which
2512 plays the role of a pressure. In such cases, the
2513 stress tensor has the form
2514 \f[
2515 \sigma^{ij} = -p G^{ij} +
2516 \overline{\sigma}^{ij}\big(\gamma_{kl}\big),
2517 \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ (23)
2518 \f]
2519 where only the deviatoric part of the stress tensor,
2520 \f$ \overline{\sigma}^{ij}, \f$ depends directly on the
2521 strain. The pressure \f$ p \f$ needs to be determined
2522 independently by enforcing the incompressibility constraint
2523 (22).
2524 Given the deformed and undeformed metric tensors,
2525 the computation of the stress tensor \f$ \sigma^{ij} \f$
2526 for an incompressible
2527 material therefore requires the computation of the following quantities:
2528 - The deviatoric stress \f$ \overline{\sigma}^{ij} \f$
2529 - The contravariant deformed metric tensor \f$ G^{ij} \f$
2530 - The determinant of the deformed
2531 metric tensor \f$ \det G_{ij} \f$ which
2532 is required in equation (22)
2533 whose solution determines the pressure.
2534 .
2535 \n
2536-# \b Nearly \b Incompressible \b Behaviour: \n If the material is nearly
2537 incompressible, it is advantageous to split the stress into
2538 its deviatoric and hydrostatic parts by writing the
2539 constitutive law in the form
2540 \f[
2541 \sigma^{ij} = -p G^{ij} +
2542 \overline{\sigma}^{ij}\big(\gamma_{kl}\big),
2543 \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ (24)
2544 \f]
2545 where the deviatoric part of the stress tensor,
2546 \f$ \overline{\sigma}^{ij}, \f$ depends on the
2547 strain. This form of the constitutive
2548 law is identical to that of the incompressible
2549 case and it involves a pressure \f$ p \f$ which needs to be
2550 determined from an additional equation. In the
2551 incompressible case, this equation was given by the incompressibility
2552 constraint (22). Here, we need to augment
2553 the constitutive law for the deviatoric stress by
2554 an additional equation for the pressure. Generally this takes the
2555 form
2556 \f[
2557 p = \kappa \ d
2558 \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ (25)
2559 \f]
2560 where \f$ \kappa \f$ is the "bulk modulus", a material property
2561 that needs to be specified by the constitutive law.
2562 \f$ d \f$ is the (generalised) dilatation, i.e. the relative change
2563 in the volume of an infinitesimal material element (or some
2564 suitable generalised quantity that is related to it). As the
2565 material approaches incompressibility, \f$ \kappa \to \infty\f$, so that
2566 infinitely large pressures would be required to achieve
2567 any change in volume. To facilitate the implementation of
2568 (25) as the equation for the pressure, we re-write
2569 it in the form
2570 \f[
2571 p \ \frac{1}{\kappa} - d\big(g_{ij},G_{ij}\big) = 0
2572 \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ (26)
2573 \f]
2574 which only involves quantities that remain finite
2575 as we approach true incompressibility.
2576 \n
2577 Given the deformed and undeformed metric tensors,
2578 the computation of the stress tensor \f$ \sigma^{ij} \f$
2579 for a nearly incompressible
2580 material therefore requires the computation of the following quantities:
2581 - The deviatoric stress \f$ \overline{\sigma}^{ij} \f$
2582 - The contravariant deformed metric tensor \f$ G^{ij} \f$
2583 - The generalised dilatation \f$ d \f$
2584 - The inverse of the bulk modulus \f$ \kappa \f$
2585 .
2586 \n
2587
2588The abstract base class \c ConstitutiveLaw provides
2589interfaces for the computation of the stress in all three forms.

2591\subsubsection non-dim_solid Non-dimensionalisation
2592The principle of virtual displacements (20)
2593is written in dimensional form.
2594We generally prefer to work with non-dimensional quantities and
2595will now discuss the non-dimensionalisation used in the
2596actual implementation of the equations in \c oomph-lib. For this purpose we
2597first re-write equation (20) as
2598\f[
2599\int_{v} \left\{ \sigma^{*ij} \ \delta \gamma_{ij} - \left( {\bf f}^* -
2600 \rho \frac{\partial^2 {\bf R}^*}{\partial t^{*2}} \right) \cdot
2601 \delta {\bf R}^* \right\} \ dv^*
2602 - \oint_{A_{tract}} {\bf t}^* \cdot \delta {\bf R}^* \ dA^* =0,
2603 \ \ \ \ \ \ \ \ (27)
2604 \f]
2605where have used asterisks to label those
2606dimensional quantities that will have non-dimensional equivalents.
2607(Some quantities, such as the strain, are already
2608dimensionless, while others, such as the density will not have
2609any non-dimensional counterparts. We do not introduce modifiers
2610for these).
2611
2612We now non-dimensionalise all lengths with a problem-specific
2613length-scale, \f$ {\cal L}\f$, given e.g. the length of the solid
2614body, so that
2615\f[
2616\xi^{*i} = {\cal L} \ \xi^{i}, \ \ \ {\bf R}^* = {\cal L} \ {\bf R},
2617\ \ \ dA^* = {\cal L}^2 \ dA, \mbox{ \ \ and \ \ }
2618dv^* = {\cal L}^3 \ dv.\ \ \
2619\f]
2620We use a characteristic stiffness, \f$ {\cal S}\f$, (e.g. the
2621material's Young's modulus \f$ E \f$) to non-dimensionalise
2622the stress and the loads as
2623\f[
2624\sigma^{*ij} = {\cal S} \ \sigma^{ij}, \ \ \ {\bf t}^* = {\cal S}\
2625{\bf t}, \mbox{ \ \ and \ \ } {\bf f}^* = {\cal S} / {\cal L} \
2626{\bf f},
2627\f]
2628and we non-dimensionalise time with a problem-specific
2629timescale \f$ {\cal T}\f$ (e.g. the period of some external forcing),
2630so that
2631\f[
2632t^* = {\cal T} t.
2633\f]
2634This transforms (27) into
2635\f[
2636\int_{v} \left\{ \sigma^{ij} \ \delta \gamma_{ij} - \left( {\bf f} -
2637 \Lambda^2 \frac{\partial^2 {\bf R}}{\partial t^{2}} \right) \cdot
2638 \delta {\bf R} \right\} \ dv
2639 -\oint_{A_{tract}} {\bf t} \cdot \delta {\bf R} \ dA =0,
2640 \ \ \ \ \ \ \ \ (28)
2641 \f]
2642 where
2643\f[
2644\Lambda = \frac{1}{\cal T} \sqrt{\frac{\rho {\cal L}}{\cal S}}
2645\f]
2646is the ratio of system's "intrinsic" timescale,
2647\f$T_{intrinsic} = \sqrt{\rho {\cal L}/{\cal S}},\f$ to the
2648time, \f${\cal T}\f$, used in the non-dimensionalisation of the
2649equations. If a given problem has no externally imposed timescale
2650(e.g. in the free vibrations of an elastic body) \f$T_{intrinsic}\f$
2651(or some suitable problem-dependent multiple thereof) provides a natural
2652timescale for the non-dimensionalisation. Therefore we use
2653\f$ \Lambda=1 \f$ as the default value in all solid mechanics
2654equations. If preferred, computations can, of course, be
2655be performed with dimensional units, provided all quantities
2656are expressed in consistent units (e.g. from the SI system).
2657In this case \f$ \Lambda^2 \f$ represents the dimensional
2658density of the material.
2659
2660We adopt a similar approach for non-dimensionalisation of the
2661constitutive equations. Typically, the constitutive parameters
2662(e.g. Young's modulus and Poisson's ratio for a Hookean
2663constitutive equation) are passed to the \c ConstitutiveLaw
2664as arguments to its constructor. Where possible, we select one of these
2665parameters as the reference stress \f$ {\cal S} \f$ and give it
2666a default value of 1.0. Hence, if a Hookean constitutive
2667law is instantiated with just one argument (the Poisson ratio \f$
2668\nu \f$), the stress is assumed to have been scaled on Young's
2669modulus. If two arguments are provided, the second argument
2670should be interpreted as the ratio of
2671the material's Young's modulus to the reference stress
2672\f$ {\cal S} \f$ used in the non-dimensionalisation of the equations.

2674\subsubsection 2D_solid 2D problems: Plane strain.
2675Many solid mechanics problems can be regarded as two-dimensional
2676in the sense that the quantities of interest only depend on two
2677spatial coordinates. In such problems it is important to consider what
2678constraints the system is subjected to in the third direction -- clearly
2679all real solid bodies are three-dimensional!
2680In plane strain problems, the displacements of material points are assumed
2681to be parallel to the 2D plane spanned by the coordinates \f$x_1
2682\f$ and \f$ x_2\f$, so that any displacements
2683normal to this plane are suppressed. In this case, significant
2684transverse stresses can develop.
2685Conversely, in plane stress problems, it is assumed that no stresses
2686develop in the transverse direction; in this case we need
2687to allow material particles to be displaced transversely.
2688Since we have formulated our problem in terms of positions
2689(i.e. in terms of the displacement of material points), our formulation
2690naturally produces a <B>plane strain</B> problem if we reduce
2691the equations to two dimensions:
2692We assume that the transverse displacement vanishes,
2693while the remaining (in-plane) displacements are only
2694functions of the in-plane coordinates, \f$ x_1\f$ and \f$ x_2\f$.
2695
2696It is important to remember that the 2D version of all equations needs to
2697produce plane-strain behaviour when new, strain-energy-based
2698constitutive equation classes are formulated:
2699When implementing such strain-energy functions, recall that any
2700invariants of metric tensors etc. are the invariants of the full
27013D quantities which are not necessarily the same as those
2702of the corresponding 2D quantities.

2706\subsubsection isotropic_growth Isotropic growth.
2707Many biological tissues undergo growth processes. For instance, the cells
2708that make up a solid tumour divide regularly and as a result the
2709total mass of the tumour increases. If the growth
2710occurs non-uniformly so that certain parts of the tumour grow faster
2711than others, regions that grow more slowly restrain the expansion
2712of their neighbours. This process can induce significant growth stresses.
2713The scenario is similar (but not identical) to that of thermal
2714growth in which a non-uniform temperature distribution in
2715a body creates thermal stresses. (The important difference between
2716these two cases is that in the latter process mass is conserved --
2717thermal expansion only leads to an increase in the volume occupied
2718by the material, whereas biological growth via cell division
2719increases the mass of the tumour).

2823U2). In this state, the body is
2824stress-free and, compared to the reference configuration "0",
2825all lengths have increased by a factor of \f$\Gamma^{1/d}\f$
2826where \f$ d \f$ is the body's spatial dimension (i.e. \f$d=2\f$ in
2827the sketch above). The covariant basis vectors in this
2828uniformly-grown, stress-free configuration are therefore given by
2829\f[
2830\widetilde{\bf g}_i = \Gamma^{1/d} \ {\bf g}_i,
2831\f]
2832the metric tensor is given by
2833\f[
2834\widetilde{g_{ij}} = \Gamma^{2/d} \ g_{ij},
2835\f]
2836and the volume of an infinitesimal material element
2837has increased from
2838\f[
2839dv = \sqrt{g} \ d\xi^1 ... \ d\xi^D
2840\f]
2841to
2842\f[
2843\widetilde{dv} = \Gamma \sqrt{g} \ d\xi^1 ... \ d\xi^D
2844\ \ \ \ \ \ \ \ \ \ \ \ (29)
2845\f]
2846We now subject the stress-free, uniformly-grown body to
2847external loads and determine its deformation, using the principle
2848of virtual displacements. Since the uniformly grown state "U2"
2849is stress-free, we may regard it as the reference state for
2850the subsequent deformation. The strain tensor that describes
2851the deformation from the stress-free (and uniformly-grown) state "U2"
2852to the final equilibrium configuration "UE" must therefore
2853be defined as
2854\f[
2855\gamma_{ij} = \frac{1}{2} \left(G_{ij} - \widetilde{g_{ij}}\right)
2856 = \frac{1}{2} \left(G_{ij} - \Gamma^{2/D} \ g_{ij} \right)
2857\ \ \ \ \ \ \ \ \ \ \ \ (30)
2858\f]
2859Equations (29) and (30) allow
2860us to express the principle of virtual displacements
2861in terms of
2862- the metric of the undeformed (and un-grown) reference
2863 state "0",
2864- the volumetric growth \f$ \Gamma \f$, and
2865- quantities associated with the final deformed configuration
2866 "UE":
2867.
2868\f[
2869\int_{v} \left\{ \sigma^{ij} \ \delta \gamma_{ij} - \left( {\bf f} -
2870 \Lambda^2 \frac{\partial^2 {\bf R}}{\partial t^{2}} \right) \cdot
2871 \delta {\bf R} \right\} \ \Gamma \ dv
2872 -\oint_{A_{tract}} {\bf t} \cdot \delta {\bf R} \ dA =0,
2873 \ \ \ \ \ \ \ \ (31)
2874\f]
2875Note that this equation does not contain any references to quantities
2876in the intermediate states "U1" and "U2".
2877
2878We will now consider the case of spatially non-uniform growth,
2879illustrated in the right half of the sketch. If the isotropic
2880growth is spatially non-uniform, \f$ \Gamma = \Gamma(\xi^i)\f$,
2881the growth will try to expand all infinitesimal material
2882elements isotropically -- but each one by a different amount as
2883illustrated by the hypothetical state N1 in which each material
2884element has expanded to its stress-free state in which its
2885metric tensor is given by
2886\f[
2887\widetilde{g_{ij}} = \Gamma(\xi^k) \ g_{ij}.
2888\f]
2889Material elements will only be stress-free if the strain
2890\f[
2891\gamma_{ij} = \frac{1}{2} \left(G_{ij} - \widetilde{g_{ij}}\right)
2892 = \frac{1}{2} \left(G_{ij} - \Gamma^{2/D}(\xi^k) \ g_{ij} \right)
2893\ \ \ \ \ \ \ \ \ \ \ \ (32)
2894\f]
2895relative to their isotropically grown shape in state N1 is zero.
2896In general, the displacements induced by such an isotropic
2897expansion will be
2898incompatible and it would be impossible to (re-)assemble the
2899individually grown material elements to a continuous body
2900unless the material elements undergo some deformation.
2901The elements' deformation relative to their
2902stress-free shape in N1 will generate internal "growth-stresses"
2903(stage N2). When subjected to external loads and body forces
2904the body will undergo further deformations until the
2905the stress (generated by the particles' total deformation relative to
2906their stress-free state N1) balances the applied loads.
2907
2908It is important to realise that, as in the case of spatially
2909uniform growth, the strain defined by (30) is
2910an intrinsic quantity that provides a measure of each particles'
2911\e local deformation relative to its stress-free shape in N1.
2912The intermediate (and in the current case hypothetical), isotropically
2913grown state N1 does not appear in the analysis -- it only serves
2914to define the stress-free shape for each infinitesimal
2915material element. Equation (31) therefore
2916remains valid.

2919\subsubsection CartesianLagrangian Specialisation to a cartesian basis and finite element discretisation
2920If the problem does not have any
2921symmetries (e.g. axisymmetry) whose exploitation would reduce
2922the spatial dimension of the problem, the most compact form
2923of the equations is obtained by resolving all vectors into
2924a fixed cartesian basis so that the undeformed position vector
2925is given by
2926\f[
2927{\bf r}(\xi^j) = r_i(\xi^j) \ {\bf e}_i,
2928 \ \ \ \ \ \ \ \ (33)
2929\f]
2930Similarly, we write
2931\f[
2932{\bf f} = f_i \ {\bf e}_i,
2933\f]
2934\f[
2935{\bf t} = t_i \ {\bf e}_i,
2936\f]
2937and
2938\f[
2939{\bf R}^{(BC)} = R^{(BC)}_i \ {\bf e}_i.
2940\f]
2941
2942We use the Eulerian coordinates in the undeformed
2943configuration as the Lagrangian coordinates so that
2944\f[
2945r_i(\xi^j) = \xi^i. \ \ \ \ \ \ \ \ (34)
2946\f]
2947With this choice, the tangent vectors to the undeformed coordinate lines
2948are the cartesian basis vectors
2949\f[
2950{\bf g}_i = {\bf e}_i,
2951\f]
2952and the undeformed metric tensor is the Kronecker delta
2953(the unit matrix)
2954\f[
2955g_{ij} = \delta_{ij}.
2956\f]
2957We now expand the (unknown) deformed position vector in the same basis,
2958\f[
2959{\bf R}(\xi^j) = R_i(\xi^j) \ {\bf e}_i,
2960\f]
2961and derive a finite element approximation for this vector field
2962from the principle of virtual displacements. For this purpose we decompose
2963the undeformed body into a number of finite elements, using the
2964standard mesh generation process described previously.
2965This establishes the Eulerian position \f$ X_{ij}^{(0)} \
2966(j=1,...,N) \f$ of the \f$ N \f$
2967nodes in the body's undeformed configuration.
2968Since the Eulerian coordinates of material points in the undeformed
2969configuration coincide with their Lagrangian coordinates (see
2970(34)), a finite-element representation of the
2971Lagrangian coordinates is established by writing
2972\f[
2973\xi^i = \sum_{j=1}^{N} \Xi_{ij} \ \psi_j,
2974\f]
2975where \f$ \Xi_{ij} = X_{ij}^{(0)} \f$ is the \f$i\f$-th
2976Lagrangian coordinate of global node \f$j\f$, and the \f$\psi_j\f$
2977are the global finite-element shape functions. In practice, the
2978\f$\psi_j\f$ are, of course, represented by local shape functions,
2979\f$\psi_j(s_k)\f$, so that the Lagrangian coordinate at local
2980coordinate \f$ s_k \f$ in element \f$ e \f$ is given by
2981\f[
2982\xi^i(s_k) = \sum_{j=1}^{n} \Xi_{i{\cal J}(j,e)} \ \psi_j(s_k).
2983\f]
2984We employ the same basis functions to represent the components of
2985the unknown vector field \f$ {\bf R}(\xi^j)\f$, by writing
2986\f[
2987R_i(\xi^k) = \sum_{j=1}^{n} X_{ij} \ \psi_j(\xi^k),
2988\ \ \ \ \ \ \ \ (35)
2989\f]
2990and treat the (Eulerian) nodal positions \f$ X_{ij} \f$ as
2991the unknowns. With this discretisation, the
2992variations in \f$ {\bf R}(\xi^j) \f$ correspond to variations
2993in the nodal positions \f$ X_{ij} \f$ so that
2994\f[
2995\delta {\bf R} = \sum_{j=1}^{N} \delta X_{ij} \ \psi_j \ {\bf e}_i
2996\f]
2997\f[
2998\delta \frac{\partial {\bf R}}{\partial \xi^k} =
2999\sum_{j=1}^{N} \delta X_{ij} \
3000\frac{\partial \psi_j}{\partial \xi^k} \ {\bf e}_i,
3001\f]
3002\f[
3003\mbox{ etc. }
3004\f]
3005The principle of virtual displacement (31)
3006therefore becomes
3007 \f[\sum_{m=1}^{N}
3008 \left\{
3009 \int \left[ \sigma^{ij} \
3010 \left(\sum_{l=1}^N X_{kl} \frac{\partial \psi_l}{\partial \xi^i} \right)
3011 \frac{\partial \psi_m}{\partial \xi^j} \
3012 - \left( f_k -
3013 \Lambda^2 \left(\sum_{l=1}^N
3014 \frac{\partial^2 X_{kl}}{\partial t^2} \psi_l \right) \right)
3015 \psi_m \right] \ \Gamma \ dv
3016 - \oint_{A_{tract}} t_k \ \psi_m \ dA \right\}
3017 \delta X_{km} =0. \ \ \ \ (36)
3018 \f]
3019[Note that summation convention enforces the summation over the index
3020\f$ k \f$.]
3021The displacement boundary condition (19)
3022determines the positions of all nodes that are located on the
3023boundary \f$A_{displ}\f$,
3024\f[
3025 X_{ij} = R^{(BC)}_i(\Xi_{lj}) \mbox{ \ \ \ if node $j$ is
3026 located on $A_{displ},$}
3027\f]
3028and equation (21)
3029requires their variations to vanish,
3030\f[
3031 \delta X_{ij} = 0 \mbox{ \ \ \ if node $j$ is located on $A_{displ}.$}
3032\f]
3033The variations of all other nodal positions are arbitrary (and independent
3034of each other), therefore the terms in the curly brackets in
3035(36) need to vanish individually. This provides one
3036(discrete) equation for each unknown \f$X_{km}\f$
3037 \f[ f_{km} =
3038 \int \left[ \sigma^{ij} \
3039 \left(\sum_{k=1}^N X_{kl} \frac{\partial \psi_l}{\partial \xi^i} \right)
3040 \frac{\partial \psi_m}{\partial \xi^j} \
3041 - \left( f_k -
3042 \Lambda^2 \left(\sum_{k=1}^N
3043 \frac{\partial^2 X_{kl}}{\partial t^2} \psi_l \right) \right)
3044 \psi_m \right] \ \Gamma \ dv \
3045 - \oint_{A_{tract}} t_k \ \psi_m \ dA
3046 =0. \ \ \ \ \ \ \ \ \ (37)
3047 \f]
3048These equations can again be assembled in an element-by-element
3049fashion.

3051\subsubsection solid_implementation Implementation
3052We will now discuss how the discrete equations (37) are
3053implemented in \c oomph-lib. To facilitate the analysis
3054of multi-physics problems, we introduce generalisations of the \c Node,
3055\c FiniteElement and \c Mesh classes which provide
3056separate storage (and access functions) for all solid mechanics
3057Data. The resulting \c SolidFiniteElements can
3058be used as stand-alone elements for the simulation of
3059pure solid mechanics problems. More importantly, however, the design
3060makes it easy to employ multiple inheritance to create
3061more complex elements that solve the equations of solid mechanics
3062together with
3063any other field equations. For instance, if we combine
3064a \c FiniteElement that solves the unsteady heat equation
3065with an \c SolidFiniteElement that describes the elastic
3066deformations, we obtain an element that can be used to simulate
3067unsteady heat conduction in an elastic body that performs
3068large-amplitude oscillations, say.

3070<CENTER> <B>The SolidNode class:</B></CENTER>
3071The class \c SolidNode is derived from \c Node and
3072implements the additional functionality required for solid
3073mechanics problems. The key new feature is that each Node
3074needs to store its (fixed) Lagrangian coordinate \f$ \Xi_{ij}\f$,
3075while its Eulerian position \f$ X_{ij}\f$ needs to be regarded
3076as an unknown. This requires the following changes to
3077member functions of the \c Node class:
3078
3079- The function \c Node::x(...) returns the
3080 Eulerian coordinates of the \c Node. Internally, this function
3081 accesses the nodal coordinates via pointers to double
3082 precision numbers. This data structure is adequate for
3083 problems in which the nodal positions are either known a priori
3084 or determined by an algebraic node update function. However, the
3085 scheme does not allow us to regard the nodal positions as
3086 unknowns -- as required by our expansion
3087 (35). In \c SolidNodes
3088 the nodal positions are therefore stored as \c Data (whose
3089 values can either be unknowns or pinned values, and which can have a time
3090 history). We overload the \c Node::x(...) function by
3091 \code
3092 SolidNode::x(...)
3093 \endcode
3094 which returns the (now variable) Eulerian nodal position of the node.
3095 The function
3096 \code
3097 SolidNode::position_eqn_number(...)
3098 \endcode
3099 gives access to the global equation number for each
3100 nodal coordinate and, following our usual convention, the function
3101 returns -1 if a coordinate is pinned.
3102- We introduce a new member function
3103 \code
3104 SolidNode::xi(...)
3105 \endcode
3106 which returns the (fixed) Lagrangian coordinates of the node.
3107- Both functions have the usual extensions to generalised
3108 coordinates
3109 \code
3110 SolidNode::x_gen(...)
3111 \endcode
3112 and
3113 \code
3114 SolidNode::xi_gen(...)
3115 \endcode
3116 which are required for Hermite elements and any other elements
3117 that use generalised nodal coordinates to interpolate the
3118 element's geometry.
3119- Similar to the \c Data member function \c Data::pin(...) which can
3120 be used to pin specific nodal values, we provide the function
3121 \code
3122 SolidNode::position_pin(...)
3123 \endcode
3124 which allows pinning of selected nodal coordinates.
3125- Dynamic problems require the evaluation of time-derivatives
3126 of the nodal positions, such as \f$ \partial^2 X_{ij}/\partial t^2\f$,
3127 see (37). These time-derivatives are evaluated by the
3128 \c TimeStepper of the positional \c Data. By default,
3129 we use the same \c TimeStepper for the nodal \c Data and for the nodal
3130 positions. In multi-physics problems this might not be
3131 appropriate. Consider, for instance, solving an unsteady
3132 heat equation in a dynamically deforming, elastic body. In this problem
3133 the 2nd time-derivatives of the nodal position might be evaluated
3134 by a \c Newmark scheme, acting on the history values of the nodal
3135 positions, whereas the time-derivatives of the temperature
3136 might be determined by a \c BDF scheme, operating on the
3137 history values of the nodal \c Data.
3138 In such cases, the default assignment for the two timesteppers
3139 can be overwritten with the access functions
3140 \code
3141 SolidNode::position_time_stepper_pt()
3142 \endcode
3143 and
3144 \code
3145 SolidNode::time_stepper_pt()
3146 \endcode
3147 where the latter is inherited from \c Data::time_stepper_pt().
3148- Our implementation is based on the displacement form of the
3149 principle of virtual displacements in which the position vector
3150 \f$ {\bf R}(\xi^i) \f$ in the deformed configuration is regarded
3151 as the unknown vector field. Equation (35)
3152 defines the representation of this vector
3153 field within each finite element in terms of the nodal coordinates.
3154 Some constitutive equations require the representation
3155 of additional (non-positional) variables.
3156 For instance, for incompressible (or nearly incompressible) materials,
3157 the stress \f$ \sigma^{ij} \f$ contains a contribution from the
3158 (scalar) pressure field \f$ p \f$; see the discussion of the
3159 constitutive equations above. If we choose a continuous
3160 representation for the pressure in which its value is
3161 interpolated between nodal values (as in Taylor-Hood-type
3162 elements), the nodal pressure values could in principle be stored
3163 in the "normal" nodal Data. However, to retain a clear
3164 distinction between
3165 solid-mechanics and non-solid-mechanics quantities we store
3166 such "additional nodal solid values" in their own \c Data object
3167 to which we provide access with the function
3168 \code
3169 SolidNode::additional_solid_data_pt()
3170 \endcode
3171 As with all other nodal quantities, the number of additional
3172 solid data values, needs to be specified as an argument to the
3173 \c SolidNode constructor when the \c SolidNode is created.
3174 The access function
3175 \code
3176 SolidNode::nadditional_solid_value()
3177 \endcode
3178 returns the number of additional solid values that are stored
3179 at an \c SolidNode; the function
3180 \code
3181 SolidNode::additional_solid_eqn_number(...)
3182 \endcode
3183 returns the global equation number of the SolidNode's
3184 additional solid values, and there are further member functions
3185 that allow pinning/unpinning of these values, etc.
3186- Finally, \c SolidNodes overload the function
3187 \c Data::assign_eqn_numbers()
3188 with
3189 \code
3190 SolidNode::assign_eqn_numbers()
3191 \endcode
3192 which creates global equation numbers for all (non-pinned)
3193 positional values and any additional (nodal) solid values, and
3194 then deals with the "normal" nodal Data by calling

3421\subsubsection Solid_IC Timestepping and the generation of initial conditions
3422In time-dependent problems, the boundary value problem defined by
3423the variational principle (28) needs to be
3424augmented by suitable initial conditions which specify the
3425state of the system at time \f$ t=t_0. \f$ The initial
3426conditions specify the initial shape of the solid body,
3427\f[
3428{\bf R}(\xi^i,t=t_0) = {\bf R}^{(IC)}(\xi^i),
3429\ \ \ \ \ \ \ \ \ \ \ \ \ (38)
3430\f]
3431and its initial velocity,
3432\f[
3433\left. \frac{\partial {\bf R}(\xi^i,t)}{\partial t} \right|_{t=t_0} =
3434{\bf V}^{(IC)}(\xi^i),
3435\ \ \ \ \ \ \ \ \ \ \ \ (39)
3436\f]
3437where \f$ {\bf R}^{(IC)}(\xi^i) \f$ and \f$ {\bf V}^{(IC)}(\xi^i) \f$
3438are given. The accelerations \f$ \partial^2 {\bf R}/\partial t^2
3439\f$ at \f$t=t_0\f$ follow from the solution of
3440(28) and can therefore not be enforced,
3441unless we wish to initialise the time-stepping procedure with a known
3442exact solution \f$ {\bf R}^{(exact)}(\xi^i,t) \f$. (Only!) in this case
3443are we allowed to assign an initial value for the acceleration via
3444\f[
3445\left. \frac{\partial^2 {\bf R}(\xi^i,t)}{\partial t^2} \right|_{t=t_0} =
3446{\bf A}^{(IC)}(\xi^i) \equiv
3447\left. \frac{\partial^2 {\bf R}^{(exact)}(\xi^i,t)}{\partial t^2}
3448\right|_{t=t_0}.
3449\ \ \ \ \ \ \ \ \ \ \ \ (40)
3450\f]
3451We will assume that time-stepping is performed with the \c Newmark
3452method which is our default timestepper for hyperbolic
3453problems. In this case the time-derivatives of the nodal positions in
3454(28) are replaced by an approximation
3455which involves the current and three "history values" of the nodal
3456positions. To start the time-integration, we need to assign
3457suitable values to these quantities to ensure that the intial
3458state of the system is represented correctly.
3459
3460To assign the intial values for the nodal positions, we
3461(temporarily) remove all boundary conditions for the nodal
3462positions and determine their initial values by solving equation
3463(38) in its weak form,
3464\f[
3465f_{il}^{(0)} = \int \left( \sum_{j=1}^N X_{ij}^{(0)} \psi_j(\xi^k)
3466- R^{(IC)}_i(\xi^k)
3467\right) \psi_l(\xi^k) \ dv = 0
3468\mbox{ \ \ \ \ for \ \ \ $l=1,...,N; \ \ i=1,..,3$}
3469\ \ \ \ \ \ \ \ \ \ \ \ \ (41)
3470\f]
3471where \f$ R^{(IC)}_i \f$ is the \f$i\f$-th component of
3472\f${\bf R}^{(IC)}.\f$ Equation (41) provides
3473\f$ 3 \times N\f$ equations for the \f$ 3 \times N \f$ components
3474of the initial nodal positions, \f$ X_{ij}^{(0)} \f$ (where
3475\f$i=1,...3; \ j=1,...,N)\f$. To determine the initial nodal
3476velocities, we repeat the same procedure with the prescribed
3477velocities and solve
3478\f[
3479f_{il}^{(1)} = \int \left( \sum_{j=1}^N X_{ij}^{(1)} \psi_j(\xi^k)
3480- V^{(IC)}_i(\xi^k)
3481\right) \psi_l(\xi^k) \ dv = 0
3482\mbox{ \ \ \ \ for \ \ \ $l=1,...,N; \ \ i=1,..,3$}
3483\ \ \ \ \ \ \ \ \ \ \ \ \ (42)
3484\f]
3485for the initial nodal velocities, \f$ X_{ij}^{(1)} \f$ (where
3486\f$i=1,...3; \ j=1,...,N)\f$. Finally, assuminig that we have
3487an exact solution for the accelerations, we solve
3488\f[
3489f_{il}^{(2)} = \int \left( \sum_{j=1}^N X_{ij}^{(2)} \psi_j(\xi^k)
3490- A^{(IC)}_i(\xi^k)
3491\right) \psi_l(\xi^k) \ dv = 0
3492\mbox{ \ \ \ \ for \ \ \ $l=1,...,N; \ \ i=1,..,3$}
3493\ \ \ \ \ \ \ \ \ \ \ \ \ (43)
3494\f]
3495for the initial nodal accelerations, \f$ X_{ij}^{(2)} \f$ (where
3496\f$i=1,...3; \ j=1,...,N)\f$. Having determined the
3497nodal positions and their first and second time-derivatives
3498at \f$ t=t_0 \f$, we can use the functions
3499\c Newmark::assign_initial_data_values_stage1(...) and
3500\c Newmark::assign_initial_data_values_stage2(...) to compute the
3501positional history values which ensure that the Newmark
3502approximations for the
3503initial velocity and acceleration are correct.
3504This procedure is fully implemented in the function
3505\code
3506SolidMesh::Solid_IC_problem.set_newmark_initial_condition_directly(...)
3507\endcode
3508whose arguments are:
3509- The pointer to the problem being solved. This is needed because
3510 the solution of equations (41) -
3511 (43) requires a temporary change to the
3512 boundary conditions and to the equation numbering scheme. Once
3513 the history values are assigned, the original
3514 boundary conditions are restored and the original equation numbers
3515 are re-generated by executing
3516 the Problem's \c Problem::assign_eqn_numbers().
3517- A pointer to the \c SolidMesh on which the initial conditions
3518 are assigned.
3519- A pointer to the \c TimeStepper (which has to be of the \c Newmark
3520 family).
3521- A pointer to the "Elasic initial condition" object (discussed below).
3522- The initial timestep.

3612If we do not know an exact solution to our problem (and in most
3613cases we obviously won't...), we can only use the procedure described above
3614to determine the initial nodal positions and velocities.
3615We then need to solve the equations (37) for the
3616remaining "history value". Since the equations (37)
3617are linear in the accelerations, this is a linear problem
3618whose Jacobian matrix is proportional to the mass matrix
3619\f[
3620M_{ij} = \int {\cal M} \ \psi_i \ \psi_j \ dv.
3621\ \ \ \ \ \ \ (44)
3622\f]
3623The procedure which determines the initial "history values"
3624from the given initial positions and velocities while ensuring
3625consistency with the governing equation at \f$ t = t_0\f$
3626is implemented in
3627\c SolidMesh::Solid_IC_problem.set_newmark_initial_condition_consistently(...)
3628which takes the same arguments as the function that assigns
3629the acceleration directly, but also requires a function pointer
3630to the "multiplier" \f$ {\cal M} \f$. If there
3631is no growth, the multiplier is given by the timescale ratio
3632\f$\Lambda^2\f$; if the body is subjected to uniform isotropic
3633growth, \f$\Gamma \ne 1\f$, the multiplier is equal to
3634\f$\Gamma \Lambda^2\f$.
3635If the wrong multiplier is specified (or if it is omitted, in
3636which the default value of 1.0 is used) the residuals (37)
3637will be nonzero (or at least larger than the tolerance specified in \c
3638SolidICProblem). In this case a warning is issued and the
3639code execution terminates. This behaviour can be suppressed
3640by increasing the tolerance suitably, but you do this at your own risk!
