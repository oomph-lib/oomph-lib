19 \section test A test problem: Time-harmonic oscillations of an elastic annulus reinforced by a T-rib
20 
21 The sketch below shows the problem setup: A 2D elastic annulus
22 which is reinforced with two T-ribs is
23 subjected to a time-periodic pressure load of magnitude
24 \f[
25 {\bf t} = P ( \exp(\alpha(\varphi-\pi/4)^2) + \exp(\alpha(\varphi-3\pi/4)^2) )
26 \f]
27 (where \f$ \varphi \f$ is the polar angle) along its outer edge.
28 The parameter \f$ \alpha \f$ controls the "sharpness" of the
29 pressure load. For \f$ \alpha=0 \f$ we obtain a uniform, axisymmetric load;
30 the sketch below shows the pressure distribution (red vectors
31 indicating the traction) for \f$ \alpha = 1. \f$
32 
33 \image html setup.gif "Sketch of the problem setup. "
34 \image latex setup.eps "Sketch of the problem setup. " width=0.8\textwidth
35 
36 The structure is symmetric and we only discretise the right half
37 (\f$ x_1 > 0 \f$) of the domain and apply symmetry conditions
38 (zero horizontal displacement) on the \f$ x_2-\f$ axis.

45 \section results Results
46 The figure below shows an animation of the structure's time-harmonic
47 oscillation. The blue shaded region shows the shape of the
48 oscillating structure while the pink region shows its initial configuration.
49 The left half of the plot is used to show the (mirror image of
50 the) adaptive unstructured mesh on which the solution was computed:
51 
52 \image html all.gif "Animation of the time-harmonic deformation. "
53 \image latex all.eps "Animation of the time-harmonic deformation. " width=0.8\textwidth
54 
55 This looks very pretty and shows that we can compute in non-trivial
56 geometries but should you believe the results? Here's an
57 attempt to convince you: If we make the rib much softer than the
58 annulus, the rib will not offer much structural resistance
59 and the annular region will deform as if the rib was not present.
60 If we then set \f$ \alpha = 0 \f$ we apply an axisymmetric forcing
61 onto the structure and would expect the resulting displacement
62 field (at least in the annular region) to be axisymmetric.
63 For this case it is easy to find an analytical solution to the
64 problem. The next two figures show a comparison between
65 the analytical (green spheres) and computed solutions (shaded)
66 for the real part of the horizontal and vertical displacements,
67 respectively.

82 \section num The numerical solution
83 The driver code for this problem is very similar to the one discussed in
84 <a href=../../elastic_annulus/html/index.html>another tutorial</a>.
85 Running \c sdiff on the two driver codes
86 <CENTER>
87 <A HREF="../../../../demo_drivers/time_harmonic_linear_elasticity/elastic_annulus/time_harmonic_elastic_annulus.cc">
88 demo_drivers/time_harmonic_linear_elasticity/elastic_annulus/time_harmonic_elastic_annulus.cc
89 </A>
90 </CENTER>
91 and
92 <CENTER>
93 <A HREF="../../../../demo_drivers/time_harmonic_linear_elasticity/elastic_annulus/unstructured_time_harmonic_elastic_annulus.cc">
94 demo_drivers/time_harmonic_linear_elasticity/elastic_annulus/unstructured_time_harmonic_elastic_annulus.cc
95 </A>
96 </CENTER>
97 shows you the differences, the most important of which are:
98 - The provision of multiple elasticity tensors and frequency
99  parameters for the two different regions (the rib and the
100  annulus).
101  \n\n
102 - The provision of a helper function \c complete_problem_setup()
103  which rebuilds the elements (by passing the problem parameters
104  to the elements) following the unstructured mesh adaptation.
105  (The need/rationale for such a function is discussed in
106  <a href="../../../meshes/mesh_from_inline_triangle/html/index.html">
107  another tutorial.</a>)
108  \n\n
109 - The mesh generation -- the specification of the curvilinear
110  boundaries and the geometry of the rib is somewhat tedious.
111  We refer to
112  <a href="../../../meshes/mesh_from_inline_triangle_internal_boundaries/html/index.html">
113  another tutorial</a> for a discussion of how to define the
114  internal mesh boundary that separates the two regions
115  (the rib and the annular region) so that we can assign
116  different material properties to them.
117  \n\n
118 .
119 All of this is reasonably straightforward and provides a
120 powerful code that automatically adapts the mesh while
121 respecting the curvilinear boundaries of the domain.
122 Have a look through the driver code and play with it.
