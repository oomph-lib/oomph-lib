169 /// Z2-error-estimator: Elements that can
170 /// be used with Z2 error estimation should be derived from
171 /// the base class ElementWithZ2ErrorEstimator and implement its
172 /// pure virtual member functions to provide the following functionality:
173 /// - pointer-based access to the vertex nodes in the element
174 /// (this is required to facilitate setup of element patches).
175 /// - the computation of a suitable "flux vector" which represents
176 /// a quantity whose FE representation is discontinuous across
177 /// element boundaries but would become continuous under infinite
178 /// mesh refinement.
179 ///
180 /// As an example consider the finite element solution of the Laplace problem,
181 /// \f$ \partial^2 u/\partial x_i^2 = 0 \f$. If we approximate the
182 /// unknown \f$ u \f$ on a finite element mesh with \f$ N \f$ nodes as
183 /// \f[
184 /// u^{[FE]}(x_k) = \sum_{j=1}^{N} U_j \ \psi_j(x_k),
185 /// \f]
186 /// where the \f$ \psi_j(x_k) \f$ are the (global) \f$ C^0 \f$ basis
187 /// functions, the finite-element representation of the flux,
188 /// \f$ f_i = \partial u/\partial x_i \f$,
189 /// \f[
190 /// f_i^{[FE]} = \sum_{j=1}^{N} U_j \ \frac{\partial \psi_j}{\partial x_i}
191 /// \f]
192 /// is discontinuous between elements but the magnitude of the jump
193 /// decreases under mesh refinement. We denote the number
194 /// of flux terms by \f$N_{flux}\f$, so for a 2D (3D) Laplace problem,
195 /// \f$N_{flux}=2 \ (3).\f$
196 ///
197 /// The idea behind Z2 error estimation is to compute an
198 /// approximation for the flux that is more accurate than the value
199 /// \f$ f_i^{[FE]} \f$ obtained directly from the finite element
200 /// solution. We refer to the improved approximation for the flux
201 /// as the "recovered flux" and denote it by \f$ f_i^{[rec]} \f$. Since
202 /// \f$ f_i^{[rec]} \f$ is more accurate than \f$ f_i^{[FE]} \f$, the
203 /// difference between the two provides a measure of the error.
204 /// In Z2 error estimation, the "recovered flux" is determined by
205 /// projecting the discontinuous, FE-based flux \f$ f_i^{[FE]} \f$
206 /// onto a set of continuous basis functions, known as the "recovery shape
207 /// functions". In principle, one could use the
208 /// finite element shape functions \f$ \psi_j(x_k) \f$ as the
209 /// recovery shape functions but then the determination of the
210 /// recovered flux would involve the solution of a linear system
211 /// that is as big as the original problem. To avoid this, the projection
212 /// is performed over small patches of elements within which
213 /// low-order polynomials (defined in terms of the global, Eulerian
214 /// coordinates) are used as the recovery shape functions.
215 ///
216 /// Z2 error estimation is known to be "optimal" for many self-adjoint
217 /// problems. See, e.g., Zienkiewicz & Zhu's original papers
218 /// "The superconvergent patch recovery and a posteriori error estimates."
219 /// International Journal for Numerical Methods in Engineering \b 33 (1992)
220 /// Part I: 1331-1364; Part II: 1365-1382.
221 /// In non-self adjoint problems, the error estimator only
222 /// analyses the "self-adjoint part" of the differential operator.
223 /// However, in many cases, this still produces good error indicators
224 /// since the error estimator detects under-resolved, sharp gradients
225 /// in the solution.
226 ///
227 /// Z2 error estimation works as follows:
228 /// -# We combine the elements in the mesh into a number of "patches",
229 /// which consist of all elements that share a common vertex node.
230 /// Most elements will therefore be members of multiple patches.
231 /// -# Within each patch \f$p\f$, we expand the "recovered flux" as
232 /// \f[
233 /// f^{[rec](p)}_i(x_k) = \sum_{j=1}^{N_{rec}}
234 /// F^{(p)}_{ij} \ \psi^{[rec]}_j(x_k) \mbox{ \ \ \ for $i=1,...,N_{flux}$,}
235 /// \f]
236 /// where the functions \f$ \psi^{[rec]}_j(x_k)\f$ are the recovery
237 /// shape functions, which are functions of the global, Eulerian
238 /// coordinates. Typically, these are chosen to be low-order polynomials.
239 /// For instance, in 2D, a bilinear representation of
240 /// \f$ f^{(p)}_i(x_0,x_1) \f$
241 /// involves the \f$N_{rec}=3\f$ recovery shape functions
242 /// \f$ \psi^{[rec]}_0(x_0,x_1)=1, \ \psi^{[rec]}_1(x_0,x_1)=x_0 \f$
243 /// and \f$ \psi^{[rec]}_2(x_0,x_1)=x_1\f$.
244 ///
245 /// We determine the coefficients \f$ F^{(p)}_{ij} \f$ by enforcing
246 /// \f$ f^{(p)}_i(x_k) = f^{[FE]}_i(x_k)\f$ in its weak form:
247 /// \f[
248 /// \int_{\mbox{Patch $p$}} \left(
249 /// f^{[FE]}_i(x_k) - \sum_{j=1}^{N_{rec}}
250 /// F^{(p)}_{ij} \ \psi^{[rec]}_j(x_k) \right) \psi^{[rec]}_l(x_k)\ dv = 0
251 /// \mbox{ \ \ \ \ for $l=1,...,N_{rec}$ and $i=1,...,N_{flux}$}.
252 /// \f]
253 /// Once the \f$ F^{(p)}_{ij} \f$ are determined in a given patch,
254 /// we determine the values of the recovered flux at
255 /// all nodes that are part of the patch. We denote the
256 /// value of the recovered flux at node \f$ k \f$ by
257 /// \f$ {\cal F}^{(p)}_{ik}\f$.
258 ///
259 /// We repeat this procedure for every patch. For nodes that are part of
260 /// multiple patches, the procedure
261 /// will provide multiple, slightly different nodal values for the recovered
262 /// flux. We average these values via
263 /// \f[
264 /// {\cal F}_{ij} = \frac{1}{N_p(j)}
265 /// \sum_{\mbox{Node $j \in $ patch $p$}}
266 /// {\cal F}^{(p)}_{ij},
267 /// \f]
268 /// where \f$N_p(j)\f$ denotes the number of patches that node \f$ j\f$ is a
269 /// member of.
270 /// This allows us to obtain a globally-continuous, finite-element based
271 /// representation of the recovered flux as
272 /// \f[
273 /// f_i^{[rec]} = \sum_{j=1}^{N} {\cal F}_{ij}\ \psi_j,
274 /// \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ (1)
275 /// \f]
276 /// where the \f$ \psi_j \f$ are the (global) finite element
277 /// shape functions.
278 /// -# Since the recovered flux in (1), is based on nodal values, we can
279 /// evaluate it locally within each of the \f$ N_e\f$ elements in the mesh
280 /// to obtain a normalised elemental error estimate via
281 /// \f[
282 /// E_{e} = \sqrt{ \frac{
283 /// \int_{\mbox{$e$}} \left(
284 /// f_i^{[rec]} - f_i^{[FE]} \right)^2 dv}
285 /// {\sum_{e'=1}^{N_e}
286 /// \int_{\mbox{$e'$}} \left(
287 /// f_i^{[rec]} \right)^2 dv} } \mbox{\ \ \ for $e=1,...,N_e$.}
288 /// \f]
289 /// In this (default) form, mesh refinement, based on this error estimate
290 /// will lead to an equidistribution of the error across all elements.
291 /// Usually, this is the desired behaviour. However, there are
292 /// cases in which the solution evolves towards a state in which
293 /// the flux tends to zero while the solution itself becomes so simple
294 /// that it can be represented exactly on any finite element mesh
295 /// (e.g. in spin-up problems in which the fluid motion approaches
296 /// a rigid body rotation). In that case the mesh adaptation tries
297 /// to equidistribute any small roundoff errors in the solution,
298 /// causing strong, spatially uniform mesh refinement throughout
299 /// the domain, even though the solution is already fully converged.
300 /// For such cases, it is possible to replace the denominator in the
301 /// above expression by a prescribed reference flux, which may be
302 /// specified via the access function
303 /// \code
304 /// reference_flux_norm()
305 /// \endcode
306 ///
307 ///
