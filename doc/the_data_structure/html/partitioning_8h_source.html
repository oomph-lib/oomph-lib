154  /// \short Use METIS to assign each element in an already-distributed mesh to
155  /// a domain. On return, element_domain_on_this_proc[e] contains the number
156  /// of the domain [0,1,...,ndomain-1] to which non-halo element e on THE
157  /// CURRENT PROCESSOR ONLY has been assigned. The order of the non-halo
158  /// elements is the same as in the Problem's mesh, with the halo
159  /// elements being skipped.
160  ///
161  /// Objective:
162  /// - objective=0: minimise edgecut.
163  /// - objective=1: minimise total communications volume.
164  /// .
165  /// The partioning is based on the dof graph of the complete mesh by
166  /// taking into
167  /// account which global equation numbers are affected by each element and
168  /// connecting elements which affect the same global equation number.
169  /// Partitioning is done such that all elements associated with the
170  /// same tree root move together. Non-refineable elements are
171  /// treated as their own root elements. If the optional boolean
172  /// flag is set to true (it defaults to false) each processor
173  /// assigns a dumb-but-repeatable equidistribution of its non-halo
174  /// elements over the domains and outputs the input that would have
175  /// gone into METIS in the file metis_input_for_validation.dat
176  extern void partition_distributed_mesh
177  (Problem* problem_pt,const unsigned& objective,
