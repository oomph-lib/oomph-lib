63 /// QuadTree class: Recursively defined, generalised quadtree.
64 ///
65 /// A QuadTree has:
66 /// - a pointer to the object (of type RefineableQElement<2>) that it
67 /// represents in a mesh refinement context.
68 /// - Vector of pointers to its four (SW/SE/NW/NE) sons (which are
69 /// themselves quadtrees).
70 /// If the Vector of pointers to the sons has zero length,
71 /// the QuadTree is a "leaf node" in the overall quadtree.
72 /// - a pointer to its father. If this pointer is NULL, the QuadTree is the
73 /// the root node of the overall quadtree.
74 /// This data is stored in the Tree base class.
75 ///
76 /// The tree can also be part of a forest. If that is the case, the root
77 /// will have pointers to the roots of neighbouring quadtrees.
78 ///
79 /// The objects contained in the quadtree are assumed to be
80 /// (topologically) rectangular elements whose geometry is
81 /// parametrised by local coordinates \f$ {\bf s} \in [-1,1]^2 \f$.
82 ///
83 /// The tree can be traversed and actions performed
84 /// at all its "nodes" or only at the leaf "nodes" ("nodes" without sons).
85 ///
86 /// Finally, the leaf "nodes" can be split depending on
87 /// a criteria defined by the object.
88 ///
89 /// Note that QuadTrees are only generated by splitting existing
90 /// QuadTrees. Therefore, the constructors are protected. The
91 /// only QuadTree that "Joe User" can create is
92 /// the (derived) class QuadTreeRoot.
93 //=================================================================

123  /// \short Return pointer to greater or equal-sized edge neighbour
124  /// in specified \c direction; also provide info regarding the relative
125  /// size and orientation of neighbour:
126  /// - The vector translate_s turns the index of the local coordinate
127  /// in the present quadtree into that of the neighbour. If there are no
128  /// rotations then translate_s[i] = i, but if, for example, the neighbour's
129  /// eastern face is adjacent to our northern face translate_s[0] = 1
130  /// and translate_s[1] = 0. Of course, this could be deduced after the
131  /// fact, but it's easier to do it here.
132  /// - In the present quadtree, the lower left (south west) vertex is
133  /// located at local coordinates (-1,-1). This point is located
134  /// at the local coordinates (\c s_lo[0], \c s_lo[1]) in the neighbouring
135  /// quadtree.
136  /// - ditto with s_hi: In the present quadtree, the upper right (north east)
137  /// vertex is located at local coordinates (1,1). This point is located
138  /// at the local coordinates (\c s_hi[0], \c s_hi[1]) in the neighbouring
139  /// quadtree.
140  /// - We're looking for a neighbour in the specified \c direction. When
141  /// viewed from the neighbouring quadtree, the edge that separates
142  /// the present quadtree from its neighbour is the neighbour's \c edge
143  /// edge. If there's no rotation between the two quadtrees, this is a
144  /// simple reflection: For instance, if we're looking
145  /// for a neighhbour in the \c N [orthern] \c direction, \c edge will
146  /// be \c S [outh]
147  /// - \c diff_level <= 0 indicates the difference in refinement levels between
148  /// the two neighbours. If \c diff_level==0, the neighbour has the
149  /// same size as the current quadtree.
150  /// - \c in_neighbouring_tree indicates whether the neighbour is actually
151  /// in another tree in the forest. The introduction of this flag
152  /// was necessitated by periodic problems where a TreeRoot can be its
153  /// own neighbour.

350  /// \short Return north equivalent of the neighbours in specified
351  /// direction: When viewed from the current quadtree's \c neighbour neighbour,
352  /// our northern direction is the neighbour's north_equivalent(neighbour)
353  /// direction. If there's no rotation, this map contains the identify
354  /// so that, e.g. \c north_equivalent(W)=N (read as: "in my Western

366  std::ostringstream error_message;

368  << Direct_string[neighbour] << std::endl;
369 
370  throw OomphLibError(error_message.str(),
371  OOMPH_CURRENT_FUNCTION,
372  OOMPH_EXCEPTION_LOCATION);
373  }
374 #endif
375  return North_equivalent[neighbour];
376  }

379  /// \short If quadtree_root_pt is a neighbour, return the direction
