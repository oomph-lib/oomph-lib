66 /// \short A class that represents a collection of data;
67 /// each Data object may contain many different individual values,
68 /// as would be natural in non-scalar problems.
69 /// Data provides storage for auxiliary `history' values that are
70 /// used by TimeStepper objects to calculate the time derivatives of the
71 /// stored data and also stores a pointer to the appropriate TimeStepper
72 /// object.
73 /// In addition, an associated (global) equation number is
74 /// stored for each value.
75 ///
76 /// The Data class permits copies of the stored data values and equation
77 /// numbers into another Data object using the copy() function.
78 /// Shallow (pointer based) copies of
79 /// the values can be obtained by using specific derived classes. In such
80 /// cases pointers to the objects that contain the pointer-based copies
81 /// should be stored in the original Data class
82 /// (in the array Copy_of_data_pt)
83 /// so that resize and destruction operations can be performed safely.

795 /// \short Nodes are derived from Data, but, in addition, have a
796 /// definite (Eulerian) position in a space of a given dimension.
797 ///
798 /// The nodal coordinates are used in the elements' mapping
799 /// between local and global coordinates and in the simplest
800 /// case (stationary nodes in Lagrange-type elements) this mapping
801 /// is given by
802 /// \f[ x_i = \sum_{j=1}^{N_{node}} X_{ij} \psi_{j}(s_k) \f]
803 /// so we need only access to the nodal coordinates
804 /// \f$ X_{ij}\ (i=1..DIM) \f$ of all nodes \f$ j \f$ : provided
805 /// by the Node member function
806 /// \code Node::x(i) \endcode
807 ///
808 /// If the nodal positions are time-dependent, the mapping becomes
809 /// \f[ x_i(t) = \sum_{j=1}^{N_{node}} X_{ij}(t) \ \psi_{j}(s_k). \f]
810 /// Within the computation (where time is only evaluated at discrete
811 /// levels) this becomes
812 /// \f[ x_{ti} = \sum_{j=1}^{N_{node}} X_{ijt} \ \psi_{j}(s_k). \f]
813 /// and we need access to the nodal coordinates \f$ X_{ijt} \ (i=1..DIM) \f$ of
814 /// all nodes \f$ j \f$ at the present (t=0) and previous (t>0) timesteps:
815 /// provided by the Node member function
816 /// \code Node::x(t,i) \endcode
817 /// \b Note: The interpretation of the history values is slightly more
818 /// subtle than that. Depending on the positional TimeStepper
819 /// used, only a limited number of the positional history values accessed
820 /// \c Node::x(t,i) represent previous nodal positions; the others
821 /// are generalised history values that the TimeStepper uses to
822 /// determine approximations for the time-derivatives of the
823 /// nodal positions.
824 ///
825 /// Finally, some elements employ mappings
826 /// that involve additional, generalised coordinates. For instance,
827 /// in Hermite elements the mapping between local and global coordinates
828 /// is based on an independent interpolation for the global coordinates
829 /// and their derivative w.r.t. to the local coordinates. In such
830 /// elements, the mapping becomes
831 /// \f[ x_i = \sum_{j=1}^{N_{node}} \sum_{k=1}^{N_{type}} X_{ijk}
832 /// \psi_{jk}(s_k) \f]
833 /// where \f$ N_{type} \f$ is the number of the different types of generalised
834 /// coordinates involved in the mapping. For instance, in 1D Hermite elements
835 /// \f$ N_{type}=2 \f$ and k=0 corresponds to the global coordinates while
836 /// k=1 corresponds to the
837 /// derivative of the global coordinates w.r.t. to the local coordinate.
838 /// In such cases we need access to the generalised nodal coordinates
839 /// \f$ X_{ijk} \ (i=1..DIM, \ k=1..N_{type}) \f$ of all nodes \f$ j \f$.
840 /// Access is provided by the Node member function
841 /// \code Node::x_gen(k,i) \endcode
842 /// and the corresponding time-dependent version
843 /// \code Node::x_gen(t,k,i) \endcode
