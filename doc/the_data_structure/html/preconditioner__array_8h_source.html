79 /// \short PreconditionerArray -
80 /// NOTE - first implementation, a number of assumptions / simplifications
81 /// were made:
82 /// 1. Only works with CRDoubleMatrices
83 /// 2. The number of processors must be greater than the number of
84 /// preconditioners
85 /// 3. Currently only very crude load balancing - each preconditioner will
86 /// be setup and applied with the same number of processors (or as near to
87 /// as possible to the same number of processors)
88 /// 4. This class will, at the appropriate time, delete the all the
89 /// Preconditioners passed setup_preconditioners(...)
90 /// 5. (but) Deletion of matrices passed to setup_preconditioners(...) is NOT
91 /// performed by this class
92 /// 6. It is assumed that preconditioners do not require access to matrix
93 /// once setup(...) is called
94 /// 7. The matrix on the subset of processors will be the same type
95 /// (distributed or global) as the matrix passed to
96 /// setup_preconditioners(...)
97 /// 8. If the matrix is a distributed matrix - it will be assembled with
98 /// a uniform distribution on the subset of processors.
99 //=============================================================================
