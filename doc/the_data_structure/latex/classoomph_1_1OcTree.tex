\doxylink{classoomph_1_1OcTree}{Oc\+Tree} class\+: Recursively defined, generalised octree. 

An \doxylink{classoomph_1_1OcTree}{Oc\+Tree} has\+:
\begin{DoxyItemize}
\item a pointer to the object (of type \doxylink{classoomph_1_1RefineableQElement_3_013_01_4}{Refineable\+QElement$<$3$>$}) it represents
\item \doxylink{classoomph_1_1Vector}{Vector} of pointers to its eight (LDB,RDB,...,RUF) sons (which are octrees themselves). If the \doxylink{classoomph_1_1Vector}{Vector} of pointers to the sons has zero length, the \doxylink{classoomph_1_1OcTree}{Oc\+Tree} is a "{}leaf node"{} in the overall octree.
\item a pointer to its father. If this pointer is NULL, the \doxylink{classoomph_1_1OcTree}{Oc\+Tree} is the the root node of the overall octree. This data is stored in the \doxylink{classoomph_1_1Tree}{Tree} base class.
\end{DoxyItemize}

The tree can also be part of a forest. If that is the case, the root will have pointers to the roots of neighbouring octrees.

The objects contained in the octree are assumed to be (topologically) cubic elements whose geometry is parametrised by local coordinates $ {\bf s} \in [-1,1]^3 $.

The tree can be traversed while actions are being performed at all of its "{}nodes"{} or only at the leaf "{}nodes"{}.

Finally, the leaf "{}nodes"{} can be split depending on criteria defined by the object.

Note that Oc\+Trees are only generated by splitting existing Oc\+Trees. Therefore, the constructors are protected. The only \doxylink{classoomph_1_1OcTree}{Oc\+Tree} that "{}\+Joe User"{} can create is the (derived) class \doxylink{classoomph_1_1OcTreeRoot}{Oc\+Tree\+Root}.
