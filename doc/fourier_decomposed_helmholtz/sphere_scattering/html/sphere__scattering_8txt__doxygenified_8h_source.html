6 In this document we discuss the finite-element-based solution of the
7 Helmholtz equation in cylindrical polar coordinates, using a
8 Fourier-decomposition of the solution in the azimuthal direction.
9 This is useful for solving time-harmonic wave problems in 3D
10 axisymmetric domains, e.g. the scattering of acoustic sound field
11 from a sphere, the example we consider below.

32 \section theory Theory: The Fourier-decomposed Helmholtz equation
33 
34 The Helmholtz equation governs time-harmonic solutions of problems
35 governed by the linear wave equation
36 \f[
37 \nabla^2 U(x,y,z,t) = \frac{1}{c^2} \frac{\partial^2 U(x,y,z,t)}{\partial t^2},
38  \ \ \ \ \ \ \ \ \ \ \ \ (1)
39 \f]
40 where \f$ c \f$ is the wavespeed. Assuming that \f$ U(x,y,z,t) \f$
41 is time-harmonic, with frequency \f$ \omega \f$, we write the real
42 function \f$ U(x,y,z,t) \f$ as
43 \f[
44 U(x,y,z,t) = Re ({\tt U}(x,y,z) \ e^{-i \omega t})
45  \ \ \ \ \ \ \ \ \ \ \ \ (2)
46 \f]
47 where \f$ {\tt U}(x,y,z) \f$ is complex-valued. This transforms
48 (1) into the Helmholtz equation
49 \f[
50 \nabla^2 {\tt U}(x,y,z) + k^2 {\tt U} = 0
51  \ \ \ \ \ \ \ \ \ \ \ \ (3)
52 \f]
53 where
54 \f[
55 k = \frac{\omega}{c}
56  \ \ \ \ \ \ \ \ \ \ \ \ (4)
57 \f]
58 is the wave number. Like other elliptic PDEs the Helmholtz equation
59 admits Dirichlet, Neumann (flux) and Robin boundary conditions.
60 
61 If the equation is solved in an infinite domain (e.g. in scattering
62 problems) the solution must satisfy the so-called
63 <a href="http://en.wikipedia.org/wiki/Sommerfeld_radiation_condition">
64 Sommerfeld radiation condition</a> which in 3D has the form
65 \f[
66 \lim_{r\to \infty} r \left(\frac{\partial {\tt U}}{\partial r} - ik{\tt U}
67 \right) =0.
68 \f]
69 Mathematically, this conditions is required to ensure the uniqueness
70 of the solution (and hence the well-posedness of the problem).
71 In a physical context, such as a scattering problem, the condition
72 ensures that scattering of an incoming wave only produces outgoing not
73 incoming waves from infinity.
74 
75  These equations can be solved using \c oomph-lib's cartesian
76 Helmholtz elements, described in
77 <a href="../../../helmholtz/scattering/html/index.html">another
78 tutorial.</a> Here we consider an alternative approach in which we
79 solve the equations in cylindrical polar coordinates \f$ (r,\varphi,z)
80 \f$, related to the cartesian coordinates \f$ (x,y,z) \f$ via
81 \f[
82 x = r \cos(\varphi),
83 \f]
84 \f[
85 y = r \sin(\varphi),
86 \f]
87 \f[
88 z = z.
89 \f]
90 We then decompose the solution into its Fourier components by writing
91 \f[
92 {\tt U}(r,\varphi,z) = \sum_{N=-\infty}^{\infty} u_N(r,z) \exp({\rm i}
93 N \varphi).
94 \f]
95 Since the governing equations are linear we can compute each Fourier
96 component \f$ u_N(r,z) \f$ individually by solving
97 \f[
98 \nabla^2 {u_{N}}(r,z) + \left(k^2-\frac{N^2}{r^2}\right) u_N(r,z) = 0
99  \ \ \ \ \ \ \ \ \ \ \ \ (5)
100 \f]
101 while specifying the Fourier wavenumber \f$ N \f$ as a parameter.

106 \section discr Discretisation by finite elements
107 The discretisation of the Fourier-decomposed Helmholtz equation itself
108 only requires a trivial modification of its
109 <a href="../../../helmholtz/scattering/html/index.html">cartesian
110 counterpart</a>. Since most practical applications
111 of the Helmholtz equation involve complex-valued solutions, we provide
112 separate storage for the real and imaginary parts of the solution --
113 each \c Node therefore stores two unknowns values. By default,
114 the real and imaginary parts are stored as values 0 and 1,
115 respectively; see the section \ref numbering for details.
116 
117 The application of Dirichlet and Neumann boundary conditions is
118 straightforward and follows the pattern employed for the solution
119 of the Poisson equation:
120 - Dirichlet conditions are imposed by pinning the relevant nodal
121  values and setting them to the appropriate prescribed values.
122 - Neumann (flux) boundary conditions are imposed via
123  \c FaceElements (here the \c FourierDecomposedHelmholtzFluxElements).
124  <a href="../../../poisson/two_d_poisson_flux_bc/html/index.html">
125  As usual</a> we attach these to the faces of the "bulk" elements
126  that are subject to the Neumann boundary conditions.
127 .
128 
129 The imposition of the Sommerfeld radiation condition for problems in
130 infinite domains is slightly more complicated. In the following
131 discussion we will assume that the infinite domain is truncated at
132 a spherical artificial boundary \f$ \Gamma \f$ of radius \f$ R. \f$
133 [The methodology can easily be modified to deal with other geometries but this
134 has not been done yet -- any volunteers?]. All methods exploit the
135 fact that the relevant solution of the Helmholtz equation can
136 be written in spherical polar coordinates \f$ (\rho,\varphi,\theta) \f$ as
137 \f[
138 {\tt U}(\rho,\theta,\varphi)=
139 \sum_{l=0}^{+\infty}\sum_{n=-l}^{l}
140 \left(
141 a_{ln} \ h_{l}^{(1)}(k\rho)+
142 b_{ln} \ h_{l}^{(2)}(k\rho)
143 \right)P_{l}^{n}
144 (\cos\theta)\exp({\rm i} n \varphi).
145  \ \ \ \ \ \ \ (6)
146 \f]
147 where the \f$a_{ln}, b_{ln} \f$ are arbitrary coefficients and the functions
148 \f[
149 h_{l}^{(1)}(x)=j_{l}(x)+{\rm i} y_{l}(x) \mbox{ \ \ \ \ and \ \ \ \ }
150 h_{l}^{(2)}(x)=j_{l}(x)-{\rm i} y_{l}(x)
151 \f]
152 are the spherical Hankel functions of first and second kind,
153 respectively, expressed in terms the spherical Bessel functions
154 \f[
155 j_{l}(x)=\sqrt{\frac{\pi}{2x}}J_{l+1/2}(x) \mbox{ \ \ \ \ and \ \ \ \ }
156 y_{l}(x)=\sqrt{\frac{\pi}{2x}}Y_{l+1/2}(x).
157 \f]
158 The functions
159 \f[
160 P_{l}^{m}(x)=(-1)^{m}(1-x^2)^{m/2}\frac{d^m}{dx^m}P_{l}(x)
161 \f]
162 are the associated Legendre functions, expressed in terms of the
163 Legendre polynomials
164 \f[
165 P_{n}(x)=\frac{1}{2^{n} \, n!}\frac{d^n}{dx^n}[(x^2-1)^{n}].
166 \f]
167 This definition shows that \f$ P_{l}^{m}(x)=0 \f$ for \f$ m>l \f$
168 which explains the limited range of summation indices in the second
169 sum in (6).
170 
171 
172 
173 The relation between the cylindrical polar coordinates \f$
174 (r,\varphi,z)\f$ and spherical polar coordinates \f$
175 (\rho,\theta,\varphi) \f$
176 is given by
177 \f[
178 \rho = \sqrt{r^2 + z^2},
179 \f]
180 \f[
181 \theta = \arctan(r/z),
182 \f]
183 \f[
184 \varphi = \varphi,
185 \f]
186 so \f$\varphi \in [0,2\pi] \f$ remains unchanged, and
187 \f$\theta \in [0,\pi] \f$ sweeps from the north pole
188 (\f$ \theta = 0 \f$), via the equator (\f$ \theta = \pi/2 \f$ )
189 to the south pole (\f$ \theta = \pi \f$).
190

192 \subsection DtN The Dirichlet-to-Neumann mapping (DtN)
193 Assuming that the artificial outer boundary \f$ \Gamma \f$is sufficiently far
194 from the region of interest, so that any near field effects
195 associated with the scatterer have decayed,
196 we have to ensure that the solution on \f$ \Gamma \f$ contains only
197 outgoing waves. For our choice of the time-dependence in
198 (2), such waves are represented by
199 the terms involving the spherical Hankel functions of the first kind,
200 \f$ h_l^{(1)} \f$, in (6).
201 
202 The solution on (and near) \f$ \Gamma \f$ is therefore given by
203 \f[
204 {\tt U}(\rho,\theta,\phi)=
205 \sum_{l=0}^{+\infty}\sum_{n=-l}^{l} \ a_{ln}\ h_{l}^{(1)}(k\rho)
206 \ P_{l}^{n}(\cos\theta) \ \exp({\rm i}n\varphi).
207 \f]
208 Restricting ourselves to the single azimuthal Fourier mode \f$ n=N \f$
209 to be determined by (5), we have
210 \f[
211 u_N(\rho,\theta)=\sum_{l=N}^{+\infty}
212 a_{l} \ h_{l}^{(1)}(k\rho)\ P_{l}^{N}(\cos\theta).
213  \ \ \ \ \ \ \ (7)
214 \f]
215 We multiply this equation by \f$ P_{n}^{N}(\cos\theta)
216 \sin(\theta)\f$, integrate over \f$ \theta \f$, and exploit the
217 orthogonality of the Legendre functions, to show that
218 \f[
219 a_{l}=\frac{2l+1}{2 h_{l}^{(1)}(kR)}
220 \frac{(l-N)!}{(l+N)!}\int_{0}^{\pi} u(R,\theta)\ P_{l}^{N}(\cos\theta)
221 \ \sin(\theta) \ d\theta.
222 \f]
223 Using these coefficients, we can differentiate
224 (7) to obtain the normal derivative of the
225 solution on the (spherical) artificial outer boundary \f$ \Gamma \f$ in terms
226 of the solution itself:
227 \f[
228 \left. \frac{\partial u}{\partial n} \right|_{\rho=R} =
229 \left. \frac{\partial u}{\partial \rho} \right|_{\rho=R} = \gamma(u) =
230 k\sum_{n=N}^{\infty}a_{n} \ h_{n}^{(1)'}(kR)\ P_{n}^{N}(\cos \theta)
231 \f]
232 i.e.
233 \f[
234 \gamma(u(R,\theta))=
235 k \sum_{n=N}^{\infty} h_{n}^{(1)'}(kR) \
236 P_{n}^{N}(\cos \theta) \
237 \frac{2n+1}{2h_{n}^{(1)}(kR)}\frac{(n-N)!}{(n+N)!}
238 \int_{0}^{\pi} u(R,\overline{\theta}) \
239 P_{n}^{N}(\cos\overline{\theta}) \ \sin(\overline{\theta}) \;
240 d\overline{\theta},
241 \ \ \ \ \ \ \ \ \ \ (8)
242 \f]
243 a Dirichlet-to-Neumann mapping.
244 
245 
246 Equation (8) provides a condition on the
247 normal derivative of the solution along the artificial boundary
248 and is implemented in the \c FourierDecomposedHelmholtzDtNBoundaryElement class.
249 Since \f$ \gamma \f$ depends on the solution everywhere
250 along the artificial boundary, the application of the Sommerfeld
251 radiation condition via (8) introduces a non-local coupling
252 between all the degrees of freedom located on that boundary.
253 This is handled by classifying the unknowns that affect \f$ \gamma \f$
254 but are not associated with an element's own nodes as its external \c Data.
255 
256 To facilitate the setup of the interaction between the
257 \c FourierDecomposedHelmholtzDtNBoundaryElements, \c oomph-lib provides the
258 class \c FourierDecomposedHelmholtzDtNMesh which provides storage for
259 (the pointers to) the \c FourierDecomposedHelmholtzDtNBoundaryElements
260 that discretise the artificial boundary. The
261 member function
262 \c FourierDecomposedHelmholtzDtNMesh::setup_gamma() pre-computes the
263 \f$ \gamma \f$ values required for the imposition of equation (8).
264 The radius \f$ R \f$ of the artificial boundary and the (finite) upper limit
265 for the sum in (8) are specified
266 as arguments to the constructor of the \c FourierDecomposedHelmholtzDtNMesh.
267 
268 <b>NOTE:</b> Since \f$ \gamma \f$ depends on the solution,
269 it must be recomputed whenever the unknowns are updated during
270 the Newton iteration. This is best done by adding a call to
271 \c FourierDecomposedHelmholtzDtNMesh::setup_gamma() to
272 \c Problem::actions_before_newton_convergence_check().
273 [If Helmholtz's equation is solved in isolation (or within
274 a coupled, but linear problem), Newton's method will converge
275 in one iteration. In such cases the unnecessary recomputation of
276 \f$ \gamma \f$ after the one-and-only Newton iteration can be
277 suppressed by setting \c Problem::Problem_is_nonlinear to
278 \c false.]

285 \section scattering A specific example
286 We will now demonstrate the methodology for a specific example
287 for which the exact solution of (5) is given by
288 \f[
289 u_N(r,z)=u_N^{[exact]}(r,z)=\sum_{l=N}^{N_{\rm terms}}
290 h_{l}^{(1)}(k\sqrt{r^2+z^2}) \ P_{l}^{N}\left(\frac{z}{\sqrt{r^2+z^2}}\right).
291 \f]
292 This solution corresponds to the superposition of several outgoing
293 waves of the form (7) with
294 coefficients \f$ a_l = 1 \f$. We solve the Helmholtz
295 equation in the infinite region surrounding the unit sphere
296 on whose surface we impose flux boundary conditions
297 consistent with the derivative of the exact solution.
298 
299 To solve this problem numerically, we discretise the annular
300 domain \f$ 1 < \sqrt{r^2 + z^2} < 3 \f$ with finite elements and
301 apply the Sommerfeld radiation condition using a Dirichlet-to-Neumann
302 mapping on the artificial
303 outer boundary \f$ \Gamma \f$ located at \f$ \sqrt{r^2 + z^2}
304 = R = 3\f$.
305 
306 The two plots below show a comparison between the exact and computed
307 solutions for \f$ N_{\rm terms}=6 \f$ , a Fourier wavenumber
308 of \f$ N=3 \f$ , and a (squared) Helmholtz wavenumber of \f$ k^2 = 10 \f$.

504 \section comm_ex Comments and Exercises
505 \subsection numbering The enumeration of the unknowns
506 As discussed in the introduction, most practically relevant
507 solutions of the Helmholtz equation are complex valued. Since \c oomph-lib's
508 solvers only deal with real (double precision) unknowns, the equations
509 are separated into their real and imaginary parts.
510 In the implementation of the Helmholtz elements, we store the real
511 and imaginary parts of the solution as two separate values at each
512 node. By default, the real and imaginary parts are accessible via
513 \c Node::value(0) and \c Node::value(1). However, to facilitate the
514 use of the elements
515 in multi-physics problems we avoid accessing the unknowns
516 directly in this manner but provide the virtual function
517 \code
518 std::complex<unsigned> FourierDecomposedHelmholtzEquations<DIM>::u_index_fourier_decomposed_helmholtz()
519 \endcode
520 which returns a complex number made of the two unsigneds that indicate
521 which nodal value represents the real and imaginary parts of the solution.
522 This function may be overloaded in combined multi-physics elements
523 in which a Helmholtz element is combined (by multiple inheritance)
524 with another element, using the strategy described in
525 <a href="../../../multi_physics/b_convection/html/index.html">
526 the Boussinesq convection tutorial</a>.
527 
528 \subsubsection acc The accuracy of the boundary condition elements
529 As discussed above, the Dirichlet-to-Neumann mapping allows an
530 "exact" implementation of the Sommerfeld radiation condition,
531 provided the artificial outer boundary is sufficiently far from the
532 scatterer that any near field effects have decayed. The actual
533 accuracy of the computational results depends on various factors:
534 - The number of \c FourierDecomposedHelmholtzDtNBoundaryElement
535  along the artificial domain boundary. Since these elements are
536  attached to the "bulk" \c FourierDecomposedHelmholtzElements
537  it is important that the bulk mesh is sufficiently fine to resolve the
538  relevant features of the solution throughout the domain.
539 - The number of terms included in the sum (8) --
540  specified in the call to the constructor of the
541  \c FourierDecomposedHelmholtzDtNMesh.
542 - The accuracy of the integration scheme used to evaluate the
543  integral in (8).
544 .

576 \subsubsection sphere Scattering of a planar acoustic wave off a sound-hard sphere
577 
578 Modify the driver code to compute the sound field created when a
579 planar acoustic wave, propagating along the \f$ z \f$-axis, impinges
580 on a sound-hard sphere. The relevant theory is described in
581 <a href="../../../helmholtz/scattering/html/index.html#scattering">another
582 tutorial</a>; you can use the fact that in spherical polar
583 coordinates a planar wave of the form
584 \f[
585 {\tt U}(x,y,z) = \exp({\rm i}kz)
586 \f]
587 can be written as
588 \f[
589 {\tt U}(\rho,\theta) = \sum_{l=0}^{\infty}
590 (2l+1) \ {\rm i}^l \ j_l(kr) \ P_l(\cos\theta),
591 \f]
592 i.e. the wave comprises a single azimuthal Fourier component with \f$
593 N=0\f$. Note that the
594 <A HREF="../../../../demo_drivers/fourier_decomposed_helmholtz/sphere_scattering/sphere_scattering.cc">driver
595 code</a> already contains a namespace \c PlanarWave with several
596 (but not all!) functions required for this task.
