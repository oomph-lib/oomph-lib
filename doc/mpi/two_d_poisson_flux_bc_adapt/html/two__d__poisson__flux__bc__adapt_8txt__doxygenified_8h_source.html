3 \mainpage Parallel solution of a 2D Poisson problem with flux boundary conditions
4 
5 This document provides an overview of how to distribute the
6 <a href="../../../poisson/two_d_poisson_flux_bc_adapt/html/index.html">2D
7 Poisson problem with flux boundary conditions</a>.
8 It is part of a
9 <a href="../../../example_code_list/html/index.html#distributed">
10 series of tutorials</a>
11 that discuss how to modify existing serial driver codes so that the
12 \c Problem object can be distributed across multiple processors.
13 
14 A feature
15 of this problem is that the flux boundary conditions are applied by
16 attaching "flux elements" (derived from the \c FaceElement
17 base class) to the "bulk elements" adjacent to the appropriate mesh
18 boundary. As discussed in the
19 <a href="../../../poisson/two_d_poisson_flux_bc_adapt/html/index.html">
20 tutorial for the serial driver code</a>, the \c FaceElements are
21 not involved in any adaptation within the bulk mesh.
22 Instead, they are detached
23 before the bulk mesh is adapted and re-attached afterwards, which
24  ensures that the \c FaceElements are only attached
25 to bulk elements present in the adapted mesh.
26 
27 The same issue arises when the \c Problem is distributed:
28 all \c FaceElements must be attached
29 before the problem is distributed to allow \c METIS to
30 analyse the interaction between face and bulk elements correctly.
31 However, after the \c Problem has been distributed, some of the bulk
32 elements on each processor will have been deleted, leaving the
33 corresponding \c FaceElement dangling. To deal with such problems,
34 \c oomph-lib provides the empty virtual functions
35 
36 \code
37 Problem::actions_before_distribute()
38 \endcode
39 
40 and
41 
42 \code
43 Problem::actions_after_distribute()
44 \endcode
45 
46 which are called automatically by \c Problem::distribute(...).
47 Specifically, \c Problem::actions_before_distribute() is called
48 \b after the problem distribution has been determined by
49 \c METIS but \b before the actual distribution (during which
50 elements are deleted) takes place. \c
51 Problem::actions_after_distribute() is called after the problem
52 distribution is complete.
53 
54 
55 In the present problem we
56 overload the functions \c Problem::actions_before_distribute() and \c
57 Problem::actions_after_distribute() to perform the same functions as \c
58 actions_before_adapt() (i.e. delete the flux elements) and \c
59 actions_after_adapt() (i.e. re-attach the flux elements).
60 We note that any \c FaceElement that is attached to a halo element in the bulk
61 mesh becomes a halo element itself; see the
62 <a href="../../general_mpi/html/index.html#face_elements">general
63 MPI tutorial</a> for further details.
64 
65 
66 Most of driver code is identical to its serial counterpart and
67 we only discuss the changes required to distribute the problem. Please
68  refer to
69 <a href="../../../poisson/two_d_poisson_flux_bc_adapt/html/index.html">
70 another tutorial</a> for a more detailed discussion of the
71 problem and its (serial) implementation.

76 \section main_body The main function
77 
78 The only changes required to the main function are the usual calls to
79 initialise and finalise \c oomph-lib's MPI routines and a single call
80 to \c Problem::distribute() after the problem has been
81 constructed. The source code is actually slightly more complicated because the
82 distribution is read in from a file so that the driver can be used as a
83 self-test. Note that the file must specify
84 the partition for \b all elements,
85 including the \c FaceElements. (We refer to
86 <a href="../../adaptive_driven_cavity/html/index.html#no_disk">
87 another tutorial</a>
88 for details on how to create the distribution file.)

92 \section problem_class The problem class
93 
94 The only additions to the problem class are the functions \c
95 actions_before_distribute() and \c actions_after_distribute(). As
96 explained above, these perform exactly the same functions as \c
97 actions_before_adapt() and \c actions_after_adapt(), respectively.
