3\mainpage The Bretherton problem: An air finger propagates into a 2D channel
4
5This tutorial discusses the solution of the classical
6Bretherton problem in which an air-finger is driven steadily into a 2D
7fluid-filled, rigid-walled channel.
8
9
10The driver code listed below was originally developed as a
11test-bed for the paper
12
13<CENTER>
14Hazel, A.L. & Heil, M. (2008) The influence of gravity on the steady
15propagation of a semi-infinite bubble into a flexible channel.
16Physics of Fluids 20, 092109
17<a href="http://www.maths.manchester.ac.uk/~mheil/MATTHIAS/PDF/HazelHeilPoF2008.pdf">(pdf reprint)</a>
18</CENTER>
19
20in which we studied the elastic-walled equivalent, paying particular
21attention to the effect of transverse gravity which plays quite an important
22role in this problem.
23
24Compared to other tutorials, there is no detailed discussion of the
25driver code itself because the implementation is somewhat messy.
26However, given that the driver code is (of course!) very well documented
27you should be able to figure out what's going on once you've read
28through the discussion of the problem formulation and our brief
29comments on the \ref implement.
30<a href="../../../contact/html/index.html">Get in touch</a>
31if you need more information.

34\section problem The problem
35
36
37The sketch below shows the problem setup: An (inviscid) air
38finger propagates at a steady speed \f$ U \f$ into a 2D rigid-walled,
39fluid-filled channel of half-width \f$ H_0^* \f$.
40In the process it displaces some of the viscous fluid (of
41density \f$ \rho \f$, viscosity
42\f$ \mu \f$ and surface tension \f$ \sigma^* \f$)
43and deposits a film of thickness \f$ h_0^* \f$ on the channel walls.
44[Note that, for the sake of simplicity, we ignore the effect of
45transverse gravity in this discussion; the driver code listed
46below allows the specification of a gravitational body force which will break
47the up-down symmetry of the solution.]
48
49
50\image html bretherton_sketch.gif "Sketch of the problem setup "
51\image latex bretherton_sketch.eps "Sketch of the problem setup " width=0.75\textwidth
52
53
54We non-dimensionalise the velocities on \f$ U \f$, the lengths
55on \f$ H_0^* \f$ and the pressure on the viscous scale, \f$ \mu U /
56H_0^* \f$ and solve the problem in a frame moving with the
57velocity of the finger. The flow is then governed by the
58non-dimensional Navier-Stokes equations
59\f[
60Re \ u_j\frac{\partial u_i}{\partial x_j} = -\frac{\partial p}{\partial x_i} + \
61\frac{\partial }{\partial x_j }\left( \frac{\partial u_i}{\partial x_j} + \frac{\partial u_j}{\partial x_i} \right)
62\f]
63and the continuity equation
64\f[
65\frac{\partial u_i}{\partial x_i}=0,
66\f]
67where the Reynolds number is defined as \f$ Re=U H_0 \rho/\mu \f$.
68On the free fluid surface, whose outer unit normal we
69denote by \f$ {\bf n} \f$, the fluid normal velocity vanishes,
70\f[
71{\bf u} \cdot {\bf n} = 0 \mbox{\ \ \ on the air-liquid interface}.
72\f]
73We measure the fluid pressure \f$ p \f$ relative to the bubble
74pressure
75\f$ p_b \f$. Then
76the dynamic boundary condition implies that
77\f[
78-p n_i + \left( \frac{\partial u_i}{\partial x_j} +
79\frac{\partial u_j}{\partial x_i} \right) n_j + \frac{1}{Ca} \
80\kappa_f n_i = 0
81\mbox{\ \ \ on the air-liquid interface,}
82\f]
83where \f$ \kappa_f = \kappa_f^* H_0 \f$ is the non-dimensional interface
84curvature and the capillary number \f$ Ca = U\mu/\sigma^* \f$ is
85a non-dimensional measure
86of the bubble velocity.
87The no-slip condition on the channel walls requires that \f$ {\bf
88u}=(-1,0) \f$ at \f$ x_2=\pm 1 \f$. Far behind the bubble tip, the fluid
89remains stationary on the walls. In the moving frame of reference, this
90corresponds to the plug flow profile \f$ {\bf u}=(-1,0) \f$ as \f$
91x_1\to -\infty \f$.
92The residual film thickness \f$ h_0 \f$ has to be determined as part of the
93solution and it determines the volume flux through the system.
94The Poiseuille velocity profile far ahead of the bubble tip,
95\f[
96{\bf u}=(-1+\frac{3}{2}(1-h_0)(1-x_2^2),0) \mbox{\ \ \ as $ x_1\to \infty$,}
97\ \ \ \ \ \ \ \ (1)
98\f]
99is then determined by overall continuity.

126\section implement Implementation
127
128The discretisation of the problem is reasonably straightforward,
129the most (human-)time-consuming part being the creation of the spine mesh,
130discussed in <a href="../../../navier_stokes/spine_channel/html/index.html">
131another tutorial</a>. The real complication arises from the fact
132that the application of the "inflow condition" (1) at
133the right end of the domain -- superficially a Dirichlet condition for the
134velocity -- depends on the non-dimensional film thickness
135\f$ h_0 \f$ at the left end of the domain. This introduces a
136non-local interaction between these degrees of freedom.
137We handle this by providing a templated wrapper class
138\c BrethertonElement (templated by the type of the "wrapped" fluid
139element) which allows the specification of the film thickness as
140external \c Data (i.e. \c Data whose values affect the element's residuals but
141are not determined by the element; see
142the discussion of \c oomph-lib's various types of elemental \c Data
143in <a href="../../../the_data_structure/html/index.html">the "bottom up"
144discussion of the library's data structure</a>). Read the driver
145code listed below to see how it works!
