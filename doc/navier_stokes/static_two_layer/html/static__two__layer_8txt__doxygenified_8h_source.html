3 \mainpage Demo problem: A static interface between two viscous fluids

5 \section overview Overview of the problem
6 
7  We consider a closed rectangular container of unit width that
8 contains two immiscible fluids at rest. The lower fluid is of a
9 prescribed volume \f$ \mathcal{V} \f$ and the interface between the
10 two fluids meets the wall of the container at a contact angle \f$
11 \theta_{c} \f$. In the absence of any body forces or external forces,
12 a static solution is obtained in which the velocity field is zero,
13 the fluid pressure in each layer is constant,
14  and free surface is
15  of constant curvature (an arc of a circle in
16  two-dimensions), set by the
17  contact angle and the geometry of the domain. From simple geometry, the
18  mean curvature of the interface in the present problem
19  is \f$ \kappa = 1/r = 2\cos\theta_{c} \f$.
20 
21  We shall be rather brief in our discussion of this problem because it is
22  extremely similar to the
23  <a href="../../static_single_layer/html/index.html"> static free surface
24  bounding a single layer of viscous fluid. </a> In fact, the main
25  difference is that the region of upper fluid is no longer treated as
26  a single external pressure, but must be meshed so that the
27  Navier--Stokes equations can be solved within it. In fact, the most
28  significant difference between the two problems is that we need to
29  <a href= #mesh> construct a two-layer mesh. </a>
30  Dynamic two-fluid problems
31  are introduced in <a href="../../two_layer_interface/html/index.html">
32  another tutorial,
33  </a> but the static problem discussed here is again complicated by the need to
34  enforce a constant volume constraint.
35 
36 \section vol_const Enforcing the volume constraint
37 
38  Unlike the <a href="../../static_single_layer/html/index.html">
39  equivalent single-fluid problem </a>, there is no external pressure,
40  so the volume constraint must be associated with an internal pressure
41  degree of freedom. Thus, we must hijack a pressure variable and we
42  choose to do so in the upper fluid.
43 
44  \dontinclude static_two_layer.cc
45  \skipline Hijack
46  \until hijack_internal
47 
48  In addition, we must fix another fluid pressure at a fixed reference value
49  so that the problem is non-degenerate. It is conceptually appealing
50  to fix the reference pressure in the other (lower) fluid because the
51  (constant) pressure in the upper fluid is already "being set" by the
52  volume constraint.

64 \section mesh Constructing the Two-Layer Elastic Mesh
65 
66  We must create a two-layer elastic mesh that will allow us access to
67  the elements in each fluid. We will also need to set the volume
68  constraint by adding \c ElasticLineVolumeConstraintBoundingElements
69  to the boundaries that surround one of the fluids and to add the \c
70  ElasticLineFluidInterfaceElements along the interface. Thus, we need
71  to change the boundaries of the existing mesh.
72

122 \section problem The problem class
123 
124  The problem class is extremely similar to that in the
125  <a href="../../static_single_layer/html/index.html"> singer-layer
126  problem. </a> The main differences are that the \c
127  ElasticTwoLayerMesh is used instead of the \c
128  ElasticRectangularQuadMesh and the boundary conditions are modified
129  to take the new boundary numbering into account. In addition, the
130  free surface elements and volume constraint elements
131  are created using the lookup schemes in the \c
132  ElasticTwoLayerMesh to ensure that only a single layer of elements
133  are included on the free surface. If we used the lookup schemes
134  assigned by the generic function \c
135  Mesh::setup_boundary_element_info() bulk elements on
136  both sides of internal boundaries will be included, so we would
137  construct twice as many interface elements as required.

143 \subsection com Comments
144 
145 - The driver code also contains a formulation in which \c SpineElements
146  are used. Happily, the answers produced by both formulations are the same.
147 
148 - An axisymmetric version of the problem is also included in the
149  library. The problem is so similar that it is not described in any
