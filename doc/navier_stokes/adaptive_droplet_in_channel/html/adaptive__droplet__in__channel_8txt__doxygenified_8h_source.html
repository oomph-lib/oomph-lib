3 \mainpage Propagation of a droplet in a channel - mesh generation and adaptation for free surface problems
4 
5 In this tutorial we demonstrate another adaptive solution of free surface
6 problems on unstructured meshes, using the example of a droplet
7 propagating along a straight channel. The problem is extremely similar
8 to <a href="../../adaptive_bubble_in_channel/html/index.html"> the
9 propagation of a bubble in a channel tutorial </a>. Thus, we shall
10 only discuss the differences from that tutorial. The key physical
11 difference is that instead of the uniform pressure state in an
12 inviscid bubble, the droplet
13 consists of a viscous fluid that can support internal stress variations.
14

18 \section example The example problem
19 We illustrate the solution
20 of the unsteady 2D Navier-Stokes equations by considering the
21 propagation of a single droplet along a straight channel as shown in
22 the sketch below. The non-dimensionalisation is
23 the same as in the
24 <a href="../../adaptive_bubble_in_channel/html/index.html"> bubble
25 tutorial, </a> and we choose the viscosity and density of the
26 surrounding liquid to be the reference values.
27 
28 \image html problem.gif "The problem setup. "
29 \image latex problem.eps "The problem setup. " width=0.9\textwidth
30 
31 The governing equations and boundary conditions are
32 the same as those in the
33 <a href="../../adaptive_bubble_in_channel/html/index.html"> bubble
34 tutorial. </a>
35 The only difference is that the Navier--Stokes equations must also be
36 solved within the droplet. In fact, this is a two-fluid problem, a
37 class of problems that is first introduced in
38 <A HREF="../../two_layer_interface/html/index.html"> another tutorial. </A>
39 
40 <HR>
41 
42 The constraint that the droplet
43 volume remains constant must be enforced explicitly in the static
44 case, as in the bubble problem. In the time-simulations, however, a
45 constant drop volume is implicitly enforced by the continuity
46 equation. For the bubble, the continuity equation is
47 not solved within the interior, which is why the volume constraint
48 must always be explicitly enforced in that case.

52 \section implementation Implementation
53 
54 We use the same method as in the
55 <a href="../../adaptive_bubble_in_channel/html/index.html"> bubble
56 problem, </a> an ALE-based finite-element method with
57 <a href="../../single_layer_free_surface/html/index.html#kinematic_condition_implementation">a pseudo-elastic
58 node-update procedure</a>. In this case,
59 there is a pressure jump across the interface between the
60 fluids, which means that we use triangular Crouzeix--Raviart
61 elements rather than the continuous-pressure Taylor--Hood
62 elements. Again, we impose the kinematic and dynamic boundary conditions
63 with \c FaceElements. The volume constraint for the static problem
64 is also imposed in a similar
65 way: we attach \c LineVolumeConstraintBoundingSolidElements to the
66 droplet surface and create an additional \c VolumeConstraintElement.
67 Here, a pressure degree of
68 freedom within the droplet is hijacked, see
69 <A HREF="../../static_two_layer/html/index.html"> another tutorial, </A>
70 to be used as the unknown associated with the volume constraint. Once
71 the static initial problem has been solved, the
72 "volume constraint" elements are deleted and the pressure degree of
73 freedom is unhijacked.

77 \section results Results
78 We perform the simulation in a two-stage procedure. We start by
79 performing a steady solve with the inflow switched off. This
80 deforms the droplet into its steady state (approximately) circular
81 configuration with the required volume. The actual time-dependent
82 simulation is then performed with an impulsive
83 start from this configuration.
84 
85 The figure below shows the location of the droplet and mesh (upper figure) and
86 a contour plot of the pressure
87 distribution with overlaid velocity vectors of the difference between
88 the background Poiseuille flow and the velocity field (lower
89 figure). The figure is a snapshot for the parameters \f$ Re = ReSt =
90 0.0 \f$, \f$ Ca = 10.0 \f$ and a droplet that is ten times as viscous
91 as the surrounding liquid.

116 
117 The first difference from the bubble problem is that the steady solver
118 does not converge when \f$ Ca = 10 \f$. Instead, we start with \f$

134 \section problem The problem class
135 
136 Other than trivial name changes from "bubble" to "drop", there are
137 a few significant changes between this problem and that in
138 the
139 <a href="../../adaptive_bubble_in_channel/html/index.html"> bubble
140 tutorial. </a> There is an additional boolean \c Use_volume_constraint
141 and an additional function \c
142 remove_volume_constraint(), which are used to manage the switch from
143 the explicit enforcement of the volume constraint in the static case
144 to the implicit enforcement in the time simulations. The other key
145 difference is that we use \c Triangle's
146 <a href="../../../meshes/mesh_from_inline_triangle_internal_boundaries/html/index.html#def_extra_regions"> region attributes </a>
147 to distinguish the elements inside
148 the droplet from those outside. The default behaviour is that all
149 elements are in region 0, but we label those element within the drop
150 as region 1.

178 \section face_elements Generation of face elements
179 
180 As usual we impose
181 the kinematic and dynamic boundary condition at the interface by
182 attaching \c FaceElements to the relevant boundaries of the bulk
183 elements. However, we must be careful to add only a \b single layer of
184 elements. If we use the standard "boundary element" functions then we
185 will be creating face elements on both sides of the internal
186 boundary. Instead, we use the elements adjacent to the boundary
187 within region 0, which ensures that only a single layer of interface
188 elements are added.
189 
190 \dontinclude adaptive_drop_in_channel.cc
191 \skipline start_of_create_free_surface_elements
192 \until end of create_free_surface_elements
193 
194 The volume constraint elements are only created if the boolean flag \c
195 Use_volume_constraint is true (the default on construction of the problem).
196 We hijack the pressure degree of freedom associated with the first
197 element in region 1 and then the construction of the elements again
198 uses the regions to ensure that a single layer of elements is created.
199 
200 \skipline start_of_create_volume_constraint_elements
201 \until end of create_volume_constraint_elements

205 \section remove Removal of the volume constraint
206 
207 The function \c remove_volume_constraint(), resets the boolean flag to
208 false, clears the hijacked data, deletes the volume constraint
209 elements and mesh and then removes the volume constraint mesh from the
210 problem's list of sub meshes, before reassigning the equation numbers.

219 \section comments Comments and Exercises
220 
221 
222  The computation of the initial static solution means that we must
223  still include the calls to the \c create_volume_constraint_elements() and
224  \c delete_volume_constraint_elements() in \c actions_before_adapt()
225  and \c actions_after_adapt(). We have chosen to have the functions
226  return immediately if the volume constraint is not being enforced,
227  rather than using \c if blocks within the \c
228  actions_before/after_adapt() functions.
