55  <h1>Welcome to the oomph-lib homepage</h1>
56  <p><code>oomph-lib</code> is an object-oriented, open-source
57  finite-element library for the simulation of multi-physics
58  problems. It is developed and maintained by <a href=
59  "http://www.maths.man.ac.uk/~mheil">Matthias Heil</a> and <a href=
60  "http://www.maths.man.ac.uk/~ahazel">Andrew Hazel</a> of the
61  <a href="http://www.maths.manchester.ac.uk/">School of
62  Mathematics</a> at <a href="http://www.manchester.ac.uk/">The
63  University of Manchester</a>, along with many <a href=
64  "../people/html/index.html">other contributors</a>.</p>
65  <p> See the <a href="../change_log/html/index.html">change
66  log page</a> for an overview of the changes made since the previous
67  release.</p>
68  <h2>What is it [and what is it not]?</h2>
69  <code>oomph-lib</code> <b>is</b>:
70  <ul>
71  <li>an object-oriented, open-source finite-element library for the
72  simulation of multi-physics problems.</li>
73  </ul>
74  <code>oomph-lib</code> <b>is not</b>:
75  <ul>
76  <li>a GUI-based, mouse-driven "package".</li>
77  </ul>
78  You must write your own C++ driver codes to specify your problem,
79  using <code>oomph-lib</code>'s high-level objects. Once the problem
80  is formulated, the <code>main</code> function can be as simple as
81  this:
82  <div class="fragment">
83  <div class="line"> main()</div>
84  <div class="line"> {</div>
85  <div class="line"> <span class="comment">// Create the problem object</span></div>
86  <div class="line"> <span class="keywordtype">MyProblem</span> problem;</div>
87  <div class="line"> </div>
88  <div class="line"> <span class="comment">// Solve the problem, using oomph-lib&#39;s default Newton solver</span></div>
89  <div class="line"> problem.newton_solve();</div>
90  <div class="line"> }</div>
91  </div>
92  <code>oomph-lib</code> provides the state of the art infrastructure
93  for the problem formulation and solution. The library contains a
94  large number of elements, meshes, timesteppers, solvers, etc. which
95  can be combined to solve any problem.
96  <p>The library is supplied with extensive on-line documentation
97  which includes:</p>
98  <ul>
99  <li>
100  <h4><a href="../intro/html/index.html"><b>Theory: Finite Elements
101  -- from procedural to object-oriented</b></a></h4>
102  This provides a relatively brief, constructive "top-down"
103  introduction to the maths of the finite element method, and
104  explains its object-oriented implementation in
105  <code>oomph-lib</code>.</li>
106  <li>
107  <h4><a href="../the_data_structure/html/index.html"><b>The Data
108  Structure</b></a></h4>
109  A bottom-up discussion of <code>oomph-lib's</code> overall data
110  structure, including a complete, cross-referenced index of all
111  <code>oomph-lib's</code> classes (generated by Doxygen).</li>
112  <li>
113  <h4><a href="../quick_guide/html/index.html"><b>(Not-So-)Quick
114  Guide</b></a></h4>
115  The <a href="../quick_guide/html/index.html">(Not-So-)Quick
116  Guide</a> provides a "quick" introduction on how to create new
117  instances of <code>oomph-lib's</code> fundamental objects:
118  <code>Problems</code>, <code>Meshes</code>, and
119  <code>Elements</code>.</li>
120  <li>
121  <h4><a href="../example_code_list/html/index.html"><b>Example
122  Codes/Tutorials</b></a></h4>
123  We provide a large number of verified and documented example
124  problems.
125  <ul>
126  <li style="margin-top:10px;">The example problems are arranged in
127  order of increasing complexity and can be used as chapters in a
128  self-study course. Comments and exercises are provided to encourage
129  further exploration of <code>oomph-lib's</code> capabilities.</li>
130  <li style="margin-top:10px;">The individual examples are as
131  self-contained as possible and can be used as quick guides to
132  particular features of <code>oomph-lib</code>. ("How do I solve
133  time-dependent problems?"; "How do I enable spatial adaptivity?";
134  "How do I write my own meshes?"; etc.)</li>
135  <li style="margin-top:10px;">The examples codes should be regarded
136  as templates for your own driver codes: Examine the <a href=
137  "../example_code_list/html/index.html">list of examples</a> and try
138  to find one that resembles the problem that you wish to solve. For
139  most "classical" problems (e.g. the solution of the Navier-Stokes
140  equations with standard boundary conditions), very few
141  modifications are required (e.g. adjusting the mesh to a different
142  domain, changing the boundary and/or initial conditions, etc). If
143  you wish to solve a non-standard problem, or one for which
144  <code>oomph-lib</code> does not (yet) provide elements or meshes,
145  it is easy to write your own, following the detailed instructions
146  provided in the documentation.</li>
147  </ul>
148  </li>
149  </ul>

154  <article class="tab-pane" id="features">
155  <h1>Features</h1>
156  <ul>
157  <li>Large number of fully-developed and carefully-validated element
158  types for the solution of:
159  <ul>
160  <li>The Poisson equation</li>
161  <li>The unsteady heat equation</li>
162  <li>The linear wave equation</li>
163  <li>The advection-diffusion equation</li>
164  <li>The Navier-Stokes equations, incl. free-surfaces, in Cartesian
165  and axisymmetric co-ordinates.</li>
166  <li>The equations of large-displacement solid mechanics, based on
167  the principle of virtual displacements in cartesian and
168  axisymmetric geometries; with arbitrary constitutive
169  equations.</li>
170  <li>Shell and beam elements, based on a geometrically non-linear
171  formulation.</li>
172  <li>...and many others.</li>
173  </ul>
174  [<b>Note:</b> Most elements are implemented in a
175  dimension-independent formulation]</li>
176  <li>It is easy to formulate new elements for problems that are not
177  included in the above list.</li>
178  <li>It is easy to combine any of the above elements to
179  "multi-physics" elements, allowing e.g.
180  <ul>
181  <li>the solution of fluid-structure interaction problems</li>
182  <li>the simulation of unsteady heat conduction in an elastic body
183  that undergoes large displacements</li>
184  <li>free-surface problems</li>
185  <li>etc.</li>
186  </ul>
187  </li>
188  <li>Parallel processing capabilities.</li>
189  <li>Full spatial and temporal adaptivity. All elements listed above
190  support quad-tree/octree-based mesh refinement procedures and
191  adaptive timestepping.</li>
192  <li>A wide range of meshes; new meshes can easily be added to the
193  library. <code>oomph-lib</code> can also use meshes that are
194  generated by third-party (commercial) mesh generators.</li>
195  <li>The ability to solve problems in domains with time-dependent,
196  curvilinear boundaries.</li>
197  <li><code>oomph-lib</code> treats all problems as non-linear
198  problems and uses Newton's method as the default non-linear solver.
199  Within this framework, linear problems are special cases for which
200  the Newton iteration converges in one step. The linear systems
201  arising during the Newton iteration can be solved with a variety of
202  direct and iterative solvers. Continuation methods (Keller's
203  arclength method and displacement-control for solid mechanics
204  problems) are implemented.</li>
205  <li>A large number of <a href=
206  "../example_code_list/html/index.html">fully documented example
207  codes.</a> All examples include an introduction to the
208  mathematical/physical problem, a detailed discussion of the driver
209  code, a discussion of the results (incl. comparisons against
210  analytical solutions where appropriate) and exercises and
211  comments.</li>
212  <li>The library has extensive self-test facilities. It can be
213  compiled with a PARANOID flag switched on or off. If the flag is
214  set during compilation, the code performs a large number of
215  self-tests during its execution. If things go wrong, code execution
216  terminates gracefully and diagnostic output is generated to help
217  pinpoint the problem. Of course, the tests introduce an additional
218  run-time overhead, so can be switched off in production runs.
219  During development, range checking on all array-based objects,
220  (vectors, matrices, tensors, etc) can be enabled separately by
221  using the RANGE_CHECKING flag, but introduces considerable run-time
222  overhead.</li>
223  </ul>

228  <h1>How do I get/install the code?</h1>
229  <h2>Downloads</h2>
230  <p><code>oomph-lib</code> is hosted on and distributed via GitHub at
231  <center><a href="https://github.com/oomph-lib/oomph-lib">
232  https://github.com/oomph-lib/oomph-lib</a>.</center><br>
233  For more information on how to download and install the library see
234  the <a href="../the_distribution/html/index.html">installation
235  guide</a>. </p>
236  <h2>Prerequisites</h2>
237  <p><code>oomph-lib</code> has been/is primarily being developed in
238  Linux so it is well tested there. However several people
239  successfully run it on Windows and Mac OS X. Our continuous
240  integration on GitHub currently checks Linux and Mac OS X builds.
241  In order to minimise
242  the dependence on external libraries we include "frozen" versions
243  of certain external libraries (<a href=
244  "http://www-users.cs.umn.edu/~karypis/metis/">METIS</a>, <a href=
245  "http://crd.lbl.gov/~xiaoye/SuperLU">SuperLU</a>, ...) with our
246  distribution. The idea is that you should only have to unpack,
247  build and install one distribution to produce the fully compiled
248  library, the html-based documentation and working example
249  codes.</p>
250  For this purpose you <b>must have</b>:
251  <ul>
252  <li>A computer with a linux (or unix) operating system. [<b>Note
253  for windows users:</b> We believe that <code>oomph-lib</code> can
254  be installed in a windows environment, using <a href=
255  "http://www.cygwin.com/">cygwin</a>, though we haven't tried this
256  for a long time.]</li>
257  <li>Compilers for C++, C and Fortran77. The open-source <a href=
258  "http://gcc.gnu.org/">GNU gcc compiler suite</a> version 9.3.0. or
259  later is fine. [ <b>Note:</b> The Fortran77 and C compilers are
260  only required to compile some external libraries --
261  <code>oomph-lib</code> itself is written entirely in C++. We assume
262  that your C++ compiler can handle the C++11 standard. If it doesn't
263  you may have to add the <code>-std=c++11</code> option to the
264  <code>CXXFLAGS</code> during the configure stage.]</li>
265  </ul>
266  The following programs are helpful but not essential:
267  <ul>
268  <li>GNU's <a href=
269  "http://www.gnu.org/software/autoconf/">autoconf</a>, <a href=
270  "http://www.gnu.org/software/automake/">automake</a> and <a href=
271  "http://www.gnu.org/software/libtool/">libtool</a> are required if
272  you wish to add additional features to your local copy of
273  <code>oomph-lib</code>; see <a href=
274  "../the_distribution/html/index.html">the <code>oomph-lib</code>
275  download page</a> for more details.</li>
276  <li><a href="http://www.python.org/download/">python3</a> is needed
277  to analyse the results of the extensive self-tests that may be
278  performed at the end of the installation to verify that the build
279  process was completed successfully. If python3 is not available, the
280  self-test will compile and run all test codes but it will not be
281  possible to verify the correctness of the results.</li>
282  <li><a href="http://www.doxygen.org">doxygen</a> is needed to build
283  a local copy of <code>oomph-lib</code>'s extensive online
284  documentation. If doxygen is not installed on your system you can
285  consult the latest version of the online documentation on <a href=
286  "http://www.oomph-lib.org">the <code>oomph-lib</code>
287  homepage.</a></li>
288  </ul>
289  The <code>oomph-lib</code> distribution is built under <a href=
290  "http://www.gnu.org/software/autoconf/">autoconf</a> / <a href=
291  "http://www.gnu.org/software/automake/">automake</a> control,
292  making the installation completely straightforward. We provide two
293  main installation mechanisms:
294  <ul>
295  <li><code>oomph-lib</code> can be built/installed/tested with GNU's
296  standard <code>configure; make; make install; make check</code>
297  procedure.</li>
298  <li>We also provide a customised build script,
299  <code>autogen.sh</code>, that guides the "non-expert" user through
300  the installation. The script facilitates the specification of
301  various build options, such as optimisation levels etc.</li>
302  </ul>
303  Subject to the minimum requirements, listed above, you should be
304  able to install and use our libraries "anywhere". However, as
305  anybody who has ever ported any code to a different platform knows,
306  things are rarely that simple -- even with <a href=
307  "http://www.gnu.org/software/autoconf/">autoconf</a>, <a href=
308  "http://www.gnu.org/software/automake/">automake</a> and <a href=
309  "http://www.gnu.org/software/libtool/">libtool</a>... Having said
310  that, we have successfully installed and tested our installation on
311  the following platforms:
312  <ul>
313  <li>Ubuntu (version 16.04) with gcc compilers (version 5.4.0). The
314  tests included the MPI part of the library, which we compiled with
315  <a href="https://www.open-mpi.org/">OpenMPI</a> (version
316  1.10.2).</li>
317  <li> ... </li>
318  </ul>

393  <h1>How do I get started?</h1>
394  <code>oomph-lib</code> contains a lot of code and a lot of
395  documentation. How to get started obviously depends on your
396  background: Are you familiar with the finite element method? How
397  good is your knowledge of C++? Etc. Here are some possible
398  "routemaps" around the library:
399  <ul>
400  <li><b>You are familiar with the finite element method and have a
401  fairly good knowledge of C++</b>
402  <ul>
403  <li>Have a look through the <a href=
404  "../example_code_list/html/index.html">list of example codes</a> to
405  get a feeling for <code>oomph-lib</code>'s capabilities. Pick a
406  problem that interests you and study the associated tutorial. Copy
407  the driver code into your own directory and play with it.</li>
408  <li>Once you have played with a few example codes, you may wish to
409  to learn more about <code>oomph-lib</code>'s <a href=
410  "../the_data_structure/html/index.html">overall data structure,</a>
411  or find out <a href="../optimisation/html/index.html">how to
412  optimise the library</a> for your particular application.</li>
413  </ul>
414  </li>
415  <li><b>You have never used finite element methods but have a fairly
416  good knowledge of C++</b>
417  <ul>
418  <li>Study the <a href="../intro/html/index.html">"Top Down"
419  introduction</a>. This document includes a "low tech" overview of
420  the mathematical/theoretical background and contrasts procedural
421  implementations of the finite element method with the
422  object-oriented approach adopted in <code>oomph-lib</code>.</li>
423  <li>Consult the <a href=
424  "../quick_guide/html/index.html">(Not-So-)Quick-Guide</a> to learn
425  how to construct basic <code>oomph-lib</code> objects for your
426  problem: <code>Problems</code>, <code>Meshes</code>,
427  <code>FiniteElements</code>, etc.</li>
428  <li>Continue with the steps suggested above.</li>
429  </ul>
430  </li>
431  <li><b>You have never used finite element methods and are a
432  newcomer to C++</b>
433  <ul>
434  <li>Buy <a href="http://www.math.wayne.edu/~yang/">Daoqi Yang</a>
435  brilliant book <a href=
436  "http://www.springeronline.com/sgw/cda/frontpage/0,11855,4-40007-22-2105335-0,00.html?changeHeader=true">
437  C++ and Object-Oriented Numeric Computing for Scientists and
438  Engineers.</a> Read it! Pretty much everything in this book is
439  relevant for some parts of <code>oomph-lib</code>. You should at
440  least understand:
441  <ul>
442  <li>The procedural aspects of C++ (basic types, functions and
443  control structures).</li>
444  <li>Namespaces.</li>
445  <li>Classes (private, protected and public members; inheritance and
446  multiple inheritance; virtual and pure virtual functions; base
447  classes and derived classes; static and dynamic casts).</li>
448  <li>Templates and template instantiations.</li>
449  <li>The standard template library (STL).</li>
450  </ul>
451  </li>
452  <li>Continue with the steps suggested above.</li>
453  </ul>
454  </li>
455  </ul>
