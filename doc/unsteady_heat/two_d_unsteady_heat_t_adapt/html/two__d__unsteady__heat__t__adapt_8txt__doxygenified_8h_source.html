3\mainpage Example problem: Solution of the 2D unsteady heat equation with temporal adaptivity
4
5This example illustrates \c oomph-lib's adaptive timestepping
6capabilities. We consider, yet again, the 2D unsteady
7heat equation
8<CENTER>
9<TABLE>
10<TR>
11<TD>
12<CENTER>
13<B>The two-dimensional unsteady heat equation in a square domain.</B>
14</CENTER>
15Solve
16\f[
17\sum_{i=1}^2\frac{\partial^2 u}{\partial x_i^2}
18= \frac{\partial u}{\partial t} + f\left(x_1,x_2,t\right),
19 \ \ \ \ \ \ \ \ \ \ (1)
20\f]
21in the square domain \f$ D = \{x_i \in [0,1]; i=1,2 \} \f$,
22subject to the Dirichlet boundary conditions
23\f[
24\left. u\right|_{\partial D}=g_0
25\ \ \ \ \ \ \ \ \ \ (2)
26\f]
27and initial conditions
28\f[
29u(x_1,x_2,t=0)=h_0(x_1,x_2),
30\ \ \ \ \ \ \ \ \ \ (3)
31\f]
32where the functions \f$ g_0\f$ and \f$ h_0\f$ are given.
33</TD>
34</TR>
35</TABLE>
36</CENTER>
37
38Here we choose the forcing function and the boundary
39and initial conditions so that
40\f[
41u_0(x_1,x_2,t) = \frac{1}{2} \bigg( 1 + \tanh\big(\gamma \cos(2\pi
42t)\big)\bigg) \sin\left(K \left( x_1 \cos \Phi + x_2 \sin
43\Phi\right)\right),
44 \ \ \ \ \ \ \ \ \ \ (4)
45\f]
46is the exact solution. The solution represents a 1D sin
47profile with wavenumber \f$ K \f$, rotated against
48the \f$x_1\f$-axis by an angle \f$ \Phi \f$, and modulated
49by a time-periodically varying amplitude. The parameter \f$ \gamma \f$
50controls the rate at which the amplitude changes. For large
51values of \f$ \gamma \f$, the amplitude remains constant for
52most of the period but changes rapidly at
53\f$ t = \left( 1/4 + i/2 \right), i=0,1,2,... \f$.
54To resolve these rapid changes accurately, very small timesteps
55are required. Conversely, relatively large timesteps could be employed
56during the phases when the solution remains approximately constant.
57The problem therefore presents an ideal test case for
58an adaptive timestepping scheme.
59

62the exact and computed
63solutions,</A> obtained from a simulation with \c oomph-lib's

65subsequent frames in this animation varies, each frame shows a blue line (in
66the top left corner) whose length is proportional to the elapsed time.
67The different rate at which the length of the line increases

91- We define a global error norm for the adaptive timestepper by overloading
92 the (empty) virtual function \c Problem::global_temporal_error_norm().
93- We replace the call to \c Problem::unsteady_newton_solve(...) by
94 a call to \c Problem::adaptive_unsteady_newton_solve(...) and specify
95 a target for the temporal error norm.

113\section main The driver code
114
115Temporal adaptivity only requires a few straightforward changes
116to the time-stepping loop. Since the number of timesteps required to
117reach the end of the simulation is not known a priori, we
118replace the \c for - loop over the fixed number of timesteps, employed
119in the <A HREF="../../two_d_unsteady_heat/html/index.html">
120non-adaptive version of the code,</A> by a \c while - loop that continues the
121time-integration until \f$ t \ge t_{max}\f$.
122
123 The adaptive timestepper, \c Problem::adaptive_unsteady_newton_solve(...)
124takes two arguments. The first one is a suggestion for the
125size of the next timestep; the second specifies the target error
126The adaptive timestepper automatically adjusts the timestep
127until the error estimate computed by \c Problem::global_temporal_error_norm()
128is less than that target. If the error estimate for the
129solution computed with the suggested value of \c dt is too large,
130\c dt is reduced by a factor of 2 and the solution is recomputed.
131This process is repeated until
132- the estimated error has become sufficiently small
133.
134or
135- \c dt has been reduced below a threshold.
136.
137The threshold is stored
138in the private member data \c Problem::Minimum_dt and is initialised
139to \f$ 10^{-12} \f$. This default can be changed with the
140access function \c Problem::minimum_dt(). It is also possible to
141specify a maximum value for the timestep by overwriting the default
142value \f$ 10^{12} \f$ for the corresponding data member,
143 \c Problem::Maximum_dt, using the function \c Problem::maximum_dt().
144
145 The adaptive unsteady Newton solver returns a suggestion for
146the size of the next timestep.
147
148 Here is the relevant code fragment from the otherwise unchanged
149\c main function:
150
151\dontinclude two_d_unsteady_heat_t_adapt.cc
152\skipline Target error for adaptive timestepping
153\until // end of timestepping loop
154
155The rest of the main function is identical
156to that in the <A HREF="../../two_d_unsteady_heat/html/index.html">
157previous, non-adaptive example.</A>

243\section error Defining the global error norm for the adaptive timestepper
244
245\subsection back Background
246
247\c oomph-lib's adaptive timesteppers employ a predictor-corrector
248scheme to control the size of the timestep. In these schemes a
249low-order explicit timestepper
250is used to predict the solution at the next timestep.
251This prediction is compared to the solution computed with the
252actual (usually implicit) timestepper itself. The difference
253between the two predictions is then used to derive an estimate of
254the error (exploiting the different truncation errors of the two
255timestepping schemes).
256
257
258Interfaces to the functions that compute the temporal error estimates
259are defined as broken virtual function in the \c TimeStepper base class.
260Specific \c TimeSteppers that allow adaptive timestepping
261therefore overload the broken virtual function
262
263
264\code
265TimeStepper::temporal_error_in_value(data_pt,i)
266\endcode
267
268which computes an estimate of the error of \c i - th value stored
269in the \c Data object pointed to by \c data_pt. In free-boundary
270problems in which the position of the nodes
271is an unknown, the corresponding function
272
273\code
274TimeStepper::temporal_error_in_position(node_pt,i)
275\endcode
276
277may be used to obtain an estimate of the error in the \c i - th nodal
278coordinate of the \c Node pointed to by \c node_pt.
279
280
281These individual error estimates must be combined into
282a problem-specific, scalar error norm, \f${\cal E}\f$ say,
283which forms the basis of the adaptive adjustment of the timestep in
284\c Problem::adaptive_unsteady_newton_solve(...).
285
286\subsection impl Implementation
287
288In the present problem, we choose the RMS of the errors
289 \f$ e_j \ (j=1,...,N_{node})\f$ at the nodes as the
290global error norm
291\f[
292{\cal E} = \sqrt{ \frac{1}{N_{node}} \sum_{j=1}^{N_{node}} e_j^2 }.
293\f]
294This may be implemented in a few lines of code:
295
296\dontinclude two_d_unsteady_heat_t_adapt.cc
297\skipline start_of_global_temporal_error_norm
298\until end of global_temporal_error_norm

305\section comments Comments and Exercises
306As demonstrated above, enabling temporal adaptivity
307for a given problem is extremely straightforward as it only
308requires the implementation of the problem-specific
309function \c Problem::global_temporal_error_norm().
310In most cases, the RMS of the nodal errors (or some suitable
311generalisation for vector-valued problems) is an obvious
312choice. In free-boundary problems, the RMS of the error estimate for
313the nodal positions is often a useful error measure.

317\subsection norm How to choose the target for the temporal error norm
318 Having decided on an error norm, how does one choose the target
319for the error norm? The answer is the same as in a simulation with
320a fixed timestep: Trial and error, followed by careful convergence
321tests. We usually employ the following strategy:
322- Implement the function \c Problem::global_temporal_error_norm()
323 and perform an initial computation with a fixed timestep, choosing
324 its size heuristically, exploiting prior knowledge that we (usually!)
325 have about our problem. For instance, if we expect a periodic solution
326 with an approximate period \f$ T \f$, we may start with a fixed timestep
327 \f$ dt = T/20,\f$ say. The computed results are likely to be very
328 inaccurate but the time-trace will usually reveal the
329 characteristic features of the solution and thus identify phases
330 during which the solution varies rapidly.
331- Now repeat the simulation with a smaller time-step, \f$ dt =
332 T/40,\f$ say, and check if any new features develop in the
333 time-trace. If the time-trace appears to be robust, monitor the
334 temporal error norm, e.g. by including the output of
335 \c Problem::global_temporal_error_norm() into the trace file.
336- Use the maximum of the temporal error norm observed during the
337 simulation with the fixed timestep as the target in a first
338 simulation with temporal
339 adaptivity. The timestepper should now increase the size of the
340 timestep in regions where the error estimate was small.
341- Now repeat the simulation with smaller and smaller target errors
342 until further reductions do not lead to further changes in the
343 computed results.
344
345\subsubsection ex Exercise:
346Employ the above procedure to determine the target error \f$ \hat{\cal
347E}\f$ required for the computed solution to be graphically
348indistinguishable from that obtained with a target error of
349\f$ \hat{\cal E}/2\f$.
