3 \mainpage Demo problem: A one-dimensional eigenproblem with complex eigenvalues
4 In this document, we demonstrate how to solve a 1D eigenproblem
5 (eigenvalues of the shifted 1D Laplace operator in a bounded domain) by
6 creating custom elements. The tutorial is related to the
7 <a href="../../harmonic/html/index.html"> harmonic eigenproblem </a>,
8 which you should read first. The fundamental difference in this case,
9 is that we treat the eigenproblem as two coupled first-order equations
10 which ensures that many of the eigenvalues are complex.
11 <CENTER>
12 <TABLE>
13 <TR>
14 <TD>
15 <CENTER>
16 <B>One-dimensional model eigenvalue problem with complex eigenvalues</B>
17 </CENTER>
18 Solve
19 \f[
20 \frac{\partial u}{\partial
21 x} = (\lambda - \mu) w, \quad \frac{\partial w}{\partial x} = \lambda u
22  \ \ \ \ \ \ \ \ \ \ (1)
23 \f]
24 in the domain \f$D =\left\{ x_1 \in
25 [0,1] \right\}\f$,
26 with homogeneous Dirichlet boundary conditions
27 \f[
28 \left. u\right|_{\partial D}=0.
29 \ \ \ \ \ \ \ \ \ \ (2)
30 \f]
31 </TD>
32 </TR>
33 </TABLE>
34 </CENTER>
35 
36  We shall treat \f$ \lambda\f$ as an eigenvalue (unknown), but the
37  other parameter \f$ \mu \f$ is a \b shift, which is used to modify
38  the spectrum of the operator.
39  The two first-order equations may be combined into a single
40  second-order equation
41 \f[ \frac{\partial^{2} u}{\partial x^{2}} = (\lambda - \mu)
42  \frac{\partial w}{\partial x} = (\lambda - \mu) \lambda u \quad
43  \Rightarrow\quad \frac{\partial^{2} u}{\partial x^{2}} +
44  (\mu - \lambda) \lambda u = 0, \f]
45 which is exactly the same equation as
46 the <a href="../../harmonic/html/index.html"> harmonic eigenproblem </a>,
47  but with a redefinition of the eigenvalues. Hence, the exact
48  solutions are given by the countably infinite set:
49 \f[
50  u_{n} = \sin (\Lambda_{n} x_{1}), \ \ \ \ \ \ \
51  \Lambda_{n} = n \pi = \sqrt{\lambda_{n}(\mu - \lambda_{n})},
52 \f]
53  which yields
54 \f[ \lambda_{n} = \frac{\mu}{2} \pm \sqrt{\mu^{2}/4 -
55  n^{2}\pi^{2}}.
56  \ \ \ \ \ \ \ \ \ \ (3) \f]
57 If the shift \f$\mu = 0\f$, then the eigenvalues are all imaginary and
58  consist of the complex conjugate pairs given by \f$
59  \lambda{n} = \pm n \pi\, \mbox{i} \f$.
60 
61  If \f$ \mu^{2}/4 >
62  \pi^{2}\f$ \f$ \Rightarrow \f$ \f$ \mu > 2\pi \f$, then the first complex
63  conjugate pair of imaginary eigenvalues becomes two distinct real
64  eigenvalues because the term under the square root in (3) becomes
65  positive. As \f$ \mu \f$ increases further subsequent complex
66  eigenvalues merge to be real. Thus, we can control the spectrum by
67  modifying the parameter \f$ \mu \f$.
68 
69 We now provide a detailed discussion of the driver code
70 <A HREF="../../../../demo_drivers/eigenproblems/harmonic/complex_harmonic.cc">
71 complex_harmonic.cc</A> which solves the problem for the first four
72 eigenvalues, when \f$ \mu = 6.5 > 2\pi. \f$

77 \section theory Overview of the theory

79 \subsection weak Weak formulation of the problem
80 
81  If we wish to solve the problem (1) using finite elements,
82 we simply multiply each equation by a test function, but do not
83 integrate by parts. We shall use the same test functions for each
84 variable and thus our governing equations become
85 \f[ \int_{D} \frac{\partial u}{\partial x} \phi^{(test)} \,\mbox{d} x =
86  (\lambda -\mu) \int_{D} w\, \phi^{(test)}\,\mbox{d} x, \quad\quad
87  \int_{D} \frac{\partial w}{\partial x} \phi^{(test)} \,\mbox{d} x =
88  \lambda \int_{D} u\, \phi^{(test)}\,\mbox{d} x, \quad
89  \f]
90 
91 If we expand \f$ u(x) = U_{j} \psi_{j}(x) \f$ and \f$ w(x) = W_{j}
92 \psi_{j}(x) \f$ in terms of
93 known basis functions \f$ \psi_{j} \f$ and use
94 the same basis functions as our test functions (Galerkin method), then the
95 weak form becomes
96 \f[ \int_{D} \frac{\partial \psi_{j}}{\partial x} \psi_{i}
97 \,\mbox{d} x U_{j} = (\lambda - \mu) \int_{D}
98 \psi_{i} \psi_{j}\,\mbox{d} x W_{j},\quad\quad
99  \int_{D} \frac{\partial \psi_{j}}{\partial x} \psi_{i}
100 \,\mbox{d} x W_{j} = \lambda \int_{D}
101 \psi_{i} \psi_{j}\,\mbox{d} x U_{j}.
102  \f]
103 Thus, in order to form the discrete eigenproblem
104 \f$ J_{ij} v_{j} = \lambda M_{ij} v_{j} \f$, we choose to order the
105 unknowns in the form
106 \f$ v = [ U | W ]^{T} \f$, leading to the block Jacobian and mass matrices
107 \f[ J = \left[\begin{array}{c|c}
108  0 & A \\\hline
109  A & \mu B \end{array}\right],\quad
110  M = \left[\begin{array}{c|c}
111  B & 0 \\\hline
112  0 & B \end{array}\right], \f]
113  where
114  \f[ A_{ij} = \int_{D} \frac{\partial \psi_{j}}{\partial x} \psi_{i}
115 \,\mbox{d} x , \quad\quad B_{ij} = \int_{D}
116 \psi_{i} \psi_{j}\,\mbox{d} x. \f]

123  The implementation closely follows that in the

127 \subsection equations Creating the elements
128 
129  For generality, we implement the mathematics to assemble contributions to the
130 Jacobian and mass matrices defined above in the
131 class \c ComplexHarmonicEquations that inherits from \c
132 FiniteElement.
133 
134 \dontinclude complex_harmonic.cc
135 \skipline A class
136 \until ComplexHarmonicEquations()
137 
138 The unknowns that represent the discretised eigenfunction are assumed
139 to be stored at the nodes, but there are now two unknowns: \f$u\f$ is
140 assumed to be the first and \f$w\f$ is the second.

143 \until nodal_value(n,1)

145 As before, the key function is \c
146 fill_in_contribution_to_jacobian_and_mass_matrix

152  Note that the \b shift \f$\mu\f$ is implemented in a global namespace
153  as \c EigenproblemShift::Mu.

155  The shape functions are specified in the \c QComplexHarmonicElement
156 class that inherits from our standard one-dimensional Lagrange elements
157  \c QElement<1,NNODE_1D> as well as \c HarmonicEquations.

160  specified in the \c ComplexHarmonicEquations class, whereas the shape
161  functions are provided by the \c QElement<1,NNODE_1D> class.

175 \section problem The problem class
176 The \c ComplexHarmonicProblem is derived from \c oomph-lib's generic
177 \c Problem class and the specific element
178 type and eigensolver are specified as template parameters to make it easy for
179 the "user" to change either of these from the driver code. Once again,
180 it is very similar to the \c HarmonicProblem class in the
181 <a href="../../harmonic/html/index.html"> harmonic eigenproblem
182 </a>. The only member function with any differences is the
183  \c solve(...) function, which requests 7 rather than 4 eigenvalues to
184  be computed and outputs the eigenfunction associated with eigenvalue
185  of smallest magnitude (which is real).

190 \section exercises Comments and exercises
191 
192 -# Modify the code to compute a different number of eigenvalues.
193  What is the maximum number of eigenvalues that could be computed?
194 -# Confirm that the eigenvalues agree with the analytic result and
195  that a transition from real to complex values occurs when \f$ \mu
196  = 2\pi \f$.
197 -# Explain why there are always two eigenvalues with the values
198  \f$\lambda = \mu\f$. What are the corresponding eigenfunctions?
199 -# When the output is a complex conjugate pair of eigenvalues, the two
200  associated eigenvectors are the real and imaginary parts of the
201  eigenvector corresponding to the first eigenvalue.
202  The complex conjugate eigenvalue has a complex
203  conjugate eigenvector (can you prove this?), so no more information
204  is required. Modify the output
205  function to examine the real and imaginary parts of a complex
206  eigenvalue. Are the results what you expect?
