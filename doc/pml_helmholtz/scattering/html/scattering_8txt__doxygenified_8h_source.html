3\mainpage Example problem: The Helmholtz equation with perfectly matched layers
4
5
6
7In this document we discuss the finite-element-based solution of the Helmholtz
8equation with the Summerfeld boundary condition, an elliptic PDE that describes
9time-harmonic wave propagation problems. Compared to the
10"standard form" of the Helmholtz equation, discussed in
11<a href="../../../helmholtz/scattering/html/index.html">
12another tutorial,</a> the formulation used here allows the
13imposition of the Sommerfeld radiation condition by means of
14so-called "perfectly matched layers" (PMLs) as an alternative
15to classical absorbing/approximate boundary conditions or DtN maps.
16
17We start by reviewing the relevant theory and then
18present the solution of a simple model problem -- the outward propagation of
19waves from the surface of a cylinder.

41\section theory Theory: The Helmholtz equation for time-harmonic scattering problems
42The Helmholtz equation governs time-harmonic solutions of problems
43governed by the linear wave equation
44\f[
45\nabla( c^2 \nabla U(x,y,t)) = \frac{\partial^2 U(x,y,t)}{\partial t^2},
46 \ \ \ \ \ \ \ \ \ \ \ \ (1)
47\f]
48where \f$ c \f$ is the wavespeed. Assuming that \f$ U(x,y,t) \f$
49is time-harmonic, with frequency \f$ \omega \f$, we write the real
50function \f$ U(x,y,t) \f$ as
51\f[
52U(x,y,t) =Re (u(x,y) \ e^{-i \omega t})
53\f]
54where \f$ u(x,y) \f$ is complex-valued. If the wavespeed is constant
55this transforms (1) into the
56<a href="../../../helmholtz/scattering/html/index.html">
57standard form of the Helmholtz equation</a>
58\f[
59\nabla^2 u(x,y) + k^2 u(x,y) = 0
60 \ \ \ \ \ \ \ \ \ \ \ \ (2)
61\f]
62where
63\f[
64k = \frac{\omega}{c}
65 \ \ \ \ \ \ \ \ \ \ \ \ (3)
66\f]
67is the wave number. Like other elliptic PDEs the Helmholtz equation
68admits Dirichlet, Neumann (flux) and Robin boundary conditions.
69
70If the equation is solved in an infinite domain (e.g. in scattering
71problems) the solution must satisfy the so-called
72<a href="http://en.wikipedia.org/wiki/Sommerfeld_radiation_condition">
73Sommerfeld radiation condition</a> which in 2D has the form
74\f[
75\lim_{r\to \infty} \sqrt{r} \left(\frac{\partial u}{\partial r} - iku
76\right) =0.
77\f]
78Mathematically, this condition is required to ensure the uniqueness
79of the solution (and hence the well-posedness of the problem).
80In a physical context, such as a scattering problem, the condition
81ensures that scattering of an incoming wave only produces outgoing not
82incoming waves from infinity.

87\section discr Discretisation by finite elements
88We provide separate storage for the real and imaginary parts of the solution --
89each \c Node therefore stores two unknowns values. By default,
90the real and imaginary parts are stored as values 0 and 1,
91respectively; see the section \ref numbering for details.
92
93The application of Dirichlet and Neumann boundary conditions is
94straightforward:
95- Dirichlet conditions are imposed by pinning the relevant nodal
96 values and setting them to the appropriate prescribed values.
97- Neumann (flux) boundary conditions are imposed via
98 \c FaceElements (here the \c PMLHelmholtzFluxElements).
99 <a href="../../../poisson/two_d_poisson_flux_bc/html/index.html">
100 As usual</a> we attach these to the faces of the "bulk" elements
101 that are subject to the Neumann boundary conditions.
102.
103
104The imposition of the Sommerfeld radiation condition for problems in
105infinite domains is slightly more complicated.
106<a href="../../../helmholtz/scattering/html/index.html">
107Another tutorial</a> shows how to impose this condition by
108means of absorbing/approximate boundary conditions or DtN maps.
109In the next section we will discuss an alternative approach to this
110problem by means of perfectly matched layers.

114\subsection pml Perfectly matched layers
115
116The idea behind perfectly matched layers is illustrated in the
117figure below. The actual physical/mathematical problem
118has to be solved in the infinite domain \f$ D \f$ (shown on the left),
119with the Sommerfeld radiation condition ensuring the suitable decay
120of the solution at large distances from the region of interest
121(the vicinity of the scatterer, say).
122
123If computations are performed in a finite computational domain, \f$
124D_c \f$ , (shown in the middle), spurious wave reflections are likely to
125be generated at the artificial boundary \f$
126\partial D_c \f$ of the computational domain.
127
128The idea behind PML methods is to surround the actual computational
129domain \f$ D_c \f$ with a layer of "absorbing" material whose
130properties are chosen such that the outgoing waves are absorbed
131within it, without creating any artificial reflected waves at the interface
132between the PML layer and the computational domain.
133
134
135\image html pml.gif "Sketch illustrating the idea behind perfectly matched layers. "
136\image latex pml.eps "Sketch illustrating the idea behind perfectly matched layers. " width=0.75\textwidth
137
138Our implementation of the perfectly matched layers follows the
139development in
140<a href="http://www.sciencedirect.com/science/article/pii/S0021999106004487">
141A. Bermudez, L. Hervella-Nieto, A. Prieto, and
142R. Rodriguez "An optimal perfectly matched layer with unbounded
143absorbing function for time-harmonic acoustic scattering problems"
144Journal of Computational Physics <b>223</b> 469-488 (2007)</a>
145and we assume the boundaries of the computational domain to be aligned
146with the coordinate axes, as shown in the sketch above.
147
148The method requires a slight further generalisation of the equations,
149achieved by introducing the complex coordinate mapping
150\f[
151\frac{\partial}{\partial x_j} \to \frac{1}{\gamma_j} \frac{\partial}{\partial x_j}
152 \ \ \ \ \ \ \ \ \ \ \ \ (4)
153\f]
154within the perfectly matched layers. This makes the problem
155anisotropic and in 2D we have
156 \f[
157 \nabla ^ 2 u =
158\frac{1}{\gamma_x}\frac{\partial}{\partial x} \left(
159\frac{1}{\gamma_x}\frac{\partial u}{\partial x} \right) +
160\frac{1}{\gamma_y}\frac{\partial}{\partial y}\left(
161\frac{1}{\gamma_y}\frac{\partial u}{\partial y}\right)
162 \ \ \ \ \ \ \ \ \ \ \ \ (5)
163\f]
164
165The choice of \f$ \gamma_x \f$ and
166\f$ \gamma_y \f$ depends on the orientation of the
167PML layer. Since we are restricting ourselves to axis-aligned
168mesh boundaries we need to distinguish three different cases, as
169shown in the sketch below:
170
171
172\image html pml_meshes.gif "Sketch illustrating the geometry of the perfectly matched layers. "
173\image latex pml_meshes.eps "Sketch illustrating the geometry of the perfectly matched layers. " width=0.75\textwidth
174
175- For layers that are aligned with the y axis (such as the left
176 and right PML layers in the sketch) we set
177 \f[
178 \gamma_{x}(x) =
179 1 +\frac{i}{k}\ \sigma_{x}(x)
180 \ \ \ \ \ \ \ \mbox{ \ \ \ with \ \ \ }
181 \sigma_{x}(x) = \frac{1}{|X_{PML}-x|}, \ \ \ \ \
182 (6)
183 \f]
184 where \f$ X_{PML} \f$ is the x-coordinate of the outer boundary
185 of the PML layer, and
186 \f[
187 \gamma_y = 1.
188 \f]
189- For layers that are aligned with the x axis (such as the top
190 and bottom PML layers in the sketch) we set
191 \f[
192 \gamma_x = 1,
193 \f]
194 and
195 \f[
196 \gamma_{y}(y) =
197 1+\frac{i}{k} \ \sigma_{y}(y)
198 \ \ \ \ \ \ \ \mbox{ \ \ \ with \ \ \ }
199 \sigma_{y}(y) = \frac{1}{|Y_{PML}-y|}, \ \ \ \ \
200 (7)
201 \f]
202 where \f$ Y_{PML} \f$ is the y-coordinate of the outer boundary
203 of the PML layer.
204- In corner regions that are bounded by two axis-aligned PML layers
205 (with outer coordinates \f$ X_{PML} \f$ and
206 \f$ Y_{PML} \f$) we set
207 \f[
208 \gamma_{x}(x) =
209 1 +\frac{i}{k}\ \sigma_{x}(x)
210 \ \ \ \ \ \ \ \mbox{ \ \ \ with \ \ \ }
211 \sigma_{x}(x) = \frac{1}{|X_{PML}-x|} \ \ \ \ \
212 (8)
213 \f]
214 and
215 \f[
216 \gamma_{y}(y) =
217 1+\frac{i}{k} \ \sigma_{y}(y)
218 \ \ \ \ \ \ \ \mbox{ \ \ \ with \ \ \ }
219 \sigma_{y}(y) = \frac{1}{|Y_{PML}-y|}. \ \ \ \ \
220 (9)
221 \f]
222- Finally, in the actual computational domain (outside
223 the PML layers) we set
224 \f[
225 \gamma_{x}(x) = \gamma_{y}(y) = 1.
226 \f]
227.

232\subsection impl Implementation of the perfectly matched layers within oomph-lib
233
234The finite-element-discretised equations (2)
235(modified by the PML terms discussed above) are implemented in
236the \c PMLHelmholtzEquations<DIM> class which is templated
237by the spatial dimension, \c DIM. As usual, we provide
238fully functional elements by combining these with geometric
239finite elements (from the Q and T families -- corresponding (in 2D)
240to triangles and quad elements). By default, the PML modifications
241are disabled, i.e. \f$ \gamma_{x}(x) \f$ and \f$ \gamma_{y}(y) \f$ are both
242set to 1.
243
244 The generation of suitable 2D PML meshes along the axis-aligned
245boundaries of a given bulk mesh is facilitated by helper
246functions which automatically erect layers of (quadrilateral)
247PML elements. The layers are built from
248\c QPMLHelmholtzElement<2,NNODE_1D> elements and
249the parameter \c NNODE_1D is automatically chosen to match that of
250the elements in the bulk mesh. The bulk mesh can contain quads or triangles
251(as shown in the specific example presented below).
252
253For instance, to erect a PML layer (of width \c width, with \c n_pml elements
254across the width of the layer) on the "right" boundary
255(with boundary ID \c b_bulk) of the bulk mesh pointed to by \c
256bulk_mesh_pt, a call to
257\code
258PMLHelper::create_right_pml_mesh(bulk_mesh_pt, b_bulk, n_pml, width);
259\endcode
260returns a pointer to a newly-created mesh that contains the
261PML elements which are automatically attached to the
262boundary of the bulk mesh (i.e. the \c Nodes on the
263outer boundary of the bulk mesh are shared (pointed to), rather than
264duplicated, by the
265elements in the PML mesh). The PML-ness of the elements is
266automatically enabled, i.e. the functions \f$ \gamma_x(x) \f$
267and \f$ \gamma_y(y) \f$ are set as described above. Finally,
268zero Dirichlet boundary conditions are applied to the real and
269imaginary parts of the solution on the outer boundary of the
270PML layer.
271
272Similar helper functions exist for PML layers on other axis-aligned
273boundaries, and for corner PML meshes; see the code listings
274provided below. Currently, we only provide this functionality for
275convex 2D computational domains, but the generalisation to non-convex
276boundaries and 3D is straightforward (if tedious) to implement
277(Any volunteers?).

283\section scattering A specific example: Outward propagation of acoustic waves from the surface of a cylindrical object
284
285We will now demonstrate the methodology for a specific example:
286the propagation of axisymmetric waves from the surface of a
287circular disk. This is a good test case because any deviations from the
288axisymmetry of the (exact) solution by spurious reflections
289from the boundaries of the computational domain are easy to
290detect visually.
291
292
293The specific domain used in this case can be seen in the figure below.
294We create an unstructured mesh of six-noded \c TPMLHelmholtzElements
295to create the finite computational domain surrounding a circular disk.
296This is surrounded by four axis-aligned PML layers and four
297corner meshes (each made of nine-noded \c QPMLHelmholtzElements).

489\section comm_ex Comments and Exercises
490\subsection numbering The enumeration of the unknowns
491As discussed in the introduction, most practically relevant
492solutions of the Helmholtz equation are complex valued. Since \c oomph-lib's
493solvers only deal with real (double precision) unknowns, the equations
494are separated into their real and imaginary parts.
495In the implementation of the Helmholtz elements, we store the real
496and imaginary parts of the solution as two separate values at each
497node. By default, the real and imaginary parts are accessible via
498\c Node::value(0) and \c Node::value(1). However,
499to facilitate the use of the elements
500in multi-physics problems we avoid accessing the unknowns
501directly in this manner but provide the virtual function
502\code
503std::complex<unsigned> HelmholtzEquations<DIM>::u_index_helmholtz()
504\endcode
505which returns a complex number made of the two unsigneds that indicate
506which nodal value represents the real and imaginary parts of the solution.
507This function may be overloaded in combined multi-physics elements
508in which a Helmholtz element is combined (by multiple inheritance)
509with another element, using the strategy described in
510<a href="../../../multi_physics/b_convection/html/index.html">
511the Boussinesq convection tutorial</a>.
512
513\subsection dampingfct PML damping functions
514The choice for the absorbing functions in our implementation of
515the PMLs is not unique. There are alternatives varying in both order and
516continuity properties. The current form is the result of several
517feasibility studies and comparisons found in both
518<a href="http://www.sciencedirect.com/science/article/pii/S0021999106004487">
519Bermudez et al.</a>
520and in the relevant papers on
521<a href="http://imperial.academia.edu/RaduCimpeanu/Papers">Radu
522Cimpeanu's webpage</a>. These damping functions produce an
523acceptable result in most practical situations without further
524modifications. For very specific applications, alternatives may
525need to be used and can easily be implemented within the
526existing framework.

530\subsection ex Exercises
531\subsubsection pmlsize Changing perfectly matched layer parameters
532Confirm that only a very small number of PML elements (across the thickness
533of the PML layer) is required to effectively damp the outgoing waves.
534Furthermore, show that (and try to explain why) PMLs with too many
535elements may not perform as expected.
536
537A second parameter
538that can be adjusted is the geometrical thickness of the perfectly matched
539layers. Relative thin layers layer tend to perform better than thick layers
540with few elements across their width. Confirm this and try to
541find an explanation for the phenomenon, given the form of the
542absorbing functions used in the complex coordinate transformation.
543
544
545\subsubsection large_k Large wavenumbers
546For Helmholtz problems in general, ill-conditioning appears as the
547wavenumber becomes very large. By altering wavespeed and/or frequency,
548explore the limitations of both the mesh and the solver in terms
549of this parameter. Try adjusting the target element size in order to alleviate
550resolution-related effects. Assess the effectiveness of the perfectly matched
551layers in high wavenumber problems.
552
553\subsubsection adaptivity Spatial adaptivity
554The driver code discussed above already contains the straightforward
555modifications required to enable spatial adaptivity. Explore this
556(by recompiling the code with -DADAPTIVE) and explain why spatial
557adaptivity is not particularly helpful for the test problem
558discussed above.
559
560\subsubsection diff_order Linear and cubic finite elements
561The driver code also contains (commented out) modifications
562that allow the simulation to be performed with
563three-node (linear) and ten-node (cubic) triangles. Explore the
564performance of these elements and confirm that the helper functions
565correctly create matching (four-node and sixteen-node) quad elements
566in the PML layers.
567
568\subsubsection default Default values for problem parameters
569Following our usual convention, we provide default values
570for problem parameters where this is sensible. For instance, the PML mapping
571function defaults to the one proposed by Bermudez et al. as this appears to be
572optimal.
573Some parameters, such as the wavenumber squared \f$ k^2 \f$ do need to be set
574since there are no obvious defaults. If \c oomph-lib is compiled in \c PARANOID
575mode, an error is thrown if the relevant pointers haven't been set.
576Without paranoia, you get a segmentation fault...
577
578Confirm that this is the case by commenting out the relevant
579assignments.
580
581\subsubsection non_convex Non-convex PML boundaries
582As discussed above, we currently provide helper functions to
583attach PML layers to axis-aligned boundaries of 2D meshes with
584convex outer boundaries. Essentially, this restricts us to
585rectangular computational domains. Extend this capability by developing
586methodologies to
587- deal with non-convex domain boundaries. We suggest you
588 create PML meshes for the non-convex corners first, then
589 create the axis-aligned meshes (note that these have to
590 share nodes with the already-created elements that occupy the
591 non-convex corners), and then create the corner meshes
592 for the convex corners (as before). When you're done, let
593 us know -- this would be a really useful addition to
594 oomph-lib's machinery. We're happy to help!
595- Repeat the same exercise in 3D -- somewhat less trivial
596 (so we're even keener for somebody to have a go!)
597.
