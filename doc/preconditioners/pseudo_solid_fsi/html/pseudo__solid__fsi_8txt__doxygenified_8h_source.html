3\mainpage Demo problem: A preconditioner for the solution of fluid-structure interaction problems with (pseudo-)solid fluid mesh updates
4
5In this tutorial we re-visit the solution of fluid-structure
6interaction problems with (pseudo-)solid fluid mesh updates,
7focusing on the efficient solution of the governing equations
8by GMRES, using the problem-specific block-preconditioner developed
9in
10<center>
11Muddle, R.L., Mihajlovic, M. & Heil, M. (2012)
12An efficient preconditioner for monolithically-coupled
13large-displacement fluid-structure interaction problems
14with pseudo-solid mesh updates. <em>Journal of Computational Physics</em>
15<b>231</b>, 7315-7334. DOI:
16<a href="http://dx.doi.org/10.1016/j.jcp.2012.07.001">
1710.1016/j.jcp.2012.07.001</a>
18</center>
19where full details of the analysis and further results from a
20variety of other test problems can be found.

26\section model Background: Problem formulation, the resulting linear system and the preconditioner
27
28We will demonstrate the development and application
29of the preconditioner using the
30problem of flow in a 2D channel with an elastic leaflet discussed in
31<a href="../../../interaction/fsi_channel_with_leaflet/html/index.html">
32another tutorial</a> and illustrated in the sketch below:
33
34\image html fsi_channel_with_leaflet.gif "Sketch of the problem. "
35\image latex fsi_channel_with_leaflet.eps "Sketch of the problem. " width=0.75\textwidth
36
37Flow is driven (via an imposed parabolic inflow profile) through a 2D channel
38which is partially obstructed by a thin-walled elastic leaflet.
39The fluid leaves the domain at the far downstream end of the channel
40where we assume parallel, axially traction-free outflow.
41
42We use the same discretisation (2D quadrilateral
43Taylor-Hood elements for the fluid; 1D Hermite beam elements for the
44leaflet) as in the
45<a href="../../../interaction/fsi_channel_with_leaflet/html/index.html">
46original tutorial</a>. However, here we perform the fluid mesh update
47in response to the deformation of the elastic leaflet using a pseudo-elastic
48approach. This is done by "wrapping" the underlying \c
49QTaylorHoodElement into a \c PseudoSolidNodeUpdateElement
50and employing \c ImposeDisplacementByLagrangeMultiplierElements
51to impose the deformation of the elastic leaflet onto the (pseudo-solid)
52fluid mesh. The following tutorials discuss the
53relevant methodologies:
54- <a href="../../../solid/prescribed_displ_lagr_mult/html/index.html">
55 The use of Lagrange multipliers to impose displacement constraints
56 onto (pseudo-)solids.</a>\n\n
57- <a href="../../../interaction/unstructured_fsi/html/index.html">
58 The use of pseudo-elasticity to deform the fluid mesh in FSI
59 problems.</a> \n\n
60.
61
62The discretised problem contains the
63following types of discrete unknowns:
64- The fluid degrees of freedom (velocities and pressures).\n\n
65- The nodal positions (real and generalised; the latter representing
66 derivative degrees of freedom in the Hermite elements used to
67 discretise the leaflet; see the
68 <a href="../../../beam/tensioned_string/html/index.html">
69 beam theory tutorial</a> for details) of the elastic leaflet.\n\n
70- The nodal positions in the fluid mesh. \n\n
71- The nodal values representing the components of the (vector-valued)
72 Lagrange multipliers which impose the motion of the elastic leaflet
73 on the pseudo-elastic fluid mesh. These unknowns only exist for the
74 fluid nodes on the FSI boundary.
75.
76Using this classification of the unknowns,
77the linear system to be solved in the course
78of the Newton iteration can be re-ordered
79into the following block structure:
80\f[
81J_{\rm FSI} \ \Delta {\bf x} =
82 \left[
83\begin{array}{c|c|cc}
84F& &C_{\rm fx}&\\ \hline C_{\rm sf}&S&C_{\rm sx}&\\ \hline
85&&E&C_{\rm xl}\\ &C_{\rm ls}&C_{\rm lx}&\\
86\end{array}
87\right] \left[
88\begin{array}{c}
89 \Delta {\bf F}\\
90\hline \Delta {\bf S}\\
91\hline \Delta {\bf X}\\
92\Delta {\bf L}\\
93\end{array}
94\right] =- \left[
95\begin{array}{c}
96{\bf r}_{\rm f}\\
97\hline {\bf r}_{\rm s}\\
98\hline {\bf r}_{\rm x}\\
99{\bf r}_{\rm l}\\
100\end{array}
101\right]
102\ \ \ \ \ \ \ \ (1)
103\f]
104where the vector \f${\bf F}\f$ contains the Navier-Stokes (fluid) unknowns
105(velocities and pressure), \f${\bf S}\f$ represents unknowns describing the
106deformation of the fluid-loaded solid, \f${\bf X}\f$ represents
107the nodal positions in the fluid mesh, and \f${\bf L}\f$ contains
108the discrete Lagrange multipliers which impose
109the deformation of the FSI boundary in the fluid mesh.
110The three diagonal blocks in the Jacobian matrix, \f$J_{\rm FSI}\f$, are
111the two ``single-physics''
112Jacobian matrices (\f$F\f$, the Navier-Stokes Jacobian; \f$S\f$, the
113tangent stiffness matrix of the fluid-loaded solid) and
114the Jacobian,
115\f[
116J_{\rm PS} =
117\left[
118\begin{array}{cc}
119E&C_{\rm xl}\\
120C_{\rm lx}&\\
121\end{array}
122\right]
123\f]
124associated with the Lagrange-multiplier-constrained pseudo-elasticity
125problem governing the fluid mesh update, discussed in more detail
126in <a href="../../prescribed_displ_lagr_mult/html/index.html">
127another tutorial</a>.
128The non-zero off-diagonal blocks arise through the interactions between
129the various single-physics
130problems: \f$C_{\rm fx}\f$ represents the effect of the pseudo-solid unknowns
131(the nodal positions in the fluid mesh) on the discretised
132Navier--Stokes equations -- this incorporates the so-called shape-derivatives
133and the time-discretised version of the no-slip condition on the FSI
134boundary which expresses the fluid velocities in
135terms of the nodal velocities of the fluid mesh.
136\f$C_{\rm sf}\f$ captures the effect of the fluid traction (pressure and shear
137stresses) on the fluid-loaded solid. Since the shear stresses depend
138on gradients of the fluid velocity, the traction
139is also affected by the nodal positions in the fluid mesh -- this
140dependency gives rise to the matrix \f$C_{\rm sx}\f$. Finally, \f$C_{\rm
141ls}\f$ arises because the prescribed boundary displacement of the fluid
142mesh depends
143on the displacement field in the fluid-loaded solid. The zero \f$ "fs"
144\f$ block
145reflects the fact that the discretised Navier-Stokes equations do not
146depend directly on the displacement field of the actual solid. The
147zero \f$ "xf" \f$ block indicates that the
148pseudo-solid equations are not affected by the fluid unknowns. The
149\f$ "xs"\f$ block is zero
150because the real solid affects the pseudo-solid only indirectly
151via the Lagrange multiplier constraint (which gives rise to \f$C_{\rm
152ls}\f$). Finally, the non-zero blocks in the last block column
153reflect the fact that the Lagrange multiplier is only used to
154enforce the displacement constraint
155in a one-way coupling in which the real solid affects the pseudo-solid
156but not vice versa.
157See <a href="http://dx.doi.org/10.1016/j.jcp.2012.07.001">
158Muddle, Mihajlovic & Heil (2012) </a> for more detail.
159
160To construct a preconditioner for the linear system
161(1) we replace the
162bottom-right \f$2 \times 2\f$ block in the Jacobian with the pseudo-solid
163preconditioner,
164\f[
165{\cal P}_{\rm PS} = \left[
166\begin{array}{cc}
167E_{\rm aug}&\\
168&W
169\end{array}
170\right]
171\ \ \ \ \ \ \ \ \ \ \ \ \ \ (2)
172\f]
173which we discussed in
174<a href="../../prescribed_displ_lagr_mult/html/index.html">
175another tutorial</a>. This yields the pseudo-solid FSI preconditioner
176\f[
177{\cal P}_{\rm FSI}=\left[
178\begin{array}{c|c|cc}
179F& &C_{\rm fx}&\\ \hline
180C_{\rm sf}&S&C_{\rm sx}&\\ \hline
181&&E_{\rm aug}&\\
182&C_{\rm ls}&&W
183\end{array}
184\right].
185\ \ \ \ \ \ (3)
186\f]
187
188The preconditioner can be shown to have a block triangular structure.
189Its application therefore
190requires the solution of four subsidiary linear systems involving the
191matrices
192\f$F\f$, \f$S\f$, \f$E_{\rm aug}\f$ and \f$W\f$ and
193four sparse matrix-vector products with the interaction
194matrices \f$C_{\rm fx}\f$, \f$C_{\rm sx}\f$, \f$C_{\rm sf}\f$ and
195\f$C_{ls}\f$.
196
197Numerical experiments in <a href="http://dx.doi.org/10.1016/j.jcp.2012.07.001">
198Muddle, Mihajlovic & Heil (2012) </a> show that an efficient
199implementation of the preconditioner is obtained by
200- replacing the solution of the linear system involving
201 \f${\cal P}_{\rm PS}\f$ by the approximation \f$\widetilde{\cal P}_{\rm PS}\f$
202 discussed in <a href="../../prescribed_displ_lagr_mult/html/index.html">
203 another tutorial,</a> \n\n
204- replacing the solution of the linear system involving the
205 Navier-Stokes Jacobian \f$ F \f$ by an application of
206 Elman, Silvester and Wathen's
207 <a href="../../lsc_navier_stokes/html/index.html">
208 Least-Squares Commutator (LSC)
209 preconditioner</a>,
210.
211and by replacing the remaining block-solves within these preconditioners
212by a small number of AMG cycles or CG iterations. We retain
213SuperLU as the (exact) direct solver for the linear system involving
214the real solid's tangent stiffness matrix \f$ S \f$.
215
216With these approximations, the computational
217cost of one application of the preconditioner is linear
218in the number of unknowns (see \ref comm_and_ex
219for a more detailed discussion of this issue).
220The optimality of the preconditioner
221can therefore be assessed by demonstrating that
222the number of GMRES iterations remains constant under
223mesh refinement.

229\section implementation Implementation and use of the preconditioner
230
231The preconditioner described above is implemented within \c oomph-lib's
232(parallel) block preconditioning framework which is described in
233<a href="../../../mpi/block_preconditioners/html/index.html">
234another tutorial</a>. For the purpose of the implementation,
235we decompose the preconditioning matrix into the 3x3
236main blocks indicated by the vertical and horizontal lines
237in (1) and (3).
238
239The degrees of freedom within these sub-blocks are classified
240into "dof-types" on an element-by-element basis, using the functions
241- \c GeneralisedElement::ndof_types(...) which returns the number of
242 dof types classified by that element.
243- \c GeneralisedElement::get_block_numbers_for_unknowns(...) which
244 associates each degree of freedom (identified by its global equation
245 number) with the dof type within its block.
246 \n\n
247.
248The default implementation of these functions within the
249Navier-Stokes elements (which differentiate between the
250fluid velocity components and the pressure)
251and the beam elements (which do not distinguish between the different types
252of degree of freedom) is appropriate for the use with the
253(subsidiary) preconditioners employed here. The degrees of freedom
254representing the nodal positions in the (pseudo-solid) fluid mesh
255need to be sub-divided into those that are and are not constrained
256by the Lagrange multipliers that impose the deformation of the
257FSI boundary. This is most conveniently done by overloading the relevant
258functions in a templated wrapper element, \c PseudoElasticBulkElement<...>,
259as discussed in <a href="../../prescribed_displ_lagr_mult/html/index.html">
260another tutorial.</a>

301As expected from the theory (see
302<a href="http://dx.doi.org/10.1016/j.jcp.2012.07.001">
303 Muddle, Mihajlovic & Heil (2012)</a>), the GMRES iteration counts
304are small and virtually mesh independent for the
305exact implementation of the preconditioner. Replacing the (costly) exact
306solves by (faster) approximate solves leads to a modest
307increase in the (absolute) number of GMRES iterations. The most
308significant deterioration of the iteration counts results from the
309use of the LSC Navier Stokes preconditioner as the inexact solver
310(subsidiary preconditioner) for the Navier-Stokes block.
311The behaviour observed here (slight mesh dependence for
312steady solves; virtual mesh independence (once the mesh is
313sufficiently fine) for unsteady solves) mirrors that observed
314in single-physics Navier-Stokes problems.
315
316The benefit of switching to the approximate solves becomes apparent
317in the next two tables which shows the average cpu times required
318for the solution of the linear systems by GMRES.
319(For comparison the times labelled as <b>"SuperLU"</b> show the times
320when the linear systems (1)
321are solved directly by SuperLU rather than by GMRES; they illustrate
322how quickly direct solvers become uncompetitive.) For sufficiently
323fine discretisations the larger number of GMRES iterations for the
324inexact solves is more than compensated for by the much lower cost
325of the preconditioning operations. The final
326implementation yields cpu times that (at least for unsteady problems)
327 are proportional to the number of unknowns -- the hallmark of an optimal solver.

350\section modifications Modifications to the driver code
351Most of the driver code is unchanged from the implementation
352discussed in the <a href="../../../interaction/fsi_channel_with_leaflet/html/index.html">original
353tutorial.</a> We therefore only discuss the modifications required
354- to perform the fluid mesh update using (pseudo-)elasticity,
355.
356and
357- to employ preconditioned GMRES as the linear solver in the Newton method.
358.
359<HR>
360
361\subsection pseudo-solid Fluid-mesh udate by pseudo-elasticity
362To perform the node update in the fluid mesh in response to the
363deformation of the leaflet using the equations of
364large-displacement elasticity we "wrap" the Navier-Stokes
365element in the doubly-templated
366\c PseudoSolidNodeUpdateElement<WRAPPED_ELEMENT, SOLID_ELEMENT> class.
367The first template argument of this class, \c WRAPPED_ELEMENT,
368specifies the type of the "wrapped"
369element -- here the nine-node, 2D quadrilateral \c
370QTaylorHoodElement<2>. The second template argument, \c SOLID_ELEMENT,
371specifies the \c SolidElement used to perform the node update -- here
372the nine-node, 2D quadrilateral \c QPVDElement<2,3>. Since the
373preconditioner requires the (re-)classification of the pseudo-solid
374degrees of freedom into constrained and unconstrained nodal positions,
375we wrap this element too, using the
376\c PseudoElasticBulkElement already discussed in
377<a href="../../prescribed_displ_lagr_mult/html/index.html">
378another tutorial</a>. The full specification of the
379fluid element is therefore given by
380\code
381PseudoSolidNodeUpdateElement<QTaylorHoodElement<2>,
382 PseudoElasticBulkElement<QPVDElement<2,3> > >
383\endcode
384

435

508\section comm_and_ex Comments and Exercises

510\subsection comm Comments
511- Further details of the theory, such as the proof of the optimality of
512 the preconditioner, can be found in
513 <a href="http://dx.doi.org/10.1016/j.jcp.2012.07.001">
514 Muddle, Mihajlovic & Heil (2012).</a>\n\n
515- Note that we retained SuperLU as the solver for the linear system
516 involving the tangent stiffness matrix \f$ S \f$ of the actual,
517 fluid-load solid. In the present example this still leads to an
518 optimal solver since the leaflet is modelled as a 1D beam structure
519 whose number of unknowns increases much more slowly than the number
520 of unknowns in the fluid mesh when the meshes are refined uniformly.
521 If the solid is modelled as a "proper" solid, the use of a direct
522 solver for the solution of the linear systems involving \f$ S \f$
523 becomes sub-optimal. The member function
524 \code
525 PseudoElasticFSIPreconditioner::set_solid_preconditioner(...)
526 \endcode
527 can then be used to specify a subsidiary preconditioner
528 (inexact solver) for the linear systems involving \f$ S \f$.
529 \n\n
